A <GlossTerm baseform="saved input">saved input</GlossTerm> is a reusable value or credential that you store once and map into <GlossTerm baseform="rde session">sessions</GlossTerm> as needed. Saved inputs are scoped to your user account rather than a single <NT><GlossTerm baseform="workspace">workspace</GlossTerm></NT>, and secret values are encrypted at rest.

## Common saved inputs

Some saved inputs have a dedicated purpose:

- **<NT>SSH public key</NT>**: registered as `SSH_PUBLIC_KEY`, this lets you connect to your sessions over SSH without a password. See [Connecting to a session](/en/bitrise-rde/rde-options/connecting-to-a-session#ssh).
- **<NT>Claude Code</NT> credential**: a `CLAUDE_CODE_OAUTH_TOKEN` or `ANTHROPIC_API_KEY` value so sessions start with <NT>Claude Code</NT> already authenticated.
- **<NT>Custom</NT> values**: any key-value pair your environment needs, such as an API token or a registry login. Mark it as secret to encrypt it.

### Configuring passwordless SSH access

To connect to your sessions over SSH without a password, add your SSH public key:

1. Select **Saved Inputs** on the left, under **User settings**.
1. Under **Passwordless SSH access**, click **+ <NT>Add</NT> public key**.

   ![Add SSH public key dialog](/img/rde/2026-06-29-rde-saved-inputs-add-ssh-key.png)

1. Paste your public key (for example, the contents of `~/.ssh/id_ed25519.pub`) into the `SSH_PUBLIC_KEY` field.
1. Click **<NT>Save</NT>**.

<NT>Bitrise</NT> adds the key to the `authorized_keys` of every new session, so you can connect with `ssh` directly.

### Configuring <NT>Claude Code</NT> credentials

To start <NT>Claude Code</NT> automatically in new sessions, add a <NT>Claude Code</NT> credential:

<Tabs>
<TabItem value="subscription-token" label="Subscription token" default>

Use this option to bill usage against your Claude Pro or Max plan.

1. Under **<NT>Claude Code</NT>**, open the **<NT>Subscription</NT> token** tab.
1. Click **+ <NT>Add</NT> subscription token**.

   ![Add Claude subscription token dialog](/img/rde/2026-06-29-rde-saved-inputs-add-subscription-token.png)

1. In your terminal, run `claude login` then `claude setup-token`. Copy the generated token (it starts with `sk-ant-oat01-`).
1. Paste the token into the `CLAUDE_CODE_OAUTH_TOKEN` field.
1. Click **<NT>Save</NT>**.

</TabItem>
<TabItem value="api-key" label="API key">

Use this option for pay-per-token billing with an Anthropic API key.

1. Under **<NT>Claude Code</NT>**, open the **<NT>API key</NT>** tab.
1. Click **+ <NT>Add API key</NT>**.

   ![Add Anthropic API key dialog](/img/rde/2026-06-29-rde-saved-inputs-api-key.png)

1. Paste your Anthropic API key (starts with `sk-ant-`) into the `ANTHROPIC_API_KEY` field.
1. Click **<NT>Save</NT>**.

</TabItem>
</Tabs>

Once saved, new sessions that include a <NT>Claude Code</NT> credential in their template will start with <NT>Claude Code</NT> already authenticated.

## Using saved inputs in a session

When you create a session, a [template](/en/bitrise-rde/configuration/templates) input can pull its value from a saved input. You can select the saved input explicitly, or let <NT>Bitrise</NT> map saved inputs to session inputs that share the same key.

Because the session references the saved input by ID, the actual value — including secrets — is resolved on the server and never exposed in the UI or in your session configuration.

## Managing saved inputs

Create, edit, and delete saved inputs from the [Bitrise RDE UI](/en/bitrise-rde/rde-options/bitrise-rde-ui), or from the CLI with `bitrise-cli rde saved-input`.
