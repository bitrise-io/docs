The Bitrise VS Code plugin brings Remote Dev Environments into your editor. Instead of connecting to an existing <GlossTerm baseform="rde session">session</GlossTerm>, you start from a local repository and let the plugin create and connect the session for you.

## What it does

With the plugin you'll be able to:

- Sign in to Bitrise from VS Code with your browser.
- Open a local repository in a brand-new session, choosing the branch, stack, and machine type — the plugin clones your repository into the session and opens it in a remote window.
- Browse your sessions in a sidebar and connect, restore, or stop them.
- Set up passwordless SSH, a Claude Code credential, and SSH agent forwarding once, so new sessions are ready to use.

Connection is handled by the VS Code [Remote - SSH](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-ssh) extension, so you edit, run terminals, and debug as if the code were on your own machine.

## Availability

The plugin is in active development and will be available from the VS Code Marketplace. Until then, use the [Bitrise RDE CLI](/en/bitrise-rde/ways-to-use-rde/bitrise-rde-cli) or the [Bitrise RDE UI](/en/bitrise-rde/ways-to-use-rde/bitrise-rde-ui) to create sessions, and connect from VS Code with Remote - SSH.
