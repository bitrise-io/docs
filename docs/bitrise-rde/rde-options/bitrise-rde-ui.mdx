The <NT>Bitrise</NT> RDE UI is the web interface for Remote Dev Environments. It's the visual way to create and manage <GlossTerm baseform="rde session">sessions</GlossTerm>, <GlossTerm baseform="rde template">templates</GlossTerm>, and <GlossTerm baseform="saved input">saved inputs</GlossTerm>, and it includes a built-in terminal so you can work in a session without leaving the browser.

Open the **<NT>Remote Dev Environments</NT>** section of the <NT>Bitrise</NT> app to get started.

## Authentication

You can configure your <NT>Claude Code</NT> credentials and passwordless SSH access on the **Saved inputs** page. <GlossTerm baseform="saved input">Saved inputs</GlossTerm> are reusable credentials and values you store once and map into sessions. See [Saved inputs](/en/bitrise-rde/configuration/saved-inputs) for details.

## Creating a session

1. In the <NT>Remote Dev Environments</NT> section, click **<NT>New</NT> session**.
1. Set a session title, and optionally a description.

   The session title must be unique. 
1. Optionally, enter an AI prompt to start <NT>Claude Code</NT> automatically when the session boots. This requires saved <NT>Claude Code</NT> credentials.
1. Choose when the session should auto-terminate.
1. Choose a template, or select **No template** and pick a stack and a machine type directly.
1. Fill in any required inputs. You can type a value or select an existing saved input.
1. Click **<NT>Create</NT>**.

The session starts within a short while. When it's running, open it to connect.

## Managing sessions

The **Sessions** list shows every session in the <NT>workspace</NT>, grouped by status, with search. From a session you can:

- Restore a terminated session, or terminate a running one.
- Permanently delete a terminated session.
- Edit its name and description, and change its auto-terminate setting.

## Connecting to a session

The session detail page is your connection hub. From it you can:

- Open the **built-in web terminal**, with support for multiple tabs.
- Copy the **SSH** and **VNC** credentials to connect from your own tools.
- Use **Open in VS Code** to connect with the VS Code Remote - SSH extension.
- Use **<NT>workspace</NT>-link buttons** to open a specific folder in your IDE.

For details on each method, see [Connecting to a session](/en/bitrise-rde/rde-options/connecting-to-a-session).

## Templates

You can create and manage reusable session configurations from the UI: see [Templates](/en/bitrise-rde/configuration/templates).
