Connecting to an <NT>Apple</NT> service (such as the <NT>App Store Connect</NT> or the <NT>Apple Developer Portal</NT>) with the <NT>Apple ID</NT> requires that you first add your <NT>Apple ID</NT> and password on the **<NT>Apple Service connection</NT>** page, then select an app to use <NT>Apple ID</NT> authentication while the build is running.

If you cannot use the API key authentication, we recommend you try this option. Please note that you can only connect one <NT>Apple ID</NT> to <NT>Bitrise</NT>.

:::important[<NT>Apple</NT> accounts with two-factor authentication enabled]

If two-factor authentication is enabled on your <NT>Apple</NT> account, you will have to provide the <NT>App-specific password</NT> during this process. Learn [how to generate an app-specific password on the Security section of your Apple ID account page](https://support.apple.com/en-us/HT204397).

:::


## Adding <NT>Apple ID</NT> authentication data on <NT>Bitrise</NT>

1. <NT>Log in</NT> to <NT>Bitrise</NT>, and select **<NT>Bitrise CI</NT>** from the left navigation menu.
1. In the upper right corner, click the profile image to open the dropdown menu.
1. Select the **<NT>Account settings</NT>** option.

   ![account-settings-page.png](/img/_paligo/uuid-b08c42b9-affd-b40d-3c6e-87410001d2fb.png)
1. On the left menu bar, select **<NT>Apple Service connection</NT>**.
1. Click the **<NT>Add connection</NT>** button.
1. Provide your **<NT>Apple ID</NT>** and **<NT>Password</NT>** in the popup window. Click **<NT>Connect</NT>**.
1. Enter your **<NT>verification code</NT>** in the **Two factor authentication** window.
1. Add the **<NT>app-specific password</NT>**. Click **<NT>Verify</NT>** to continue.
1. You will be prompted to provide your verification code.
1. Your authentication expires in 30 days and you will have to authenticate again. When that happens, go to the **<NT>Apple Service connection</NT>** page, it will automatically ask for the two-factor authentication (2FA) code to authenticate again. There will be a list of the <NT>Apple</NT> Developer accounts that you have defined. To the far right of the <NT>Apple ID connection</NT>, there are 3 dots. Click the dots and select **<NT>Re-authenticate</NT>** (**2SA/2FA**).
1. Continue with Assigning an <NT>Apple</NT> Developer Account for your app.


## Assigning <NT>Apple ID</NT> authentication to your app

Connecting to an <NT>Apple</NT> service (such as the <NT>App Store Connect</NT> or the <NT>Apple Developer Portal</NT>) with <NT>Apple ID</NT> authentication requires that you first add your <NT>Apple</NT> service authentication data on bitrise.io and then assign your <NT>Apple</NT> credentials to the app. With this method, you don't have to worry about authentication when using <NT><GlossTerm baseform="step">Step</GlossTerm></NT>s that connect to <NT>Apple</NT> services: all <NT>Step inputs</NT> related to <NT>Apple</NT> authentication are automatically filled in.

:::important[Can't assign other users' <NT>Apple</NT> credentials]

You can only set your own user account's <NT>Apple ID</NT> authentication data to an app. If you have an app that needs someone else's <NT>Apple</NT> credentials, that user must log in and make the change themselves.

:::

To assign <NT>Apple ID</NT> authentication data to your app:

1. <Partial_GettingToTheProjectSettingsPage />
1. On the left, select **<NT>Integrations</NT>**.
1. Scroll down to the **Connection to <NT>Apple</NT> services** section.
1. Find the **<NT>Apple ID authentication</NT>** option.

   ![add-api-key-apple.png](/img/_paligo/uuid-95e5fcc6-46bd-1efe-69a0-538ef9c8c8cb.png)
1. Click **<NT>Change</NT> to &lt;username&gt;**.

   If you can't find this option, you probably haven't set up your <NT>Apple</NT> credentials on bitrise.io. Note that changing the selected user removes the previous user. If you need to use the previous user's <NT>Apple</NT> credentials again, that user must log in and change it themselves.

:::tip[Test connection]

By clicking **<NT>Test connection</NT>**, you can check whether <NT>Bitrise</NT> can access <NT>Apple</NT> services using the provided credentials.

:::

<NT>Steps</NT> are now able to connect to an <NT>Apple</NT> service during your build.
