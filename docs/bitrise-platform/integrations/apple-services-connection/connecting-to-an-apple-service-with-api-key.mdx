[Connecting to an Apple service](/en/bitrise-platform/integrations/apple-services-connection/about-connecting-to-apple-services) (such as the <NT>App Store Connect</NT> or the <NT>Apple Developer Portal</NT>) with the API key requires generating an API key, adding the authentication data on <NT>Bitrise</NT>, and assigning the key to the app.

The API key authentication is the recommended way when connecting <NT>Bitrise</NT> to <NT>Apple</NT> Services. You can have 50 API keys added to the **<NT>Apple Services</NT> connection** page but your app can use only one.

<Partial_IMPORTANTNoStepInputModificationsNeeded />


## Adding API key authentication data on <NT>Bitrise</NT>

You can add API key authentication data on the **<NT><GlossTerm baseform="workspace">Workspace</GlossTerm></NT> settings** page. You can add multiple API keys here. You can select between different API keys for each project, using them either for <NT>Bitrise CI</NT> or <NT>Bitrise</NT> <NT>Release Management</NT>.

1. On [App Store Connect](https://appstoreconnect.apple.com/login), [generate a new API key with Admin access](https://developer.apple.com/documentation/appstoreconnectapi/creating_api_keys_for_app_store_connect_api) under **<NT>Users</NT> and Access**.

   You will need the name of the key, the <NT>key ID</NT> and the <NT>issuer ID</NT> on <NT>Bitrise</NT>.

   :::warning[Requirements]

   - The API key must be [a team API key](https://developer.apple.com/documentation/appstoreconnectapi/creating-api-keys-for-app-store-connect-api#Generate-a-Team-Key-and-Assign-It-a-Role). An individual API key won't work.

     ![apple-app-store-connect.png](/img/_paligo/uuid-b104fac1-d553-69d3-ebcb-b8260e8a3595.png)
   - The API key must be set up with the **Admin** role on the App Store. Your builds will fail otherwise.
   - On <NT>Bitrise</NT>, you must be a <NT>workspace</NT> **<NT>Owner</NT>** or **<NT>Manager</NT>** to be able to add the API key data. For more information, check out [Workspace-level roles](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/roles-and-permissions-in-workspaces#workspace-level-roles).

   :::
1. On <NT>Bitrise</NT>, select your <NT>workspace</NT> from the dropdown menu and go to **<NT>Settings</NT>**.
1. On the left, select **<NT>Integrations</NT>**.
1. Select **<NT>App Store Connect</NT>**.
1. Click **<NT>Add API key</NT>**.

   :::note[Required roles]

   If you can't see the button, it's because you only have a **<NT>Viewer</NT>** or **<NT>Contributor</NT>** role in the <NT>workspace</NT>. You need to be either an **<NT>Owner</NT>** or a **<NT>Manager</NT>**.

   For more information, check out [Workspace-level roles](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/roles-and-permissions-in-workspaces#workspace-level-roles).

   :::

   ![api-key-app-store.png](/img/_paligo/uuid-1bbd8347-eb11-4439-9ece-315d004d74fe.png)
1. Fill out all required fields:

   - **<NT>Name</NT>**: Your generated API key's name.
   - **<NT>Issuer ID</NT>**: To get your <NT>issuer ID</NT>, log in to <NT>App Store Connect</NT> and select **<NT>Users</NT> and Access**, then select the **<NT>API Keys</NT>** tab.The <NT>issuer ID</NT> appears near the top of the page. To copy the <NT>issuer ID</NT>, click **<NT>Copy</NT>** next to the ID.
   - **<NT>Key ID</NT>**: When you successfully generate a new API key, <NT>App Store Connect</NT> shows you the <NT>key ID</NT>.

   ![api-key-dialog.png](/img/_paligo/uuid-22e6048e-7f47-9480-fea3-958eb9a0348b.png)
1. Upload the `.p8` file: either drag and drop it to the upload field, or click it and select the file from your computer.
1. Click **<NT>Connect</NT>**.

Once done, you can now assign this API key to any app that you have access to on this account. From then, the app will use that API connection to access <NT>App Store Connect</NT>.


## Assigning API key authentication to your project

To use API key authentication for <NT>Apple</NT> services for your project, you must:

- [Add an API key on Bitrise](/en/bitrise-platform/integrations/apple-services-connection/connecting-to-an-apple-service-with-api-key#adding-api-key-authentication-data-on-bitrise): You can add multiple API keys to the same account.
- Assign an API key to your project: Each project can only have one API key assigned to it.

To assign the API key to the project:

1. <Partial_GettingToTheProjectSettingsPage />
1. On the left, select **<NT>Integrations</NT>**.
1. Select the **<NT>Stores</NT>** tab.
1. Find the **<NT>App Store Connect</NT>** section.

   ![add-api-key-apple.png](/img/_paligo/uuid-95e5fcc6-46bd-1efe-69a0-538ef9c8c8cb.png)
1. Find the **<NT>API key</NT> authentication (recommended)** option.
1. Click **<NT>Add API key</NT>**.

   :::note[Multiple projects with the same API key]

   With the **<NT>Add API key</NT>** button, you can add a new <NT>App Store Connect</NT> API key to your <NT>Bitrise</NT> account. Other projects you have access to can use this API key, too.

   :::
1. If your API key is an <NT>Apple</NT> Enterprise API key, check **This <NT>API key</NT> is used for the <NT>Apple</NT> Enterprise Program API**.

   ![api-key-enterprise.png](/img/_paligo/uuid-7dc1ff71-7abf-8c37-8117-a2f7abb13861.png)

The <NT><GlossTerm baseform="Step">Step</GlossTerm></NT> is now able to connect to the <NT>App Store Connect</NT> or the <NT>Apple Developer Portal</NT> during your build.


## API key authentication for <NT>Apple</NT> Enterprise users

<NT>Apple</NT> accounts that are part of the Enterprise developer program can also use API key-based authentication.

If your Enterprise account still uses <NT>Apple ID</NT> authentication, we strongly recommend switching to API key authentication because:

- It's more reliable and secure.
- It doesn't have to be re-authorized every 30 days.
- The <NT>Apple ID</NT> authentication method will be deprecated at some point.

To change your authentication method:

1. [Assign API key authentication](/en/bitrise-platform/integrations/apple-services-connection/connecting-to-an-apple-service-with-api-key#assigning-api-key-authentication-to-your-project) for your project(s) on <NT>Bitrise</NT>. Make sure to check the **This <NT>API key</NT> is used for the <NT>Apple</NT> Enterprise Program API** option for your API key during the process!

   ![api-key-enterprise.png](/img/_paligo/uuid-7dc1ff71-7abf-8c37-8117-a2f7abb13861.png)

   :::caution[Keep the <NT>Apple ID connection</NT>]

   We strongly recommend not to remove your <NT>Apple ID connection</NT>. If there are any issues with the API key connection, you can safely roll back to <NT>Apple ID</NT>-based authentication.

   :::
1. Adjust all <NT>Step inputs</NT> that use <NT>Apple ID</NT> authentication to access <NT>Apple</NT> services. The following <NT>Steps</NT> are affected:

   - **<NT>Xcode Archive & Export for iOS</NT>**: version 5.3.0 and above. The `automatic_code_signing` input should be set to `auto` or `api-key`.
   - **<NT>Xcode</NT> <NT>Build</NT> for Testing**: version 3.1.0 and above. The `automatic_code_signing` input should be set to `auto` or `api-key`.
   - **<NT>Manage iOS Code Signing</NT>**: version 2.1.0 and above. The `apple_service_connection` input should be set to `auto` or `api-key`.
   - **Export iOS and tvOS <NT>Xcode archive</NT>**: version 4.6.0 and above. The `automatic_code_signing` input should be set to `auto` or `api-key`.

### Multiple API keys for the same project

You can only set up one API connection for each project on the <NT>Bitrise website</NT>. That means the project can handle the <NT>code signing</NT> files of only one <NT>Apple</NT> developer team: API keys only give authorization to a single developer team.

![api-key-chart.png](/img/_paligo/uuid-ea47d90b-0ff0-7cd5-2f80-4013e10f7fb3.png)

If your <NT>Bitrise</NT> <NT><GlossTerm baseform="project">project</GlossTerm></NT> needs to handle the <NT>code signing</NT> of multiple developer teams, you can pass additional API keys as <NT>Step inputs</NT>. Doing so overrides the <NT>Bitrise</NT>-managed API connection. The following inputs are available to all <NT>Steps</NT> that can access <NT>Apple</NT> services:

- `api_key_path`: This can be a local file path or a download URL. You can use the <NT>Bitrise</NT> <NT>project</NT>’s file storage (<NT>Project settings</NT>/Files) to store and get a download URL for the API Keys.
- `api_key_id`: Private <NT>key ID</NT>. You can find the <NT>key ID</NT> for each API key on <NT>App Store Connect</NT>.
- `api_key_issuer_id`. The private key <NT>issuer ID</NT>. You can find the <NT>issuer ID</NT> for each API key on <NT>App Store Connect</NT>.
- `api_key_enterprise_account`: Use it to indicate whether the API key belongs to an account that is part of the <NT>Apple</NT> Enterprise developer program. Takes two values: `yes` and `no`.

:::note[Inputs on the GUI]

On the graphical UI of the <NT>Bitrise</NT> <NT><GlossTerm baseform="workflow editor">Workflow Editor</GlossTerm></NT>, you can find these inputs under the **<NT>App Store Connect connection override</NT>** input group on the **<NT>Configuration</NT>** tab of each <NT>Step</NT> that can access <NT>Apple</NT> services.

:::
