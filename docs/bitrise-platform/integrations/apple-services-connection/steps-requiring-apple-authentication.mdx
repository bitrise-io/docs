In this guide we list all the <NT>Steps</NT> that require authentication and the authentication methods that you can choose from depending on your app’s requirement.

## [Manage iOS Code Signing](https://github.com/bitrise-steplib/bitrise-step-manage-ios-code-signing) <NT>Step</NT>

[This Step](https://www.bitrise.io/integrations/steps/manage-ios-code-signing) takes care of setting up the required <NT>code signing</NT> assets before your project is built on <NT>Bitrise</NT>. The <NT>Step</NT> uses your API key or your <NT>Apple ID</NT> and password authentication to connect to an <NT>Apple</NT> service. Once connection is configured, the <NT>Step</NT> will:

- Generate, update and download the provisioning profiles needed for your iOS project.
- Verify and register the project's Bundle IDs on the <NT>Apple</NT> Developer Site.
- Register the iOS devices connected to your <NT>Bitrise</NT> account with the <NT>App Store Connect</NT>.

## [Xcode Archive & Export for iOS](https://github.com/bitrise-steplib/steps-xcode-archive) <NT>Step</NT>

The <NT>Step</NT> archives your <NT>Xcode</NT> project by running the `xcodebuild archive` command and then exports the archive into an IPA file with the `xcodebuild -exportArchive` command. This IPA file can be shared, installed on test devices, or uploaded to the <NT>App Store Connect</NT>.

The <NT>Step</NT> can also perform iOS <NT>code signing</NT> if the **<NT>Automatic code signing method</NT>** input specifies a method. By default, it's turned off. To perform iOS <NT>code signing</NT>, it can use API key or <NT>Apple ID</NT> authentication.

## [Export iOS and tvOS Xcode archive](https://github.com/bitrise-steplib/steps-export-xcarchive) <NT>Step</NT>

Exports an IPA from an existing iOS and tvOS .xcarchive file. You can add multiple [Export iOS and tvOS Xcode archive](https://github.com/bitrise-steplib/steps-export-xcarchive) <NT>Steps</NT> to your <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>s to create multiple different signed IPA files.

The <NT>Step</NT> can also perform iOS <NT>code signing</NT> if the **<NT>Automatic code signing method</NT>** input specifies a method. By default, it's turned off. To perform iOS <NT>code signing</NT>, it can use API key or <NT>Apple ID</NT> authentication.

## [Xcode Build for testing for iOS](https://github.com/bitrise-steplib/steps-xcode-build-for-test) <NT>Step</NT>

The <NT>Step</NT> runs <NT>Xcode</NT>'s `xcodebuild` command with the build-for-testing option. This builds your app and associated tests so that you can, for example, upload it to a third-party testing service to run your tests on a real device. The <NT>Step</NT> also creates an `.xctestrun` file. To be able to run your tests on a real device it needs <NT>code signing</NT>.

The **<NT>Automatic code signing method</NT>** <NT>Step</NT> input allows you to log you into your <NT>Apple</NT> Developer account based on the <NT>Apple service connection</NT> you provide on <NT>Bitrise</NT> and download any provisioning profiles needed for your project based on the **<NT>Distribution method</NT>**.

## <NT>Deploy to App Store Connect</NT> with Deliver <NT>Step</NT>

With [this Step](https://www.bitrise.io/integrations/steps/deploy-to-itunesconnect-deliver), you can upload screenshots, metadata and binaries to [https://appstoreconnect.apple.com/](https://appstoreconnect.apple.com/) and submit your app for App Store review using the fastlane [deliver](https://docs.fastlane.tools/actions/deliver/) action. The [**Deploy to App Store Connect with Deliver (formerly iTunes Connect)**](https://github.com/bitrise-steplib/steps-deploy-to-itunesconnect-deliver) <NT>Step</NT> can connect to your <NT>Apple</NT> Developer Account either with <NT>Apple ID</NT> or with the <NT>App Store Connect</NT> API, or through <NT>Step inputs</NT>. Please note that in the case of 2FA enabled <NT>Apple ID</NT>, the **<NT>Deploy to App Store Connect</NT> with Deliver (formerly iTunes Connect)** <NT>Step</NT> can only work with <NT>Apple ID</NT> authentication which you can set on the <NT>Apple</NT> <NT>Service</NT> page of your profile. The default method is the API key authentication. Choose the <NT>connection method</NT> that works with your project:

[Connecting to an Apple service with API key](/en/bitrise-platform/integrations/apple-services-connection/connecting-to-an-apple-service-with-api-key)

[Connecting to an Apple service with Apple ID](/en/bitrise-platform/integrations/apple-services-connection/connecting-to-an-apple-service-with-apple-id)

[Connecting to an Apple Service with Step inputs](/en/bitrise-platform/integrations/apple-services-connection/connecting-to-an-apple-service-with-step-inputs)

## <NT>Deploy to App Store Connect</NT> - Application Loader (formerly iTunes Connect) <NT>Step</NT>

With [this Step](https://www.bitrise.io/integrations/steps/deploy-to-itunesconnect-application-loader), you can upload binaries (.ipa or .pkg files) to [https://appstoreconnect.apple.com/](https://appstoreconnect.apple.com/). The [**Deploy to App Store Connect - Application Loader (formerly iTunes Connect)**](https://github.com/bitrise-steplib/steps-deploy-to-itunesconnect-application-loader) <NT>Step</NT> can connect to your <NT>Apple</NT> Developer Account either with the <NT>App Store Connect</NT> API, the <NT>Apple ID</NT> and password, or through <NT>Step inputs</NT>. Choose the <NT>connection method</NT> that works with your project:

## [fastlane](https://github.com/bitrise-io/steps-fastlane) <NT>Step</NT>

With this <NT>Step</NT> you can run your [*fastlane*](https://fastlane.tools/) lanes on <NT>Bitrise</NT> just like you would locally.

Please note that in the case of 2FA enabled <NT>Apple ID</NT>, the [**Fastlane**](https://github.com/bitrise-io/steps-fastlane) <NT>Step</NT> can only work with <NT>Apple ID</NT> authentication which you can set on the **<NT>Apple</NT> <NT>Service</NT>** page of your profile.

:::note[Two-factor authentication fails with the <NT>Fastlane</NT> <NT>Step</NT>]

In some cases, the [**Fastlane**](https://github.com/bitrise-io/steps-fastlane) <NT>Step</NT> doesn't get the value of the FASTLANE_SESSION variable correctly when attempting to connect to the <NT>Apple Developer portal</NT>, causing two-factor authentication to fail. Check out the [potential workaround](https://support.bitrise.io/hc/en-us/articles/360017174577) for this issue.

:::
