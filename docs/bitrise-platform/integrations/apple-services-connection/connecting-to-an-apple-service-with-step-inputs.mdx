If you wish to deploy to multiple teams or deploy to a team where authentication is different from the connected one you’ve been using, then you can add your preferred authentication into the <NT><GlossTerm baseform="step">Step</GlossTerm></NT>’s inputs. This will override the connection previously set in **<NT>Bitrise</NT> Developer Connection**.

This way connection is restricted to the given <NT>Step</NT> where you set up authentication.

:::important[2FA and <NT>Apple ID</NT> authentication]

If 2FA is required for your <NT>Apple ID</NT>, then you must use the <NT>Apple ID</NT> authentication with the [**Deploy to App Store Connect with Deliver (formerly iTunes Connect)**](https://github.com/bitrise-steplib/steps-deploy-to-itunesconnect-deliver) and the [**fastlane**](https://github.com/bitrise-io/steps-fastlane) <NT>Steps</NT> instead of authenticating through <NT>Step inputs</NT>.

:::


## Authenticating in a <NT>Step</NT> with API key

<Tabs>
<TabItem value="api-key-apple-id" label="API-key & Apple ID" default>

1. Add one of the following <NT>Steps</NT> to your <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT>:

   - fastlane
   - <NT>Deploy to App Store Connect</NT> with Deliver (formerly iTunes Connect)
   - <NT>Deploy to App Store Connect</NT> - Application Loader (formerly iTunes Connect)

   :::caution[Either API key or <NT>Apple ID</NT> authentication in <NT>Step inputs</NT>]

   You can only add one type of authentication into the <NT>Step</NT>, either the API key one or the <NT>Apple ID</NT> one. In either case, make sure the **<NT>Bitrise Apple Developer Connection</NT>** input is set to `off`, otherwise the <NT>Step</NT> will go with the configured authentication method found in **<NT>Apple Service connection</NT>** (unless you decide to remove the connection on the **<NT>Teams</NT>** tab).

   :::
1. Upload the API key to the **<NT>GENERIC FILE STORAGE</NT>** section of the **<NT>Code Signing & Files</NT>** tab.
1. Set the **<NT>Bitrise Apple Developer Connection</NT>** input to `off`.
1. Set the **<NT>Issuer ID</NT>** as a secret Environment Variable in the respective field of the <NT>Step</NT>.
1. Configure the **<NT>API Key</NT> path** and **API <NT>Issuer</NT>** inputs.
1. Save your changes and run a new build.

</TabItem>
<TabItem value="api-key-only-steps" label="API-key only Steps">

1. Add one of the following <NT>Steps</NT> to your <NT>Workflow</NT>:

   - <NT>Manage iOS Code Signing</NT>
   - <NT>Xcode Archive & Export for iOS</NT>
   - Export iOS and tvOS <NT>Xcode archive</NT>
   - <NT>Xcode Build for testing for iOS</NT>
1. Upload your <NT>App Store Connect</NT> API private key file (.p8 file) to the **Generic File <NT>Storage</NT>** in the **<NT>Code Signing & Files</NT>** tab.
1. Set the **<NT>Automatic code signing method</NT>** to `api-key`.
1. Set the following <NT>Step inputs</NT> under the **<NT>App Store Connect connection override</NT>** input group:

   - **<NT>App Store Connect</NT> API private key**: Local path or remote URL to the private key (.p8 file) for <NT>App Store Connect</NT> API. Set this input to the automatically generated Env Var connected to the <NT>App Store Connect</NT> API private key file (.p8 file) you uploaded to the Generic File Storage in step 2.
   - **<NT>App Store Connect</NT> API <NT>key ID</NT>**: Private <NT>key ID</NT> used for <NT>App Store Connect</NT> authentication.
   - **<NT>App Store Connect</NT> API <NT>issuer ID</NT>**: Private key <NT>issuer ID</NT> used for <NT>App Store Connect</NT> authentication.

</TabItem>
</Tabs>


## Authenticating in a <NT>Step</NT> with <NT>Apple ID</NT> and password

:::caution[Either API key or <NT>Apple ID</NT> authentication in <NT>Step inputs</NT>]

You can only add one type of authentication into the <NT>Step</NT>, either the API key one or the <NT>Apple ID</NT> one. In either case, make sure the **<NT>Bitrise Apple Developer Connection</NT>** input is set to `off`, otherwise the <NT>Step</NT> will go with the configured authentication method found in **<NT>Apple Service connection</NT>** (unless you decide to remove the connection on the **<NT>Teams</NT>** tab).

:::

1. Add the <NT>Step</NT> to your <NT>Workflow</NT>.
1. Set the **<NT>Apple ID</NT>** and **<NT>Password</NT>**.
1. Set the **<NT>Bitrise Apple Developer Connection</NT>** input to `off`.
1. Save your changes and run a new build.
