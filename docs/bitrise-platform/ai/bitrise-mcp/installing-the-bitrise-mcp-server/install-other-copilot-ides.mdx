# Install <NT>Bitrise MCP</NT> Server in Copilot IDEs

Quick setup guide for the <NT>Bitrise MCP</NT> server in <NT>GitHub</NT> Copilot across different IDEs. For VS Code instructions, refer to the [Install Bitrise MCP Server in VS Code](/en/bitrise-platform/ai/bitrise-mcp/installing-the-bitrise-mcp-server/install-vscode)

### Requirements:
1. <NT>GitHub</NT> Copilot License: Any Copilot plan (Free, Pro, Pro+, Business, Enterprise) for Copilot access
2. <NT>Bitrise</NT> Account: <NT>Bitrise</NT> account for <NT>Bitrise MCP</NT> server access
3. MCP Servers in Copilot Policy: Organizations assigning Copilot seats must enable this policy for all MCP access in Copilot for VS Code and Copilot Coding Agent – all other Copilot IDEs will migrate to this policy in the coming months
4. For local setup: [Go](https://go.dev/) (>=1.23) installed and a <NT>Bitrise</NT> Personal Access Token

The remote setups below use OAuth — your IDE opens a browser on first tool use, you sign in to <NT>Bitrise</NT>, no token to paste. For older Copilot IDE builds without MCP OAuth, a PAT-based fallback configuration is included for each IDE.

## Visual Studio

Requires Visual Studio 2022 version 17.14.9 or later.

### Remote Server (Recommended)

The remote <NT>Bitrise MCP</NT> server is hosted by <NT>Bitrise</NT> and provides automatic updates with no local setup required.

#### Configuration
1. Create an `.mcp.json` file in your solution or %USERPROFILE% directory.
2. Add this configuration:

   ```json
   {
     "servers": {
       "bitrise": {
         "url": "https://mcp.bitrise.io"
       }
     }
   }
   ```
3. Save the file. Wait for CodeLens to update to offer a way to authenticate to the new server, activate that and complete the <NT>Bitrise</NT> sign-in in your browser.
4. In the <NT>GitHub</NT> Copilot Chat window, switch to Agent mode.
5. Activate the tool picker in the Chat window and enable one or more tools from the "bitrise" MCP server.

#### Fallback: PAT-based authentication

```json
{
  "servers": {
    "bitrise": {
      "url": "https://mcp.bitrise.io",
      "headers": {
        "Authorization": "Bearer YOUR_BITRISE_PAT"
      }
    }
  }
}
```

[Create a PAT](https://devcenter.bitrise.io/api/authentication) under [Account Settings → Security](https://app.bitrise.io/me/account/security).

### Local Server (Go required)

#### Configuration
1. Create an `.mcp.json` file in your solution or %USERPROFILE% directory.
2. Add this configuration:

   ```json
   {
     "servers": {
       "bitrise": {
         "type": "stdio",
         "command": "go",
         "args": ["run", "github.com/bitrise-io/bitrise-mcp/v2@v2"],
         "env": {
           "BITRISE_TOKEN": "YOUR_BITRISE_PAT"
         }
       }
     }
   }
   ```
3. Save the file. Wait for CodeLens to update to offer a way to provide user inputs, activate that and paste in a PAT you generate from your [Bitrise Account Settings/Security](https://app.bitrise.io/me/account/security).
4. In the <NT>GitHub</NT> Copilot Chat window, switch to Agent mode.
5. Activate the tool picker in the Chat window and enable one or more tools from the "bitrise" MCP server.

**<NT>Documentation</NT>:** [Visual Studio MCP Guide](https://learn.microsoft.com/visualstudio/ide/mcp-servers)

## JetBrains IDEs

Agent mode and MCP support available in public preview across IntelliJ IDEA, PyCharm, WebStorm, and other JetBrains IDEs.

### Remote Server (Recommended)

#### Configuration <NT>Steps</NT>
1. Install/update the <NT>GitHub</NT> Copilot plugin
2. Click **<NT>GitHub</NT> Copilot icon in the status bar** → **<NT>Edit</NT> <NT>Settings</NT>** → **Model Context Protocol** → **<NT>Configure</NT>**
3. <NT>Add configuration</NT>:

   ```json
   {
     "servers": {
       "bitrise": {
         "url": "https://mcp.bitrise.io"
       }
     }
   }
   ```
4. Press `Ctrl + S` or `Command + S` to save, or close the `mcp.json` file. On first tool use the IDE will open your browser for <NT>Bitrise</NT> sign-in.

#### Fallback: PAT-based authentication

```json
{
  "servers": {
    "bitrise": {
      "url": "https://mcp.bitrise.io",
      "requestInit": {
        "headers": {
          "Authorization": "Bearer YOUR_BITRISE_PAT"
        }
      }
    }
  }
}
```

### Local Server (Go required)

```json
{
  "servers": {
    "bitrise": {
      "command": "go",
      "args": [
        "run",
        "github.com/bitrise-io/bitrise-mcp/v2@v2"
      ],
      "env": {
        "BITRISE_TOKEN": "YOUR_BITRISE_PAT"
```

## Eclipse

MCP support available with Eclipse 2024-03+ and <NT>latest version</NT> of the <NT>GitHub</NT> Copilot plugin.

### Remote Server (Recommended)

#### Configuration <NT>Steps</NT>
1. Install <NT>GitHub</NT> Copilot extension from Eclipse Marketplace
2. Click the **<NT>GitHub</NT> Copilot icon** → **<NT>Edit</NT> Preferences** → **MCP** (under **<NT>GitHub</NT> Copilot**)
3. Add <NT>Bitrise MCP</NT> server configuration:

   ```json
   {
     "servers": {
       "bitrise": {
         "url": "https://mcp.bitrise.io"
       }
     }
   }
   ```
4. Click the "Apply and Close" button. On first tool use, complete the <NT>Bitrise</NT> sign-in in your browser.

#### Fallback: PAT-based authentication

```json
{
  "servers": {
    "bitrise": {
      "url": "https://mcp.bitrise.io",
      "requestInit": {
        "headers": {
          "Authorization": "Bearer YOUR_BITRISE_PAT"
        }
      }
    }
  }
}
```

### Local Server (Go required)

```json
{
  "servers": {
    "bitrise": {
      "command": "go",
      "args": [
        "run",
        "github.com/bitrise-io/bitrise-mcp/v2@v2"
      ],
      "env": {
        "BITRISE_TOKEN": "YOUR_BITRISE_PAT",
        "PATH": "PATH to bin directory of go:PATH to directory of git"

```

**<NT>Documentation</NT>:** [Eclipse Copilot plugin](https://marketplace.eclipse.org/content/github-copilot)

## Usage

After setup:
1. Restart your IDE completely
2. Open Agent mode in Copilot Chat
3. Try: *"List my <NT>Bitrise</NT> apps"* — first call triggers OAuth sign-in for the remote server
4. Copilot can now access <NT>Bitrise</NT> data and perform operations

## Troubleshooting

- **<NT>OAuth flow doesn't open browser</NT>**: Make sure your Copilot integration is on a build that supports MCP OAuth. Fall back to PAT-based auth in older versions.
- **Connection issues**: Verify IDE version compatibility
- **<NT>Authentication</NT> errors**: Check if your organization has enabled the MCP policy for Copilot
- **<NT>Tools not appearing</NT>**: Restart IDE after configuration changes and check error logs
