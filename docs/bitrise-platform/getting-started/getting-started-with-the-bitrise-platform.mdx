The <NT>Bitrise platform</NT> is organized around accounts, <NT>workspaces</NT>, and projects. We'll quickly go through how to start using them.

## Signing up

You can sign up via email or via one of three Git providers: <NT>GitHub</NT>, <NT>GitLab</NT>, or <NT>Bitbucket</NT>: [Signing up for Bitrise](/en/bitrise-platform/getting-started/signing-up-for-bitrise).

<NT>Bitrise</NT> also supports SAML SSO: [SAML SSO in Bitrise](/en/bitrise-platform/accounts/saml-sso-in-bitrise/configuring-saml-sso-on-bitrise).

## First <NT>workspace</NT>

After signing up, we automatically create your first <NT>workspace</NT>. This <NT>workspace</NT> is owned by your user account. You can create additional <NT>workspaces</NT> at any point and be invited to other <NT>workspaces</NT>.

<NT>Workspaces</NT> are very important in <NT>Bitrise</NT>: all your work is organized in <NT>workspaces</NT>. You can:

- Add users and organize them into user groups: [Workspace collaboration](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/workspace-collaboration)
- Configure <NT>workspace</NT>-level integrations for third-party services such as the <NT>Apple</NT> Store or <NT>Google Play</NT>: [About integrations](/en/bitrise-platform/integrations/about-integrations).
- <NT>Create project</NT>s for <NT>Bitrise CI</NT> and <NT>Release Management</NT>.

## Projects

When you have your first <NT>workspace</NT>, you will be prompted to add your first project.

![gettingstarted.png](/img/_paligo/uuid-4f8203e0-c79b-b9ec-7537-53e96b0f1924.png)

There are two ways of creating a new project:

- Starting with <NT>Bitrise CI</NT>: you will be automatically taken to the **<NT>Add</NT> <NT>new project</NT>** flow. This creates a CI project with a linked Git repository: [Adding a new project](/en/bitrise-ci/getting-started/adding-a-new-project).
- Starting with <NT>Release Management</NT>: you can add a new app to <NT>Release Management</NT> and <NT>Bitrise</NT> will automatically link it to a new project: [Adding a new app to Release Management](/en/release-management/getting-started-with-release-management/adding-a-new-app-to-release-management).

  This project will not have a CI configuration but you can extend it with one.

Configure your project by entering **<NT>Project settings</NT>**. You can access it from both <NT>Bitrise CI</NT> and from <NT>Release Management</NT>.
