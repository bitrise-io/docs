This guide provides instructions on setting up SAML SSO using [Auth0](https://auth0.com).

<Partial_CAUTIONSAMLSSORestrictions />

Before connecting SAML SSO to your <NT><GlossTerm baseform="workspace">Workspace</GlossTerm></NT>:

- An <NT>Auth0</NT> administrator who is logged into <NT>Auth0</NT> is at hand.
- <Partial_SAMLSSOBeforeYouStart />

You will need to:

1. [Create Bitrise as a web application on Auth0](#creating-bitrise-as-a-web-application-on-auth0).
1. [Retrieve SAML SSO information from Auth0](#retrieving-saml-sso-information-from-auth0).
1. [Setting up a mapping rule for your Bitrise app's Client ID](#setting-up-a-mapping-rule-for-your-bitrise-apps-client-id).


## Creating <NT>Bitrise</NT> as a web application on <NT>Auth0</NT>

1. Log into [Auth0](https://auth0.com) as an admin.
1. Click **Applications** on the left menu bar then click the **+ <NT>Create</NT> Application** button on the right hand side of the **Applications** page.

   ![authzero_createapp1.png](/img/_paligo/uuid-47aacb1a-a7a9-f93e-8130-83ad51654298.png)
1. On the **<NT>Create</NT> application** window, type <NT>Bitrise</NT> in the **<NT>Name</NT>** field. In the **Choose application type** select **Regular Web Applications**, and click the **<NT>Create</NT>** button.

   ![authzero_addappname2.png](/img/_paligo/uuid-fc4a47cb-98f9-e217-042d-73780f712eb2.png)
1. You are landing on your newly created <NT>Bitrise</NT> app’s **Quick <NT>Start</NT>** page. Click the **Addons** tab. Toggle the **SAML2 WEB <NT>APP</NT>**’s switch to the right. This will take to to the **<NT>Addon: SAML2 WEB APP</NT>** page automatically.

   ![authzero_addon4.png](/img/_paligo/uuid-bf544047-689a-3954-029d-58b65a35c540.png)
1. Copy the **Assertion Consumer Service URL (ACS URL)** from <NT>Bitrise</NT> and paste it into the **Application Callback URL** field on the **<NT>Settings</NT>** tab of the **<NT>Addon: SAML2 WEB APP</NT>** page. Scroll down to the bottom of the **<NT>Addon: SAML2 WEB APP</NT>** page and hit **<NT>SAVE</NT>**.

   ![authzero_settingsapplicationurl6.png](/img/_paligo/uuid-193b49ae-3ccd-9f68-5288-55ed12c9edb1.png)
1. <NT>Go back</NT> to the **Addons** page where you can find the switch turned on.

   ![authzero_samlappenabled8.png](/img/_paligo/uuid-c79c0bb4-943a-7604-d40b-f8478be408bd.png)


## Retrieving SAML SSO information from <NT>Auth0</NT>

Once you have enabled <NT>Bitrise</NT> as a web application on <NT>Auth0</NT>, it’s time to grab the certificate and the Identity provider’s unique login URL to finish up the SAML configuration on <NT>Bitrise</NT>.

1. On <NT>Auth0</NT>, go to **Application**, then select the **Addons** tab to find your enabled <NT>Bitrise</NT> app.
1. Click the **SAML2 WEB <NT>APP</NT>** web app and select the **<NT>Usage</NT>** tab. Click on the **<NT>Download</NT> <NT>Auth0</NT> certificate** link next to the **Identity Provider Certificate** label. Open the downloaded certificate file and copy its content into the **<NT>SAML SSO provider certificate</NT>** field of <NT>Bitrise</NT> or upload the file itself from your local computer. Go back to <NT>Auth0</NT> and copy paste the **Identity Provider Login URL** into <NT>Bitrise</NT>’s **<NT>SAML SSO</NT> provider <NT>Single Sign-On</NT> URL (SSO URL)**.

   ![saml_sso_setup.png](/img/_paligo/uuid-26ad84be-65c7-1916-f079-6b553b4c436d.png)

   ![authzero_downloadcertificateusage9.png](/img/_paligo/uuid-e6cb3a32-c73c-f0cb-e7fe-9eab8e908679.png)
1. Click **<NT>Configure SSO</NT>** button on <NT>Bitrise</NT>.
1. Now you can close the dialog on <NT>Auth0</NT>.


## Setting up a mapping rule for your <NT>Bitrise</NT> app’s Client ID

<NT>Bitrise</NT> authenticates SAML SSO users via email address so before you’d test SAML SSO, make sure you create a new mapping rule on <NT>Auth0</NT>. This way you map <NT>Auth0</NT> Client ID to email for successful SAML authentication on <NT>Bitrise</NT>.

1. On <NT>Auth0</NT>, click the **Auth <NT><GlossTerm baseform="pipeline">Pipeline</GlossTerm></NT>** on the left menu bar. Click **Rules**.
1. Click **+ <NT>Create</NT>** to set up a new mapping rule.
1. On the **Pick a rules template** page, click **&lt;&gt; Empty rule**.
1. Add the following codeblock to the **Script** box: You will need your new <NT>Bitrise</NT> app’s Client ID which you can get on the **Applications**’ page.

   ```javascript
   function mapSamlAttributes(user, context, callback) {
    if (context.clientID === '{your app's clientID'}')
      context.samlConfiguration.mappings = {
        "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier": "email"
      }; 
    }
    callback(null, user, context);
   }
   ```
1. Click **<NT>Save changes</NT>**.
