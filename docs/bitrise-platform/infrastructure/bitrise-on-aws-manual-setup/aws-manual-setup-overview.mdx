<NT>Bitrise</NT> is available as an Amazon Machine Image (AMI) in the <NT>AWS</NT> Marketplace. With the <NT>Bitrise</NT> AMI, you can run <NT>Bitrise</NT> builds using your own <NT>AWS</NT> resources on <NT>Amazon EC2</NT> instances connected to a <NT>Bitrise</NT> <NT><GlossTerm baseform="Workspace">workspace</GlossTerm></NT>. This section focuses on how to manually set up a fully customized <NT>AWS</NT> configuration for <NT>Bitrise</NT> builds.

<Partial_IMPORTANTNoInboundTrafficRequired />

![aws-manual-setup-diagram.png](/img/_paligo/uuid-035193f8-4a3b-65de-4c0b-8054d2e63586.png)

The <NT>manual setup</NT> also requires an active subscription to a <NT>Bitrise</NT> AMI: [Subscribing to the Bitrise on AWS AMI](/en/bitrise-platform/infrastructure/bitrise-on-aws--cloud-controller/subscribing-to-the-bitrise-on-aws-ami).

:::note[<NT>Cloud Controller</NT>]

Instead of the <NT>manual setup</NT>, we strongly recommend using the <NT>Cloud Controller</NT> to set up <NT>AWS</NT> for <NT>Bitrise</NT> builds: [Bitrise on AWS: Cloud Controller](/en/bitrise-platform/infrastructure/bitrise-on-aws--cloud-controller/aws-cloudformation-templates).

:::


## Setting up the <NT>EC2</NT> Mac or <NT>Linux</NT> instance

To run <NT>Bitrise</NT> builds on an <NT>Amazon EC2</NT> Mac or <NT>Linux</NT> instance, you need to:

1. [Configure runner pools on Bitrise](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/launching-an-ec2-instance-for-the-bitrise-ami#configuring-the-instance): Set up a runner pool for your builds and get the token required to connect your <NT>Bitrise</NT> <NT>Workspace</NT> to your <NT>Amazon EC2</NT> instance.
1. [Subscribe to the Bitrise AMI and launch an Amazon EC2 instance](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/launching-an-ec2-instance-for-the-bitrise-ami#preparing-your-ec2-mac-and-linux-instance). You can launch either an <NT>EC2</NT> Mac or an <NT>EC2</NT> <NT>Linux</NT> instance. For an <NT>EC2</NT> Mac instance, you'll need to allocate a dedicated host before launching the instance.
1. [Use the token to connect the instance to your Bitrise Workspace](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/launching-an-ec2-instance-for-the-bitrise-ami#connecting-the-ec2-mac-or-linux-instance-to-your-bitrise-workspace). This will allow the <NT>Bitrise</NT> agent to run your builds on your <NT>EC2</NT> Mac or <NT>Linux</NT> instances.
