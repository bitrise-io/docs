We offer a dedicated <NT>Bitrise</NT> Amazon Machine Image (AMI) to run the <NT>Bitrise</NT> agent on your own <NT>Amazon EC2</NT> Mac or <NT>Linux</NT> instance.

This document guides you through launching an instance on the <NT>Amazon EC2</NT> console UI. The process has three main phases but you can complete it in one sitting:

1. [Preparing your EC2 Mac or Linux instance.](#preparing-your-ec2-mac-and-linux-instance)
1. [Configuring the instance on AWS.](#configuring-the-instance)
1. [Connecting the instance to your Bitrise workspace.](#connecting-the-ec2-mac-or-linux-instance-to-your-bitrise-workspace)


## Preparing your <NT>EC2</NT> Mac and <NT>Linux</NT> instance

<Tabs>
<TabItem value="mac-instance" label="Mac instance" default>

1. [Configure a runner pool](/en/bitrise-platform/infrastructure/configuring-runner-pools) and copy the token required for authentication.
1. Get the token from the process of adding the runner pool on <NT>Bitrise</NT>.
1. [Create an AWS Secrets Manager secret](https://docs.aws.amazon.com/secretsmanager/latest/userguide/create_secret.html) and store the token in the secret.
1. [Create an IAM role](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/iam-roles-for-amazon-ec2.html) with permission to read the secret. You will need to attach it to the <NT>EC2</NT> instance.
1. [Allocate a dedicated host](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/allocating-a-dedicated-host-for-mac-instances) on <NT>AWS</NT>. You will need the automatically generated host ID.

</TabItem>
<TabItem value="linux-instance" label="Linux instance">

1. [Configure a runner pool](/en/bitrise-platform/infrastructure/configuring-runner-pools) and copy the token required for authentication.
1. Get the token from the process of adding the runner pool on <NT>Bitrise</NT>.
1. [Create an AWS Secrets Manager secret](https://docs.aws.amazon.com/secretsmanager/latest/userguide/create_secret.html) and store the token in the secret.
1. [Create an IAM role](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/iam-roles-for-amazon-ec2.html) with permission to read the secret. You will need to attach it to the <NT>EC2</NT> instance.

</TabItem>
</Tabs>


## Configuring the instance

<Tabs>
<TabItem value="mac-instance" label="Mac instance" default>

1. Go to the <NT>AWS</NT> Marketplace page, and on the left, select **<NT>Manage subscription</NT>s**.
1. Choose the AMI you need, and select **Launch new instance**.

   ![aws-markplace-amis.png](/img/_paligo/uuid-81bb90b8-85ea-ae9e-7349-3783f7af230e.png)
1. Make sure the software version and the region are correct, then click **<NT>Continue</NT> to launch through <NT>EC2</NT>**.
1. Fill out the required fields on the **Launch an instance** page:

   **<NT>Name</NT> and tags** section:

   - **<NT>Name</NT>**: The name of your instance.

   **<NT>Instance type</NT>** section:

   - **<NT>Instance type</NT>**: The supported types are: `mac2.metal`, `mac2-m2.metal`, `mac2-m2pro.metal`.

     :::important[Dedicated host type]

     The selected <NT>instance type</NT> must match the type of the dedicated host!

     :::

   **<NT>Key</NT> pair (login)** section:

   - **<NT>Key</NT> pair name - *required***:

   **Network settings** section - click **<NT>Edit</NT>** to modify the settings:

   - **VPC - *required***: Select a virtual private cloud from the dropdown menu.
   - **Subnet**: Select your preferred subnet from the dropdown menu, or leave it on **No preference**.
   - **Firewall (<NT>security groups</NT>)**: Create a new security group or select an existing one.

   **<NT>Configure</NT> storage** section:

   - Set the size of the storage volume in gigabytes:

     For the bare metal macOS AMI, the minimum number is 400 GB but we recommend at least 450 GB.

     For the virtualized macOS AMI, we recommend 700GB with 1 VM per host and 1000GB with 2 VM per host.

     :::note[Increasing the disk size]

     If you need to increase the disk size from the default 400 GB, re-partition the disk so macOS can use all the allocated storage. You can do this when setting up the instance by adding a script to **<NT>User data</NT>** in the **<NT>Advanced</NT> details** section: [Increasing disk size on a Mac instance](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances#increasing-disk-size-on-an-ec2-mac-instance).

     :::
   - Select a root volume type from the dropdown menu.

     :::note[Storage type]

     We recommend using at least the **gp3** root volume, the higher the IOPS the better.

     :::

   **<NT>Advanced</NT> details** section:

   - **Tenancy**: <NT>Select</NT> the **Dedicated host - launch this instance on a dedicated Host** option from the dropdown menu.
   - **<NT>Target</NT> host**: <NT>Select</NT> the **<NT>Host</NT> ID** option from the dropdown menu.
   - **Tenancy host ID**: Select the host ID of the previously created dedicated host.

</TabItem>
<TabItem value="linux-instance" label="Linux instance">

1. [Configure a runner pool](/en/bitrise-platform/infrastructure/configuring-runner-pools) and copy the token required for authentication.
1. Navigate to the <NT>EC2</NT> Dashboard.
1. Go to the <NT>AWS</NT> Marketplace page, and on the left, select **<NT>Manage subscription</NT>s**.
1. Choose the AMI you need, and select **Launch new instance**.

   ![aws-markplace-amis.png](/img/_paligo/uuid-81bb90b8-85ea-ae9e-7349-3783f7af230e.png)
1. Make sure the software version and the region are correct, then click **<NT>Continue</NT> to launch through <NT>EC2</NT>**.

   :::important[Service quotas]

   Make sure you have enough service quotas in your selected region to be able to launch as many instances as you need.

   - [Viewing current quotas](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-resource-limits.html#view-limits).
   - [Request an increase](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-resource-limits.html#request-increase).

   :::
1. Fill out the required fields on the **Launch an instance** page:

   **<NT>Name</NT> and tags** section:

   - **<NT>Name</NT>**: The name of your instance.

   **<NT>Instance type</NT>** section:

   - **<NT>Instance type</NT>**: Select an x86_64 <NT>Linux</NT> metal instance.

   **<NT>Key</NT> pair (login)** section:

   - **<NT>Key</NT> pair name - *required***:

   **Network settings** section - click **<NT>Edit</NT>** to modify the settings:

   - **VPC - *required***: Select a virtual private cloud from the dropdown menu.
   - **Subnet**: Select your preferred subnet from the dropdown menu, or leave it on **No preference**. The subnet must be able to access the internet either via a NAT Gateway or an Internet Gateway.
   - **Firewall (<NT>security groups</NT>)**: Create a new security group or select an existing one.

   **<NT>Configure</NT> storage** section:

   - Set the size of the storage volume in gigabytes. The minimum value is 450 GB.
   - Select a root volume type from the dropdown menu.

</TabItem>
</Tabs>


## Connecting the <NT>EC2</NT> Mac or <NT>Linux</NT> instance to your <NT>Bitrise</NT> <NT>Workspace</NT>

<Tabs>
<TabItem value="mac-bare-metal-instance" label="Mac bare metal instance" default>

1. Make sure that your instance can access the following endpoints:

   - https://den.services.bitrise.io
   - https://build-log.services.bitrise.io
1. [Modify the User data of the instance](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/user-data.html#user-data-cloud-init): add the command to launch the <NT>Bitrise</NT> runner, using the Secret you created in the <NT>AWS Secrets Manager</NT>:

   :::important

   When modifying the <NT>user data</NT> scripts for an <NT>EC2</NT> Mac instance, make sure that there is no empty space before the start of the script. The first line should always be `#!/bin/bash`. If there is empty space before this line, the instance won't work.

   :::

   ```bash
   TOKEN=$(aws secretsmanager get-secret-value --secret-id MY_SECRET | jq -r '.SecretString | fromjson | .["MY_SECRET"]')

   sudo sed -i '' "s/BITRISE_AGENT_TOKEN/$TOKEN/" /Users/ec2-user/Library/LaunchDaemons/io.bitrise.self-hosted-agent.plist
   sudo launchctl load -w /Users/ec2-user/Library/LaunchDaemons/io.bitrise.self-hosted-agent.plist
   ```
1. You can also set shell scripts or other custom data in the **<NT>User data</NT>** section.

   In the [Advanced options for EC2 Mac and Linux instances](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances) section, you can find recommendations for optimizing your instance with <NT>user data</NT>:

   - [Instance passwords](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances#connectivity-and-security-of-your-ec2-mac-and-linux-instance)
   - [Pre-warming the disk after booting](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances#pre-warming-the-disk-after-booting)
   - [Increasing disk size on a Mac instance](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances#increasing-disk-size-on-an-ec2-mac-instance)

</TabItem>
<TabItem value="mac-virtualized-instance" label="Mac virtualized instance">

1. Make sure that your instance can access the following endpoints:

   - https://den.services.bitrise.io
   - https://build-log.services.bitrise.io
1. [Modify the User data of the instance](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/user-data.html#user-data-cloud-init): add the command to launch the <NT>Bitrise</NT> runner, using the Secret you created in the <NT>AWS Secrets Manager</NT>. You also need to provide the number of concurrencies and the <NT><GlossTerm baseform="stack">stack</GlossTerm></NT> you wish to run.

   :::important

   When modifying the <NT>user data</NT> scripts for a macOS instance, make sure that there is no empty space before the start of the script. The first line should always be `#!/bin/bash`. If there is empty space before this line, the instance won't work.

   :::

   You can get the list of available stacks with the following command:

   ```bash
   /opt/virtualization-cli/bin/virtualization-cli list
   ```

   The allowed values for your concurrency preference are 1 or 2.

   - With 1 CC, the runner will schedule 1 VM with 8 vCPU and 12 GB RAM.
   - With 2 CC, the runner will schedule 2 VMs with 4 vCPU and 6 GB RAM each.

   ```bash
   TOKEN=$(aws secretsmanager get-secret-value --secret-id MY_SECRET | jq -r '.SecretString | fromjson | .["MY_SECRET"]')
   sudo sed -i '' 's/BITRISE_AGENT_CC/<YOUR_CONCURRENCY_PREFERENCE>/' /Users/ec2-user/Library/LaunchDaemons/io.bitrise.self-hosted-agent.plist
   sudo sed -i '' 's/BITRISE_AGENT_STACK/<YOUR_DESIRED_STACK>/' /Users/ec2-user/Library/LaunchDaemons/io.bitrise.self-hosted-agent.plist
   sudo sed -i '' "s/BITRISE_AGENT_TOKEN/$TOKEN/" /Users/ec2-user/Library/LaunchDaemons/io.bitrise.self-hosted-agent.plist
   sudo launchctl load -w /Users/ec2-user/Library/LaunchDaemons/io.bitrise.self-hosted-agent.plist
   ```
1. You can also set shell scripts or other custom data in the **<NT>User data</NT>** section.

   In the [Advanced options for EC2 instances](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances) section, you can find recommendations for optimizing your instance with <NT>user data</NT>:

   - [Instance passwords](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances#connectivity-and-security-of-your-ec2-mac-and-linux-instance)
   - [Pre-warming the disk after booting](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances#pre-warming-the-disk-after-booting)
   - [Increasing disk size on a Mac instance](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances#increasing-disk-size-on-an-ec2-mac-instance)

</TabItem>
<TabItem value="linux-instance" label="Linux instance">

1. Make sure that your instance can access the following endpoints:

   - https://den.services.bitrise.io
   - https://build-log.services.bitrise.io
1. [Modify the User data of the instance](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/user-data.html#user-data-cloud-init): add the command to launch the <NT>Bitrise</NT> runner, using the Secret you created in the <NT>AWS Secrets Manager</NT>:

   ```bash
   TOKEN=$(aws secretsmanager get-secret-value --secret-id MY_SECRET | jq -r '.SecretString | fromjson | .["MY_SECRET"]')
   /opt/bitrise/releases/bitrise-den-agent-configure.sh $TOKEN
   ```

   :::caution[<NT>Linux</NT> image version]

   If your <NT>Linux</NT> image version is EARLIER than `v2025W10`, the required command is slightly different:

   ```bash
   TOKEN=$(aws secretsmanager get-secret-value --secret-id MY_SECRET | jq -r '.SecretString | fromjson | .["MY_SECRET"]')

   sudo sed -i “s/BITRISE_AGENT_TOKEN/$TOKEN/” /etc/systemd/system/bitrise-den-agent.service
   sudo systemctl start bitrise-den-agent.service
   ```

   :::
1. You can also set shell scripts or other custom data in the **<NT>User data</NT>** section.

   In the [Advanced options for EC2 instances](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances) section, you can find recommendations for optimizing your instance with <NT>user data</NT>:

   - [Instance passwords](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances#connectivity-and-security-of-your-ec2-mac-and-linux-instance)
   - [Pre-warming the disk after booting](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances#pre-warming-the-disk-after-booting)
   - [Increasing disk size on a Mac instance](/en/bitrise-platform/infrastructure/bitrise-on-aws--manual-setup/advanced-options-for-ec2-instances#increasing-disk-size-on-an-ec2-mac-instance)

</TabItem>
