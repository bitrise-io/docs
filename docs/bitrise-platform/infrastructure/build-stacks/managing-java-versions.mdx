By default, every Bitrise <GlossTerm baseform="Stack">stack</GlossTerm> comes with [multiple Java versions](https://<GlossTerm baseform="stack">stacks</GlossTerm>.bitrise.io/tools/java/) pre-installed and ready to use. If you do not switch to another version, your build will use the default Java versions.

You can switch between the versions at any time. You can also install a different Java version.

:::tip[Configuring other tools]

For other tools, you can specify the version you want to use in your builds: [Configuring tool versions](/en/bitrise-ci/configure-builds/configuring-build-settings/configuring-tool-versions).

:::


## Setting Java version with the Set Java version <GlossTerm baseform="step">Step</GlossTerm>

Each Bitrise stack has multiple different [Java versions](https://stacks.bitrise.io/tools/java/) pre-installed and ready to use. You can easily switch between the different Java versions with our [**Set Java version** Step](https://www.bitrise.io/integrations/steps/set-java-version). The <GlossTerm baseform="Step">Step</GlossTerm> allows you to set the global Java version of the virtual machine that runs your build. This Step cannot install any Java version. It can only switch between the versions that are pre-installed on our stacks.

1. Add the **Set Java version** Step to your <GlossTerm baseform="Workflow">Workflow</GlossTerm>. We recommend setting it as the first Step of the <GlossTerm baseform="workflow">Workflow</GlossTerm>.
1. Find the **Java version to be set globally for the build** input.
1. Set it to the version you need.

**YAML example**

In this example, we're setting the Java version to 17 in the `bitrise.yml` file.

```yaml
primary:
  steps:
  - set-java-version@1:
      inputs:
      - set_java_version: '17'
```
