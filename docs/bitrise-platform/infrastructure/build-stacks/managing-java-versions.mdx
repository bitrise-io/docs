By default, every <NT>Bitrise</NT> <NT><GlossTerm baseform="Stack">stack</GlossTerm></NT> comes with [multiple Java versions](https://stacks.bitrise.io/tools/java/) pre-installed and ready to use. If you <NT>do not switch</NT> to another version, your build will use the default <NT>Java</NT> versions.

You can switch between the versions at any time. You can also install a different <NT>Java</NT> version.

:::tip[Configuring other tools]

For other tools, you can specify the version you want to use in your builds: [Configuring tool versions](/en/bitrise-ci/configure-builds/configuring-build-settings/configuring-tool-versions).

:::


## Setting <NT>Java</NT> version with the <NT>Set Java version</NT> <NT>Step</NT>

Each <NT>Bitrise</NT> stack has multiple different [Java versions](https://stacks.bitrise.io/tools/java/) pre-installed and ready to use. You can easily switch between the different <NT>Java</NT> versions with our [**Set Java version** Step](https://www.bitrise.io/integrations/steps/set-java-version). The <NT><GlossTerm baseform="Step">Step</GlossTerm></NT> allows you to set the global <NT>Java</NT> version of the virtual machine that runs your build. This <NT>Step</NT> cannot install any <NT>Java</NT> version. It can only switch between the versions that are pre-installed on our stacks.

1. Add the **<NT>Set Java version</NT>** <NT>Step</NT> to your <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>. We recommend setting it as the first <NT>Step</NT> of the <NT>Workflow</NT>.
1. Find the **<NT>Java</NT> version to be set globally for the build** input.
1. Set it to the version you need.

**YAML example**

In this example, we're setting the <NT>Java</NT> version to 17 in the `bitrise.yml` file.

```yaml
primary:
  steps:
  - set-java-version@1:
      inputs:
      - set_java_version: '17'
```
