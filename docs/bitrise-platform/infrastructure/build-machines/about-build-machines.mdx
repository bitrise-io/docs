For [Bitrise CI](/en/bitrise-ci), both <NT>Linux</NT> and <NT>Xcode</NT> <NT><GlossTerm baseform="stack">stack</GlossTerm></NT>s are available on several different <NT>build machine</NT> types. Each machine type offers several options with varying computing power, depending on [your subscription plan](http://www.bitrise.io/pricing).

You can configure a default machine type for each of your projects and you can also set <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>-specific machine types. You can do this when selecting stacks for your project: [Setting the stack for your builds](/en/bitrise-ci/configure-builds/configuring-build-settings/setting-the-stack-for-your-builds).

:::tip[Changing machine types using the API]

You can also change <NT>build machine</NT> types for all projects owned by a single user or <NT><GlossTerm baseform="workspace">workspace</GlossTerm></NT> using the API: [Changing machine types in all apps at the same time](/en/bitrise-ci/api/adding-and-managing-apps#changing-machine-types-in-all-apps-at-the-same-time).

:::

Each <NT>Bitrise</NT> <NT>build machine</NT> has its own IP address range: you can allowlist these IP addresses to be able to access our <NT>build machines</NT> from, for example, a private cloud: [Configuring your network to access our build machines](/en/bitrise-platform/infrastructure/build-machines/configuring-your-network-to-access-our-build-machines)

For more information about build and code security, see [Code security](/en/bitrise-platform/infrastructure/code-security).
