To run builds, <NT>Bitrise CI</NT> must be able to access a Git repository: when a build starts, we create a virtual machine and clone your repository on it, which requires authentication. <NT>Bitrise</NT> can authenticate to the Git repository in one of three ways:

- [GitHub app](#github-app): <NT>Bitrise</NT> provides a <NT>GitHub</NT> app for both <NT>GitHub Cloud</NT> and <NT>GitHub</NT> Enterprise.
- [SSH keys](#ssh-key-authorization): Create a public and private key, and register the public key at your Git provider, allowing <NT>Bitrise</NT> to authenticate when running builds.
- [Personal access token](#https-authorization): Generate a personal access token at your Git provider and register it at <NT>Bitrise</NT>. You can use this for HTTPS authorization.

You configure repository access when adding a new project but you can change every setting, including the authentication method and the repository URL itself, later.

There are two ways to select a repository when [adding a new project](/en/bitrise-ci/getting-started/adding-a-new-project):

- Manually pasting a repository URL.
- Connect your <NT>Bitrise</NT> account to a Git provider account. This allows you to select from a list of available repositories when adding a new project.

Selecting a repository URL is not final: you can always [change it later](/en/bitrise-platform/projects/configuring-the-repository-url-and-the-default-branch#changing-the-repository-url).

If you connect your <NT>Bitrise</NT> account to a Git provider account, authentication depends on the <NT>connection type</NT>:

- The <NT>GitHub</NT> app uses short-lived access tokens and therefore doesn't require either an SSH key or a personal access token.
- The OAuth connection requires either an SSH key or a personal access token, depending on the repository URL.


## <NT>GitHub</NT> app

The best way to connect your <NT>Bitrise</NT> account to your <NT>GitHub</NT> repositories is by integrating with [a GitHub app](https://docs.github.com/en/apps/overview). The <NT>Bitrise</NT> <NT>GitHub</NT> app has a number of features that make integration easier:

- The app eliminates the need for an SSH key, a Personal Access Token, and a <NT>service credential user</NT>. For access, it generates a temporary access token before every build, significantly increasing security.
- It provides granular access to your repositories: no need to grant access to an entire <NT>GitHub</NT> organization, instead select the specific repositories the app can access.
- With the app installed, you automatically receive <NT>Bitrise</NT> status updates directly on <NT>GitHub</NT> with the <NT>GitHub</NT> Checks app. No need for configuring status reports separately. Note that while only one <NT>GitHub</NT> account can be connected to a <NT>Bitrise</NT> <NT><GlossTerm baseform="workspace">Workspace</GlossTerm></NT>, <NT>GitHub</NT> Checks can still be configured for repositories owned by other <NT>GitHub</NT> accounts.

:::important[HTTPS URL required]

The <NT>GitHub</NT> App requires an HTTPS URL for your repository instead of an SSH one. Normally, you don't have to worry about this: setting up the <NT>GitHub</NT> App connection changes the URL of your project. If there's an issue, you can change the URL manually: [Changing the repository URL](/en/bitrise-platform/projects/configuring-the-repository-url-and-the-default-branch#changing-the-repository-url).

:::

Repository access with the <NT>GitHub</NT> app is configured differently for <NT>GitHub Cloud</NT> and <NT>GitHub</NT> Enterprise users.

- [GitHub Cloud guide](/en/bitrise-platform/repository-access/github-app-integration)
- [GitHub Enterprise guide](/en/bitrise-platform/repository-access/integrating-github-enterprise-with-bitrise)


## OAuth connection

An OAuth connection allows a third-party application to access a user's data on a service without sharing their login credentials. On <NT>Bitrise</NT>, an OAuth connection means connecting your <NT>Bitrise</NT> account to your Git provider account.

<NT>Bitrise</NT> supports an OAuth connection with three Git providers: <NT>GitHub</NT>, <NT>GitLab</NT>, and <NT>Bitbucket</NT>. With an OAuth connection, <NT>Bitrise</NT> can:

- Retrieve and display a list of the available repositories when [adding a new project](/en/bitrise-build-cache/getting-started-with-the-build-cache/getting-started-with-the-build-cache#adding-a-new-connection-to-the-build-cache).
- [Automatically register webhooks](/en/bitrise-platform/integrations/webhooks/adding-incoming-webhooks#registering-an-incoming-webhook-automatically). Webhooks allow setting up <NT><GlossTerm baseform="Trigger">build triggers</GlossTerm></NT> and enable the use of [Git Insights](/en/insights/git-insights).
- [Automatically register SSH keys](/en/bitrise-platform/repository-access/configuring-ssh-keys).

To set up an OAuth connection, check out: [Repository access with OAuth](/en/bitrise-platform/repository-access/repository-access-with-oauth).


## SSH key authorization

If your repository URL is an SSH URL and you use an OAuth connection, you can authorize <NT>Bitrise</NT> with SSH keys.

SSH key authorization requires a public and private SSH key. The public key is registered to your <NT>Bitrise</NT> <NT><GlossTerm baseform="project">project</GlossTerm></NT>'s Git repository. Information encrypted with the public key requires the private key, registered at <NT>Bitrise</NT>, to be decrypted.

You can add an SSH key pair when adding a new project on <NT>Bitrise</NT> or you can configure them later at the **<NT>Project settings</NT>** page. You have the option to:

- **<NT>Auto-add a generated SSH key to your repository</NT>**: <NT>Bitrise</NT> automatically registers a public SSH key to your <NT>GitHub</NT> repository. Choose this if you have administrator rights to the repository.
- **<NT>Copy a generated SSH key to your Git provider manually</NT>**: <NT>Bitrise</NT> generates an SSH keypair for you, and you have to manually register the public key to your Git repository.
- **<NT>Add</NT> your own <NT>SSH key</NT>**: You provide your own SSH keypair for authentication, and you have to manually register the public key to your Git repository.

Set up SSH key authorization: [Configuring SSH keys](/en/bitrise-platform/repository-access/configuring-ssh-keys).


## HTTPS authorization

If your repository URL is an HTTPS URL and you use an OAuth connection, you can authorize <NT>Bitrise</NT> with a personal access token.

HTTPS authorization requires a personal access token for private projects. You create the token at your Git provider and register it at <NT>Bitrise</NT>.

For most projects with OAuth connection, we recommend SSH URLs and SSH key authorization. We recommend using HTTPS URLs only for [public projects](/en/bitrise-platform/projects/public-projects): those can't have SSH keys.

Set up HTTPS authorization: [Configuring HTTPS authorization credentials](/en/bitrise-platform/repository-access/configuring-https-authorization-credentials).
