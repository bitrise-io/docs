<NT>Bitrise</NT> needs access to your code in a Git repository to be able to build. You can provide access on a project-by-project basis but we recommend connecting your <NT>Bitrise</NT> account to your Git provider accounts (<NT>GitHub</NT>, <NT>GitLab</NT>, and <NT>Bitbucket</NT>).

One of the ways to connect your accounts is by setting up an OAuth connection. You can connect all three Git provider accounts using OAuth, by either:

- Connecting the account when adding a new <NT>Bitrise</NT> <NT><GlossTerm baseform="project">project</GlossTerm></NT>.
- Connecting the account from your [Account settings](http://app.bitrise.io/me/profile#/edit_profile) page.

:::important[One account per Git provider]

Please note that you cannot connect two accounts from the same Git provider (for example, two <NT>GitHub</NT> accounts) to <NT>Bitrise</NT>.

:::

Connecting one Git provider account is not final. You can disconnect an account and connect a different one at any time.


## Connecting a Git provider with OAuth when adding a project

1. <NT>Log in</NT> to your [bitrise.io](https://www.bitrise.io) account and select **<NT>Bitrise CI</NT>** on the left.
1. Click **<NT>New CI project</NT>**.
1. Set the project’s privacy setting.
1. In the **<NT>Select repository</NT>** section, choose between selecting a remote repository or entering an URL manually.

   If you opt to enter a git URL manually, add it in the **<NT>Git repository (clone) URL</NT>** field and click **<NT>Next</NT>**.
1. Choose the <NT>Git</NT> service provider of the app’s repository, if you chose to select a remote repository. If no account with that provider has been connected to your <NT>Bitrise</NT> account, the UI will display the option to connect.

   ![select_git_repo.png](/img/_paligo/uuid-909bb98f-3195-8726-8b09-a60a33209ccb.png)
1. Click **<NT>Connect account</NT>**. This will take you to the login page of the Git provider.
1. <NT>Log in</NT> to the Git provider account.
1. You should be prompted to authorize bitrise.io - do it! If successful, you should be redirected to <NT>Bitrise</NT>, and a pop-up message should inform you that you successfully linked the account. Click **Okay**.


## Connecting a Git provider with OAuth from the <NT>Account settings</NT> page

Connecting a Git provider account with an OAuth application allows <NT>Bitrise</NT> to:

- List the available repositories when [adding a new project](/en/bitrise-build-cache/getting-started-with-the-build-cache/getting-started-with-the-build-cache#adding-a-new-connection-to-the-build-cache).
- [Automatically register webhooks](/en/bitrise-platform/integrations/webhooks/adding-incoming-webhooks#registering-an-incoming-webhook-automatically). Webhooks allow setting up <NT><GlossTerm baseform="Trigger">build triggers</GlossTerm></NT> and enable the use of [Git Insights](/en/insights/git-insights).
- [Automatically register SSH keys](/en/bitrise-platform/repository-access/configuring-ssh-keys).

1. <NT>Log in</NT> to <NT>Bitrise</NT>, and select **<NT>Bitrise CI</NT>** from the left navigation menu.
1. In the upper right corner, click the profile image to open the dropdown menu.
1. Select the **<NT>Account settings</NT>** option.

   ![account-settings-page.png](/img/_paligo/uuid-b08c42b9-affd-b40d-3c6e-87410001d2fb.png)
1. On the left, under **<NT>Connected</NT> <NT>Git providers</NT>**, click the toggle next to the name of the Git provider you wish to connect.
1. <NT>Log in</NT> to the Git provider account.
1. You should be prompted to authorize bitrise.io - do it! If successful, you should be redirected to <NT>Bitrise</NT>, and a pop-up message should inform you that you successfully linked the account. Click **Okay**.


## Disconnecting a Git provider account

1. <NT>Log in</NT> to <NT>Bitrise</NT>, and select **<NT>Bitrise CI</NT>** from the left navigation menu.
1. In the upper right corner, click the profile image to open the dropdown menu.
1. Select the **<NT>Account settings</NT>** option.

   ![account-settings-page.png](/img/_paligo/uuid-b08c42b9-affd-b40d-3c6e-87410001d2fb.png)
1. On the left under **<NT>Connected</NT> <NT>Git providers</NT>**, click the toggle next to the name of the Git provider you wish to disconnect.
