A <NT><GlossTerm baseform="workspace">Workspace</GlossTerm></NT> API token allows <NT>Workspace</NT> members to access the [Bitrise API](/en/bitrise-ci/api/api-overview) without each member having to set up their own [Personal Access Token](/en/bitrise-platform/accounts/personal-access-tokens). For API authentication, the <NT>Workspace</NT> token functions the exact same way as the Personal Access Token.

A <NT>Workspace</NT> owner can set up multiple <NT>Workspace</NT> API tokens, each with different access rights to the apps owned by the <NT>Workspace</NT>. The access rights are determined by [user roles](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci), just as it is for [Workspace members](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/workspace-collaboration) or [Workspace groups](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/workspace-groups).

We recommend setting up any integrations that require an access token for API access using a <NT>Workspace</NT> API token instead of individual users' Personal Access Token.


## Creating a <NT>Workspace</NT> API token

1. <Partial_OpeningTheWorkspaceSettingsPage />
1. On the left, select **<NT>Security</NT>**.
1. Click **<NT>Create token</NT>**.

   ![workspace-api-token.svg](/img/_paligo/uuid-cb123ebc-f548-8e87-615f-aab39fefa078.svg)
1. In the dialog box, set a name and an expiration date then click **<NT>Next</NT>**.

   ![create-token.svg](/img/_paligo/uuid-1610488e-2548-99b4-8386-9f2f66bb7f03.svg)
1. <NT>Set up</NT> the user roles in apps. Under **Set roles in apps**, you can:

   - Select the **All apps** option: the token can be used to access all current and future apps of the <NT>Workspace</NT> with the same user role. Select the role in the **<NT>Roles</NT>** menu.
   - Select the **<NT>Roles</NT> per apps** option: the token can be used to access only the specified apps. You have to select the user roles for each app. If you leave the user role on **<NT>None</NT>** for a given app, you can't access that app with the <NT>Workspace</NT> API token.

   ![create-per-app.png](/img/_paligo/uuid-774a69fb-d5b9-333d-f373-eab6cab793cb.png)
1. Click **<NT>Next</NT>** to get the actual token.
1. Click **<NT>Copy and close</NT>** to copy the token and close the dialog box.

   :::caution[Save the token]

   Once you closed the dialog box, you won't be able to see the token again!

   :::


## Regenerating a <NT>Workspace</NT> API token

If a <NT>Workspace</NT> API token is lost for any reason, there is no way to view it again. You have the option of regenerating the existing token: this way you don't have to create an entirely <NT>new token</NT> and configure app access again. The regenerated token will:

- Have a new expiration date.
- Have access to the exact same apps with the exact same roles as before.

1. <Partial_OpeningTheWorkspaceSettingsPage />
1. On the left, select **<NT>Security</NT>**.
1. Find your token and click the **<NT>View details</NT>** icon next to its name.

   ![view-details-token.png](/img/_paligo/uuid-efe62dac-0331-a3e4-6bf8-57318826bcd3.png)
1. Click **<NT>Regenerate</NT>**.

   ![regenerate-token.png](/img/_paligo/uuid-3850ebd0-6ab7-934f-c232-4305ccffe249.png)
1. Select the new expiration date.
1. Click **<NT>Regenerate</NT>** to get the <NT>new token</NT>.
1. Click **<NT>Copy and close</NT>**.

   Make sure to save the token in a secure way: you won't be able to see it again.
