Members of a <NT>Bitrise</NT> <NT><GlossTerm baseform="workspace">workspace</GlossTerm></NT> can be organized into <NT>workspace</NT> groups. Groups allow flexible allocation of <NT>Workspace</NT> members between apps, simplifying the process of managing <NT>workspaces</NT> that own multiple projects.

<NT>Workspace</NT> members can belong to multiple different groups. Groups can be assigned user roles on any project's team: each member of the group will have the group's role and the access rights that come with that role.

:::note[Difference between project collaborators and <NT>workspace</NT> groups]

Collaborators on a project are handled on a project basis, while <NT>workspace</NT> groups are handled globally within a <NT>workspace</NT>.

A <NT>workspace</NT> group has no inherent roles on its own. [You can assign groups to projects](/en/bitrise-platform/projects/managing-user-access-to-a-project#adding-workspace-groups-to-a-project) and choose their [role](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci) there, which means the same group can have different roles on different projects.

:::

<NT>Workspace</NT> groups can have three main types of product access:

- **<NT>Admin access</NT>**: Gives access to all aspects of the <NT>selected projects</NT> with the project admin role. This doesn't necessarily mean access to all projects owned by the <NT>workspace</NT>!
- **<NT>Bitrise CI</NT> access**: The group can only access the <NT>Bitrise CI</NT> configuration of a project, with the specific [Bitrise CI roles and permissions](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci).
- **<NT>Release Management</NT> access**: The group can only access the <NT>Release Management</NT> apps of the project, with the specific [Release Management permissions](/en/release-management/configuring-connected-apps/release-management-roles-and-permissions).


## Global access groups

You can grant a <NT>workspace</NT> member access to all projects on all products. When you do that, the member is added to a global access group.

A global access group is a <NT>workspace</NT> group, with a few differences compared to regular <NT>workspace</NT> groups:

- It can't be removed from the <NT>workspace</NT>.
- Its roles can't be changed.

You can add members to global access groups manually, just like to any other group. You can also rename global access groups at any time.


## Creating groups for <NT>Workspaces</NT>

:::important[Owners and managers only]

Only <NT>Workspace</NT> owners and managers can create groups for a <NT>workspace</NT>.

:::

When you create a new group, you must decide which products it should have access to.

1. <Partial_OpeningTheWorkspaceSettingsPage />
1. On the left, select **<NT>Collaboration</NT>** from the menu options.
1. Select the **<NT>Groups</NT>** tab.
1. Click the **<NT>Create group</NT>** button to bring up a dialogue box.
1. Enter a group name in the **<NT>Group name</NT>** field.
1. <NT>Configure access to products</NT>:

   - **<NT>Admin access</NT>**: Grants access to all products, managing all aspects of <NT>selected projects</NT> with the project admin role.
   - **<NT>Bitrise CI</NT>**: The group has access to <NT>Bitrise CI</NT> only, and it can be assigned [Bitrise CI roles](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci).
   - **<NT>Release Management</NT>**: The group has access to <NT>Release Management</NT> only, and it can be assigned [Release Management roles](/en/release-management/configuring-connected-apps/release-management-roles-and-permissions).
1. If you toggle on a specific product instead of admin access, you will be prompted to select the projects for the group and its user roles on the <NT>selected projects</NT>.

   - For <NT>Bitrise CI</NT>, you can select projects and assign a role to each project. You can also select the **<NT>All projects</NT>** option: in this case, the user will be added to a global access group with the selected role.

   - For <NT>Release Management</NT> in particular, you can select:

   - Specific projects. Note that the group will be able to access all currently existing apps pf the project but not all future apps.
   - Specific <NT>Release Management</NT> apps: select the already existing apps you want the group to have access to.
1. When finished with setting up roles, click **<NT>Done</NT>**.
1. Click **<NT>Create group</NT>**.


## Adding members to a <NT>Workspace</NT> group

You can add <NT>Workspace</NT> members to <NT>Workspace</NT> groups for more convenient management of <NT>Workspace</NT> members. Groups allow you to assign several members to project teams at the same time.

1. <Partial_OpeningTheWorkspaceSettingsPage />
1. On the left, select **<NT>Collaboration</NT>** from the menu options.
1. Select the **<NT>Groups</NT>** tab.
1. Find the group you need and click the 3 dots to the right of the group to bring up the group dropdown menu.
1. Click on the **+ <NT>Add member</NT>** button.

   ![Add_member_new.png](/img/_paligo/uuid-642e1898-479a-7590-b0ae-3d3a5cc6b9c7.png)
1. <NT>Enter an email address</NT> or add an existing member to the group using the radio buttons in the dialogue box.

   - Select **Invite a <NT>new member</NT> to the <NT>Workspace</NT>** and <NT>enter an email address</NT>. Click **<NT>Send</NT> Invite** on the bottom to send the invitation.
   - Select **<NT>Add</NT> an existing member** and select one of the existing <NT>Workspace</NT> members from the dropdown menu. Click **<NT>Add member</NT>** on the bottom to add the team member to the group.
