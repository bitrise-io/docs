Your <NT><GlossTerm baseform="Workspace">workspace</GlossTerm></NT> isn't just an environment to manage <NT><GlossTerm baseform="Project">project</GlossTerm></NT>s: it also allows seamless collaboration with other <NT>Bitrise</NT> users. You can invite other users to your <NT>workspaces</NT>, and of course you can be invited to other <NT>workspaces</NT>, too.

You can manage <NT>workspace</NT> membership on the **<NT>Collaboration</NT>** page of the <NT>workspace settings</NT>. You can add members to a <NT>workspace</NT> in three ways:

- Invite individual members: [Adding members to workspaces](#adding-members-to-workspaces).
- Add them to a project as outside contributors.
- Organize them into [workspace groups](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/workspace-groups).

<NT>Workspace</NT> members can have different roles in a <NT>workspace</NT>. <NT>Workspace</NT> roles determine the different actions that the members have permissions for. You can configure product access for <NT>workspace</NT> members: each member can have access to <NT>Bitrise CI</NT>, <NT>Release Management</NT>, or both. For each project within a product, you can set up different project-level roles for all <NT>workspace</NT> members: [Roles and permissions in workspaces](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/roles-and-permissions-in-workspaces).

If SAML SSO is enabled and configured for a <NT>Bitrise</NT> <NT>workspace</NT>, <NT>workspace</NT> members can use SAML SSO to log in to their <NT>Bitrise</NT> account: [Configuring SAML SSO on Bitrise](/en/bitrise-platform/accounts/saml-sso-in-bitrise/configuring-saml-sso-on-bitrise).


## Checking <NT>workspace</NT> member status

You can check out the most important information about <NT>workspace</NT> members at any point.

1. <Partial_OpeningTheWorkspaceSettingsPage />
1. Select **<NT>Collaboration</NT>**.
1. Go to the **<NT>Members</NT>** tab. On this tab:

   - You can [add and remove workspace members](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/workspace-collaboration#adding-members-to-workspaces).
   - If [SAML SSO](/en/bitrise-platform/accounts/saml-sso-in-bitrise/configuring-saml-sso-on-bitrise) is enabled for a member.
   - See each member's [workspace role](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/roles-and-permissions-in-workspaces) and [product access](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/roles-and-permissions-in-workspaces#project-level-roles).

You can search for a specific <NT>Workspace</NT> member or use the filter function to filter members based on their roles.


## Adding members to <NT>workspaces</NT>

:::caution[Starter plan restrictions]

On Starter plans, collaboration features for <NT>workspaces</NT> are not available. You have to add members on a project basis: [Adding outside contributor to a project](/en/bitrise-platform/projects/managing-user-access-to-a-project#adding-an-outside-contributor-to-a-project).

:::

You can invite people to a <NT>workspace</NT> if they have a <NT>Bitrise</NT> account. <NT>Workspace</NT> members can be assigned to the projects owned by the <NT>workspace</NT>. When inviting new members, you can:

- Add users in bulk: You can invite as many members as the number of available seats on the <NT>workspace</NT>.
- Add them to all projects owned by the <NT>workspace</NT>, or just to particular projects.
- Assign them [user access roles](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci) to manage their permissions on the project. When adding users in bulk, all of them will be assigned the same role.
- Add users to [global access groups](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/workspace-groups#global-access-groups): The invited users will have the same role on all projects owned by the <NT>workspace</NT>. You can't select specific projects when adding users this way.

:::note[Invitation limits]

To prevent abuse, invitations sent from workspaces on non-paid plans may be rate-limited. If you reach a limit while inviting members, wait a while before trying again.

:::

:::note[SAML SSO]

<NT>Bitrise</NT> supports SAML SSO for your <NT>workspace</NT>. To learn more, check out [SAML SSO in Bitrise](/en/bitrise-platform/accounts/saml-sso-in-bitrise/configuring-saml-sso-on-bitrise).

Inviting a new member to a <NT>workspace</NT> with SAML SSO enforced is a different process: [Adding a new user to a Workspace with enforced SAML SSO](/en/bitrise-platform/accounts/saml-sso-in-bitrise/configuring-saml-sso-on-bitrise#adding-a-new-user-to-a-workspace-with-enforced-saml-sso).

:::

You can modify a member's user roles on projects at any time after inviting them. You can also add members to [workspace groups](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/workspace-groups#creating-groups-for-workspaces).

To add individuals as a <NT>workspace</NT> member:

1. <Partial_OpeningTheWorkspaceSettingsPage />
1. On the left, select **<NT>Collaboration</NT>**.
1. On the **<NT>Members</NT>** tab, click **<NT>Add members</NT>**.
1. In the dialog box, find the **<NT>New members</NT>** section and add the email addresses of the people you want to invite.

   ![adding-users.png](/img/_paligo/uuid-e09eda1b-3f0e-58ce-ab3f-979d05e557e6.png)

   :::note[<NT>Workspace</NT> limits]

   You can't invite people who are already part of the <NT>workspace</NT> and the number of invitations can't exceed the available seats of the <NT>workspace</NT>.

   :::
1. When done, click **<NT>Next</NT>**. It takes you to the **Project access** page.

   - **<NT>Admin access</NT>**: Grants access to all products, managing all aspects of <NT>selected projects</NT> with the project admin role.
   - **<NT>Bitrise CI</NT>**: The user has access to <NT>Bitrise CI</NT> only, and it can be assigned [Bitrise CI roles](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci).
   - **<NT>Release Management</NT>**: The user has access to <NT>Release Management</NT> only, and it can be assigned [Release Management roles](/en/release-management/configuring-connected-apps/release-management-roles-and-permissions).
1. If you toggle on a specific product instead of admin access, you will be prompted to select the projects for the user and its user roles on the <NT>selected projects</NT>.

   - For <NT>Bitrise CI</NT>, you can select projects and assign a role to each project. You can also select the **<NT>All projects</NT>** option: in this case, the user will be added to [a global access group](/en/bitrise-platform/workspaces/collaboration-and-permissions-in-workspaces/workspace-groups#global-access-groups) with the selected role.

   - For <NT>Release Management</NT>, you can select:

   - Specific projects. The group will be able to access all currently existing apps of the project but not all future apps.
   - Specific <NT>Release Management</NT> apps: Select the already existing apps you want the group to have access to.
1. When finished with setting up roles, click **<NT>Done</NT>**.
1. Click **<NT>Confirm access</NT>** to finish the process.


## Removing members from <NT>Workspaces</NT>

:::important[Owners and managers only]

Only <NT>Workspace</NT> owners and <NT>Workspace</NT> managers can remove users from a <NT>Workspace</NT>.

:::

1. <Partial_OpeningTheWorkspaceSettingsPage />
1. On the left, select **<NT>Collaboration</NT>**.
1. Select the **<NT>Members</NT>** tab.
1. Search for the user you would like to remove, and click "![remove_png.png](/img/_paligo/uuid-ac29f14a-dd0f-9360-8e51-6bc1c83f724d.png)".
1. Click **<NT>Remove</NT>** to remove the selected user from your <NT>Workspace</NT>.

   ![remove_member_confirm.png](/img/_paligo/uuid-ecd84027-5433-4ce8-cfbe-bbd61a77c095.png)
