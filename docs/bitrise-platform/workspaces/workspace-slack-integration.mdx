You can configure a <NT>Slack</NT> integration for your <NT><GlossTerm baseform="workspace">workspace</GlossTerm></NT> that can send notification messages to selected <NT>Slack</NT> channels. You can use this integration to send messages of:

- Build events in <NT>Bitrise CI</NT>, with the **[Send a Slack message](https://github.com/bitrise-io/steps-slack-message)** <NT><GlossTerm baseform="step">Step</GlossTerm></NT>.
- Release events in <NT>Release Management</NT>.
- Alerts in Insights.

The integration is an OAuth application with an incoming <NT>Slack</NT> webhook. You can configure the integration for multiple channels: if, for example, you want <NT>Release Management</NT> events posted to a different channel than CI notifications, you can do that.

The integration can view basic information about public channels in your <NT>Slack</NT> <NT>workspace</NT> and post to both public and private channels without being a member of the channel.

If you are logged in to <NT>Slack</NT>, you can use the below button to connect the <NT>Bitrise</NT> <NT>Slack</NT> app to your <NT>Slack</NT> <NT>workspace</NT>:

[Add to Slack](https://app.bitrise.io/workspace/integrations/slack/oauth)

Read the [Bitrise privacy policy](https://go.bitrise.io/privacy-policy) before connecting the <NT>Slack</NT> app and check out our [Pricing page](https://bitrise.io/pricing) to learn about the <NT>Bitrise</NT> plans available for your <NT>workspace</NT>.

You can also configure the <NT>Slack</NT> integration from the **<NT>Workspace settings</NT>** page:

## Configuring the <NT>Slack</NT> integration

To set up the <NT>Slack</NT> integration for a given channel:

1. <Partial_OpeningTheWorkspaceSettingsPage />
1. Select **<NT>Integrations</NT>** and then select the <NT>Slack</NT> card.

   ![slack-inte.png](/img/_paligo/uuid-952a4ad0-8b1d-4707-2062-f4683aaa854d.png)
1. Click **<NT>Add configuration</NT>**.
1. In the dialog, view the permissions of the integration and select a channel.

   You can select private and public channels, as well as direct messages with <NT>Slack</NT> users.
1. Click **<NT>Allow</NT>**.

## Renaming the configuration

By default, the integration gets an automatically generated name. You can rename it at any time to create an easily identifiable configuration.

1. Open the **<NT>Workspace settings</NT>** page.
1. Select **<NT>Integrations</NT>** and then the <NT>Slack</NT> card.

   ![slack-inte.png](/img/_paligo/uuid-952a4ad0-8b1d-4707-2062-f4683aaa854d.png)
1. Find your configuration and click the options menu (⋮) next to its name.
1. Select **<NT>Rename</NT>**.

   ![rename-int.png](/img/_paligo/uuid-a5864e82-1efc-0191-b37c-a162cfcb8f42.png)
1. Type a new name and click **<NT>Save changes</NT>**.

## Getting the integration ID

The integration ID identifies the <NT>Slack</NT> integration: it allows you to use it with the [**Send a Slack message**](https://github.com/bitrise-io/steps-slack-message) <NT>Step</NT>. To get it:

1. Open the **<NT>Workspace settings</NT>** page.
1. Select **<NT>Integrations</NT>** and then the <NT>Slack</NT> card.

   ![slack-inte.png](/img/_paligo/uuid-952a4ad0-8b1d-4707-2062-f4683aaa854d.png)
1. Find your configuration and click the options menu (⋮) next to its name.
1. Select **<NT>Copy integration ID</NT>**.

   ![integrations-id.png](/img/_paligo/uuid-3b1378b0-3006-c0d1-a710-7c4a51821841.png)
