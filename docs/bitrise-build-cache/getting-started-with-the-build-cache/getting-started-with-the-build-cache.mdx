The [Bitrise Build Cache](https://bitrise.io/platform/devops/build-caching) is a fully managed caching solution that reduces CI build durations for applications built with <NT>Gradle</NT>, Bazel, and <NT>Xcode</NT>. It specifically caches build and test outputs to minimize how much work is done in subsequent builds, making it more efficient in environments with frequent updates. Compatible with any CI tool, it accelerates the build cycle without requiring you to manage a caching infrastructure.

We offer a <NT>30-day free trial</NT> at no cost; you don't even need to provide payment information. The trial starts automatically when you set up the <NT>Bitrise</NT> <NT>Build Cache</NT>.

[Click here to get started with the Bitrise Build Cache](https://app.bitrise.io/build-cache). If you don't have a <NT>Bitrise</NT> account, you will be prompted to create one first before proceeding to set up the <NT>Build Cache</NT>.

:::

<NT>Bitrise</NT> supports remote build caching for the following build systems:

- [Build Cache for Gradle](/en/bitrise-build-cache/build-cache-for-gradle/configuring-the-build-cache-for-gradle-in-the-bitrise-ci-environment).
- [Build Cache for Bazel](/en/bitrise-build-cache/build-cache-for-bazel/configuring-the-build-cache-for-bazel-in-the-bitrise-ci-environment).
- [Build Cache for Xcode](/en/bitrise-build-cache/build-cache-for-xcode/configuring-the-build-cache-for-xcode-in-the-bitrise-ci-environment).

## Adding a <NT>new connection</NT> to the <NT>Build Cache</NT>

To start using the <NT>Bitrise</NT> <NT>Build Cache</NT>, you have to add a <NT>new connection</NT>. The process consists of:

- Selecting a CI provider: you can use either <NT>Bitrise</NT> or another CI provider.
- Selecting a build tool: currently, Bazel, <NT>Gradle</NT>, and <NT>Xcode</NT> are supported.
- If you use <NT>Bitrise</NT> as your CI provider, selecting a <NT>Bitrise</NT> <NT>project</NT>.
- If you use a different CI provider, adding [a personal access token](/en/bitrise-platform/accounts/personal-access-tokens#creating-a-personal-access-token) to allow the <NT>Bitrise</NT> <NT>Build Cache</NT> access to your CI.
- Adding the cache activation scripts to your CI process. On <NT>Bitrise</NT>, we have dedicated [Steps](/en/bitrise-ci/workflows-and-pipelines/steps/steps-overview) for this.

To add a <NT>new connection</NT>:

1. <NT>Log in</NT> to <NT>Bitrise</NT> and select the **<NT>Build Cache</NT>** from the left navigation menu.
1. On the top right corner, click **<NT>New connection</NT>**.

   ![cache-new-connection.png](/img/_paligo/uuid-bdbb02c7-b8e7-bf04-fb52-7cacd2231760.png)
1. Follow the instructions.

   For more information on configuring the <NT>Build Cache</NT>, refer to our dedicated guides:

   - [Build Cache for Gradle](/en/bitrise-build-cache/build-cache-for-gradle/configuring-the-build-cache-for-gradle-in-the-bitrise-ci-environment).
   - [Build Cache for Bazel](/en/bitrise-build-cache/build-cache-for-bazel/configuring-the-build-cache-for-bazel-in-the-bitrise-ci-environment).
   - [Build Cache for Xcode](/en/bitrise-build-cache/build-cache-for-xcode/configuring-the-build-cache-for-xcode-in-the-bitrise-ci-environment).
