You can add the activation either through the <NT><GlossTerm baseform="workflow editor">Workflow Editor</GlossTerm></NT> or directly in `bitrise.yml`.

<Tabs>
  <TabItem value="workflow-editor" label="Workflow Editor">

  1. Make sure you have:

     - An active <NT>Bitrise</NT> <NT>Build Cache</NT> trial or subscription. You can check your subscription status on the [Bitrise Build Cache page](https://app.bitrise.io/build-cache/).
     - A <NT>React Native</NT> project that already builds successfully without <NT>Build Cache</NT> (so you have a clear baseline).
     - For iOS projects: **<NT>Xcode 26 or later</NT>**.
  1. <Partial_OpeningTheWorkflowEditor />
  1. Select the <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT> you need.
  1. Add the **<NT>Build Cache for React Native</NT>** step *before* any <NT><GlossTerm baseform="step">Step</GlossTerm></NT> that <NT><GlossTerm baseform="trigger">trigger</GlossTerm></NT>s a native build.

     For example, before **Script** <NT>Steps</NT> that run `yarn`, `npm`, `npx`, `expo`, `fastlane`, or direct <NT>Gradle</NT>/<NT>Xcode</NT> commands. You can place it right after `git-clone`.
  1. Leave the inputs at their defaults unless you have a specific reason to change them. The step has two main inputs:

     - **<NT>Enable</NT> <NT>Xcode</NT> cache**: Activates <NT>Bitrise</NT> <NT>Build Cache</NT> for iOS builds. A background proxy is started automatically.
     - **<NT>Enable</NT> <NT>Gradle</NT> cache**: Activates <NT>Bitrise</NT> <NT>Build Cache</NT> for <NT>Android</NT> builds. C++ native modules are also cached via `ccache`, with a background storage helper started automatically.
  1. In **Script** <NT>Steps</NT> that invoke a native build, prefix the command with `bitrise-build-cache react-native run`.

     :::note

     If you use the official <NT>Bitrise Steps</NT> **<NT>Gradle Runner</NT>**, **<NT>Android Build</NT>**, or **<NT>Xcode Archive</NT>** to drive your <NT>React Native</NT> build, make sure you are on the <NT>latest version</NT>: the wrapping is handled automatically and you do not need to change inputs.

     For more information about wrapping, see [Wrapping native build commands](/en/bitrise-build-cache/build-cache-for-react-native/wrapping-native-build-commands).

     :::
  1. Save the <NT>Workflow</NT> and run a build.

  </TabItem>
  <TabItem value="yaml" label="Configuration YAML">

  1. Make sure you have:

     - An active <NT>Bitrise</NT> <NT>Build Cache</NT> trial or subscription. You can check your subscription status on the [Bitrise Build Cache page](https://app.bitrise.io/build-cache/).
     - A <NT>React Native</NT> project that already builds successfully without <NT>Build Cache</NT> (so you have a clear baseline).
     - For iOS projects: **<NT>Xcode 26 or later</NT>**.
  1. Add the `activate-build-cache-for-react-native` <NT>Step</NT> before any step that runs a build:

     ```yaml
     workflows:
       build-react-native:
         steps:
         - activate-ssh-key@4: {}
         - git-clone@8: {}
         - activate-build-cache-for-react-native@0: {}
         # JS dependencies — no wrapping needed.
         - script@1:
             title: Install dependencies
             inputs:
             - content: yarn install
     ```
  1. Wrap commands that trigger native builds with the CLI:

     :::note

     If you use the official <NT>Bitrise Steps</NT> **<NT>Gradle Runner</NT>**, **<NT>Android Build</NT>**, or **<NT>Xcode Archive</NT>** to drive your <NT>React Native</NT> build, make sure you are on the <NT>latest version</NT>: the wrapping is handled automatically and you do not need to change inputs.

     For more information about wrapping, see [Wrapping native build commands](/en/bitrise-build-cache/build-cache-for-react-native/wrapping-native-build-commands).

     :::

     ```yaml
     workflows:
       build-react-native:
         steps:
         - activate-ssh-key@4: {}
         - git-clone@8: {}
         - activate-build-cache-for-react-native@0: {}
         # JS dependencies — no wrapping needed.
         - script@1:
             title: Install dependencies
             inputs:
             - content: yarn install
         - script@1:
             title: Build Android
             inputs:
             - content: bitrise-build-cache react-native run npx react-native run-android --mode=release
         - script@1:
             title: Build iOS
             inputs:
             - content: bitrise-build-cache react-native run npx react-native run-ios --configuration=Release
         - deploy-to-bitrise-io@2: {}
     ```

  </TabItem>
</Tabs>

## Validating the setup

1. Run a build with the new configuration. The activation step should complete successfully.
1. Open the **<NT>Build details</NT>** page on <NT>Bitrise</NT> and check the **<NT>Build Cache</NT>** tab. You should see the wrapped commands listed with their cache stats.
1. The first build will report 0% cache hit rate: the cache is empty at this point. This is expected.
1. Run 1–3 additional builds to warm the cache. Subsequent builds should report a hit rate above 0%.
1. You can monitor cache performance per build and across builds on the [Build Cache list page](https://app.bitrise.io/build-cache/).
