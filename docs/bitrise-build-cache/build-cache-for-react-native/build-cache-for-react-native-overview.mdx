<NT>Bitrise</NT> <NT>Build Cache for React Native</NT> speeds up your CI builds by caching native compilation artifacts across builds. It covers all three native build systems used in a <NT>React Native</NT> project:

- **<NT>Gradle</NT>**: <NT>Android</NT> build outputs (compiled classes, resources, dex files) are cached via <NT>Bitrise</NT>'s remote <NT>build cache</NT>.
- **<NT>Xcode</NT>**: iOS compilation results are cached via <NT>Bitrise</NT>'s remote <NT>build cache</NT> (the same backend used for <NT>Gradle</NT> and Bazel).
- **C++ native modules**: Compiled native bridge code and third-party native modules are cached via `ccache`, with cache entries shared across builds through <NT>Bitrise</NT>'s remote <NT>build cache</NT>.

By caching native compilation outputs, subsequent CI builds can skip recompiling unchanged native code. This is especially effective for <NT>React Native</NT> projects where the native layer changes infrequently compared to the JS layer.
