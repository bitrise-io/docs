If you have started a release on <NT>Bitrise</NT> but decided to abandon it or complete it externally, you can stop managing that in-progress release regardless of the stage you are at on <NT>Bitrise</NT>. When stopped, the release becomes read-only and cannot be restarted.

:::note[Feature access]

Only [Release Managers](/en/release-management/configuring-connected-apps/release-management-roles-and-permissions), [Project Admins](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci), and [workspace owners](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci#owners) have access to this feature.

:::

To stop managing a release:

1. Open your app in <NT>Release Management</NT> and select **<NT>Releases</NT>**.
1. Select a release from the list and click the arrow next to its name.
1. Select **<NT>Configuration</NT>** on the left.
1. Scroll down and click **<NT>Stop managing release</NT>**. This brings up a dialog.

   ![2025-08-08-rm-stop-managing-release.png](/img/_paligo/uuid-187924e0-afdb-8c78-53fe-809168944982.png)
1. Select a reason for stopping this release.

   ![2025-08-08-rm-stop-managing-release-dialog.png](/img/_paligo/uuid-b2fabd61-e0f2-af2c-3074-70940d45d4cc.png)

   - **Completed externally**: The release will be completed outside of <NT>Bitrise</NT>, either on <NT>App Store Connect</NT> or on <NT>Google Play Console</NT>.
   - **Abandoned**: The release is stopped on this stage and will not be completed. In both cases, the release turns to a read-only status on <NT>Bitrise</NT>.

After you confirm the changes, you can see a breakdown of all completed and not completed release stages. If you go back to the **<NT>Releases</NT>** page, you will see the stopped release with the status **Stopped managing** and a short explanation (**Abandoned**, **Completed externally**) you set before.
