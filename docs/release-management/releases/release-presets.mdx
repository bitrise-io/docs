<NT>Release presets</NT> are <NT>default value</NT>s and configurations for new releases of [a connected app](/en/release-management/getting-started-with-release-management/connecting-an-app). To set a preset for a release, you need a preset template. A preset template is a combination of various preset options. You can apply a template to a release when [adding a new release](/en/release-management/releases/adding-a-new-release).

You can configure presets for:

- [The initial release configuration](#release-preset-options).
- [The release candidate stage](/en/release-management/releases/managing-the-release-process/selecting-a-release-candidate).

## Creating a preset template

1. <Partial_OpeningAnAppInReleaseManagement />
1. On the left, select **<NT>Release presets</NT>**.

   ![20251216-rm-presets.png](/img/_paligo/uuid-bf243542-7a52-3b3e-e6d5-26d0248c3951.png)
1. Click **+ <NT>New template</NT>**.
1. Add a name and click **<NT>Create</NT>**.
1. Click the options menu (⋮) next to the name of your template and select **<NT>Change</NT>**.
1. Select and configure the available [release preset options](#release-preset-options):

   - In the **<NT>Configuration</NT>** section, configure preset options for the initial release configuration.
   - In the **<NT>Release candidate</NT>** section, create a build configuration for the <NT>release candidate stage</NT>.
   - **Release rollout/<NT>App Store release settings</NT>** allow you to configure phased release for <NT>Android</NT> and iOS apps, respectively. By default, updates are released immediately to all users.
   - **<NT>Release notes</NT>/<NT>App Store metadata</NT>** allow you to add release notes to your release.

## Release preset options

- **<NT>Auto-upload</NT>**: Enable or disable automatic upload with the toggle switch. Read more: [Configuring auto-upload](/en/release-management/releases/configuring-a-release/configuring-auto-upload).
- **<NT>Approvals</NT>**: Create default approval tasks for new releases. Read more: [Creating tasks for the approvals stage](/en/release-management/releases/managing-the-release-process/creating-tasks-for-the-approvals-stage).
- **<NT>Notifications</NT>**: Enable <NT>Slack</NT> and Teams notifications for all new releases with the same settings. Read more: [Configuring Slack and Teams notifications](/en/release-management/releases/configuring-a-release/configuring-slack-and-teams-notifications).
- **<NT>Automation</NT>**: Configure default automation events with their triggered <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT> or <NT><GlossTerm baseform="pipeline">Pipeline</GlossTerm></NT>. Read more: [Release automation](/en/release-management/releases/configuring-a-release/release-automation).
- **<NT>Outgoing webhooks</NT>**: Select release events that should send a JSON payload to a service of your choice. Read more: [Outgoing webhooks in Release Management](/en/release-management/releases/configuring-a-release/outgoing-webhooks-in-release-management).

Preset options for the <NT>release candidate stage</NT>:

- **<NT>Build</NT> configuration**: You can select the <NT>release branch</NT> for the app and the <NT>Workflow</NT> that generates the build with a signed IPA or AAB file.

Preset options for the review and release stage:

- **Release note**: You can create a release note preset for <NT>Android</NT> apps.
- **<NT>App Store metadata</NT>**: You can create a release note preset for iOS apps.

For details, read [Release note preset](#release-note-preset).

## Release note preset

You can create a release note preset. This means every release will be submitted to the App Store or <NT>Google Play</NT> with the same release notes.

A release note preset can be identical in all localizations.

:::note[Manual edit]

You can edit a release note in the release stage, before submitting your app to an online store. A manual edit will override the release note preset.

:::

### Editing a release note preset

To create and save a release note preset:

<Tabs>
<TabItem value="ios" label="iOS" default>

1. Open your app in <NT>Release Management</NT>.
1. On the left, select **<NT>Release presets</NT>**.

   ![20251216-rm-presets.png](/img/_paligo/uuid-bf243542-7a52-3b3e-e6d5-26d0248c3951.png)
1. Click the options menu (⋮) next to the name of your preset template and select **<NT>Change</NT>**.
1. Find **<NT>App Store metadata</NT>**.
1. From the dropdown menu, select a localization.

   ![2025-08-07-app-store-metadata.png](/img/_paligo/uuid-e5531ad0-351f-de58-e68e-320c039ab434.png)
1. Click **<NT>What's new in this version</NT>** and then **<NT>Edit metadata</NT>**.
1. Add your release note in the **What's in your release** field.
1. Click **<NT>Save changes</NT>**.

</TabItem>
<TabItem value="android" label="Android">

1. Open your app in <NT>Release Management</NT>.
1. On the left, select **<NT>Release presets</NT>**.

   ![20251216-rm-presets.png](/img/_paligo/uuid-bf243542-7a52-3b3e-e6d5-26d0248c3951.png)
1. Click the options menu (⋮) next to the name of your template and select **<NT>Change</NT>**.
1. Find **<NT>Release notes</NT>**.
1. From the dropdown menu, select a localization.

   ![release-notes-menu.png](/img/_paligo/uuid-04f62e3f-a98e-7b0f-4b1f-7d592e42db65.png)
1. Click **<NT>Edit</NT>**. This opens the **<NT>Edit</NT> <NT>release notes</NT>** dialog.
1. Add your release note in the **What's in your release** field.

   Optionally, you can check **Use same content for all localizations** to include the same text for all languages.

   ![release-note-preset.png](/img/_paligo/uuid-588d950a-8b3c-1984-18f6-d251651222a2.png)
1. Click **<NT>Save changes</NT>**.

</TabItem>
</Tabs>

### Copying a release note preset to other localizations

If you don't want to automatically use the same preset for all localizations, you can copy your preset to specific localizations.

<Tabs>
<TabItem value="ios" label="iOS" default>

1. Open your app in <NT>Release Management</NT>.
1. On the left, select **<NT>Release presets</NT>**.

   ![20251216-rm-presets.png](/img/_paligo/uuid-bf243542-7a52-3b3e-e6d5-26d0248c3951.png)
1. Click the options menu (⋮) next to the name of your template and select **<NT>Change</NT>**.
1. Find **<NT>App Store metadata</NT>**.
1. Click into **What's new in this verseion**, and then click **<NT>Copy to</NT>**. This opens the **<NT>Copy</NT> metadata dialog**.

   ![copy-metadat.png](/img/_paligo/uuid-55c4990e-7f00-98a4-fa17-c3a9d0e3ba76.png)
1. Open the **<NT>Copy to</NT>** dropdown menu, and select all localizations you want to copy to.
1. Click **<NT>Copy</NT>**.

</TabItem>
<TabItem value="android" label="Android">

1. Open your app in <NT>Release Management</NT>.
1. On the left, select **<NT>Release presets</NT>**.

   ![20251216-rm-presets.png](/img/_paligo/uuid-bf243542-7a52-3b3e-e6d5-26d0248c3951.png)
1. Click the options menu (⋮) next to the name of your template and select **<NT>Change</NT>**.
1. Find **<NT>Release notes</NT>**.
1. Click **<NT>Copy to</NT>**. This opens the **<NT>Copy</NT> metadata** dialog.

   ![copy-metadat.png](/img/_paligo/uuid-55c4990e-7f00-98a4-fa17-c3a9d0e3ba76.png)
1. Open the **<NT>Copy to</NT>** dropdown menu, and select all localizations you want to copy to.
1. Click **<NT>Copy</NT>**.

</TabItem>
</Tabs>
