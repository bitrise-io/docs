A connected app in <NT>Release Management</NT> uses a [Google Play Console service account](/en/bitrise-platform/integrations/connecting-a-google-service-account-to-bitrise) to connect to <NT>Google Play</NT> and an [App Store Connect account](/en/bitrise-platform/integrations/apple-services-connection/connecting-to-an-apple-service-with-api-key) to connect to the App Store. When [connecting an app for the first time](/en/release-management/getting-started-with-release-management/connecting-an-app), you have to set up the store connections. You can modify these settings at any time.

You can add multiple service accounts or API keys to your <NT><GlossTerm baseform="workspace">workspace</GlossTerm></NT>. Each project and each connected app in <NT>Release Management</NT> can use different service accounts or API keys.

An app in <NT>Release Management</NT> can either use:

- The store connection of its project.
- A store connection specific to the app.

If, for example, you have multiple <NT>App Store Connect API keys</NT> set up in your <NT>Workspace</NT>, you can set one as your project's API key but use a different one for the connected app in <NT>Release Management</NT>.

## Using a project-level store connection

:::important[Changing the connection]

If you use a project-level store connection, changing the project's service account or API key will change the connected app's connection, too. This can break your releases!

:::

<Tabs>
<TabItem value="ios" label="iOS" default>

1. [Set up an API key for your project](/en/bitrise-platform/integrations/apple-services-connection/connecting-to-an-apple-service-with-api-key).
1. On the <NT>Release Management</NT> home page, select <NT>your app</NT> in the **<NT>Connected apps</NT>** list.

   ![2025-08-07-rm-your-apps-list.png](/img/_paligo/uuid-c446a6a9-f922-2641-53ad-49099a7921fe.png)
1. On the left, select **<NT>Integrations</NT>**.
1. On the **<NT>Store</NT>** tab, find the **<NT>API key</NT>** card and click **<NT>Replace</NT>**.

   ![app-store-api-rm.png](/img/_paligo/uuid-e39aa886-e484-fe54-2563-8d5295796112.png)
1. In the dialog, select **Use project's API key**.
1. Click **<NT>Save changes</NT>**.

</TabItem>
<TabItem value="android" label="Android">

1. [Set up a service account for your project](/en/bitrise-platform/integrations/connecting-a-google-service-account-to-bitrise).
1. On the <NT>Release Management</NT> home page, select <NT>your app</NT> in the **<NT>Connected apps</NT>** list.

   ![2025-08-07-rm-your-apps-list.png](/img/_paligo/uuid-c446a6a9-f922-2641-53ad-49099a7921fe.png)
1. On the left, select **<NT>Integrations</NT>**.
1. On the **<NT>Store</NT>** tab, find the **<NT>Service account</NT>** card and click **<NT>Replace</NT>**.

   ![service-account-rm.png](/img/_paligo/uuid-14a7856e-16ca-8a64-04fa-acf92ba93cc2.png)
1. In the dialog, select **Use project's service account**.
1. Click **<NT>Save changes</NT>**.

</TabItem>
</Tabs>

## Using an app-level store connection

An app-level store connection means you set the service account or API key specifically for a given connected app. The app-level connection can be different from the project's configured connection.

<Tabs>
<TabItem value="ios" label="iOS" default>

1. [Set up an API key for your project](/en/bitrise-platform/integrations/apple-services-connection/connecting-to-an-apple-service-with-api-key).
1. On the <NT>Release Management</NT> home page, select <NT>your app</NT> in the **<NT>Connected apps</NT>** list.

   ![2025-08-07-rm-your-apps-list.png](/img/_paligo/uuid-c446a6a9-f922-2641-53ad-49099a7921fe.png)
1. On the left, select **<NT>Integrations</NT>**.
1. On the **<NT>Store</NT>** tab, find the **<NT>API key</NT>** card and click **<NT>Replace</NT>**.

   ![app-store-api-rm.png](/img/_paligo/uuid-e39aa886-e484-fe54-2563-8d5295796112.png)
1. In the dialog, select **Set app-level API key**.
1. <NT>Select an API key</NT> from the **<NT>API key</NT>** dropdown menu.
1. Click **<NT>Save changes</NT>**.

</TabItem>
<TabItem value="android" label="Android">

1. [Set up a service account for your project](/en/bitrise-platform/integrations/connecting-a-google-service-account-to-bitrise).
1. On the <NT>Release Management</NT> home page, select <NT>your app</NT> in the **<NT>Connected apps</NT>** list.

   ![2025-08-07-rm-your-apps-list.png](/img/_paligo/uuid-c446a6a9-f922-2641-53ad-49099a7921fe.png)
1. On the left, select **<NT>Integrations</NT>**.
1. On the **<NT>Store</NT>** tab, find the **<NT>Service account</NT>** card and click **Set <NT>service account</NT>**.
1. In the dialog, select **Set app-level service account**.

   ![service-account-rm.png](/img/_paligo/uuid-14a7856e-16ca-8a64-04fa-acf92ba93cc2.png)
1. Select the <NT>service account</NT> from the **<NT>Service account</NT>** dropdown menu.
1. Click **<NT>Save changes</NT>**.

</TabItem>
</Tabs>
