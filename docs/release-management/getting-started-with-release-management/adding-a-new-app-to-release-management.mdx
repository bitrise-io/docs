To start using <NT>Release Management</NT>, you need to add at least one app. It doesn't require any store connection or a code repository. You will need to link the app to a [Bitrise project](/en/bitrise-platform/projects/projects-overview). If you don't have an existing <NT>Bitrise</NT> <NT>project</NT>, we'll automatically create one for you.

To add an app:

1. <NT>Log in</NT> to <NT>Bitrise</NT> and select **<NT>Releases</NT>** from the left navigation menu.
1. If it's your first app, you will see the **<NT>Get started with Release Management</NT>** page. Start with adding your app.

   If it's not your first app, look for the **<NT>New app</NT>** button above the list of your apps.
1. Set a project for your app.

   - To link your app to an existing <NT>Bitrise</NT> <NT>project</NT>, select the **Existing project** option and choose a project from the dropdown menu.
   - To automatically create a new project, select the **<NT>New project</NT>** option. This project will be a <NT>Release Management</NT> project but you can add a CI configuration to it at any time.

   ![add-new-app-newproj.png](/img/_paligo/uuid-8821de29-e79d-0932-c3dd-f5deea8eba32.png)
1. On the next page, specify:

   - The app's title.
   - A <NT>mobile OS</NT>.
   - A <NT>package name</NT> or bundle ID, depending on the <NT>mobile OS</NT>.

   These are not validated at this point. We recommend using a <NT>package name</NT> or bundle ID that already exists in an online store but you can change it later anyway.

   ![adding-app.png](/img/_paligo/uuid-9073242e-afd9-262e-25cf-395ab1881ea6.png)
1. Click **<NT>Add</NT> app**.

Once an app is added, you can start uploading installable artifacts and use our build distribution feature: [Distributing builds to testers](/en/release-management/build-distribution/distributing-builds-to-testers).

To manage releases, you need to connect the app to an app store: [Connecting an app](/en/release-management/getting-started-with-release-management/connecting-an-app).
