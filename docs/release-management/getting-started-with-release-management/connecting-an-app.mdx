After successfully adding a new app to <NT>Release Management</NT>, connect it to an app in either the App Store or <NT>Google Play</NT> to be able to create releases for the app.

- For iOS, this means an app with a valid bundle ID on <NT>App Store Connect</NT>.
- For <NT>Android</NT>, this means an app with a valid <NT>package name</NT> on <NT>Google Play</NT>.

<Tabs>
<TabItem value="ios" label="iOS" default>

1. Make sure your iOS app is registered on the App Store.

   Later in the connecting process, you will have to enter the bundle ID of the app.
1. Make sure you have a <NT>Bitrise CI</NT> project.

   If you created your project when adding the app to <NT>Release Management</NT>, you can extend it with a CI configuration: [Adding a new project](/en/bitrise-ci/getting-started/adding-a-new-project).
1. Make sure you have at least one <NT>App Store Connect</NT> API key added to your <NT>workspace</NT>: [Connecting to an Apple service with API key](/en/bitrise-platform/integrations/apple-services-connection/connecting-to-an-apple-service-with-api-key).
1. Open your app in <NT>Release Management</NT> and select **<NT>Releases</NT>** in the left navigation.
1. Click **<NT>Connect app</NT>**.

   If you don't have a valid CI configuration, you will see **<NT>Add</NT> <NT>CI configuration</NT>** instead of the **<NT>Connect app</NT>** button.

   ![release-man-connect-app.png](/img/_paligo/uuid-58be0b4a-94fa-0a4b-1bac-91d7f81078a1.png)
1. Set an <NT>App Store Connect</NT> API key. You have two options in the dialog:

   **Use the project's API key**: This means connecting to the App Store with the API key set on the project level. Changing the API key of the project changes the API key for the <NT>Release Management</NT> app, too.

   **Set app-level API key**: You can select any of the API keys added to your <NT>workspace</NT> from the **<NT>API key</NT>** dropdown menu. With this option, it doesn't matter what API key is configured on the project level.
1. Enter the bundle ID and click **Validate**.

   The bundle ID must be an exact match of the bundle ID of an existing app on the App Store. If the validation is successful, the **<NT>Connect</NT>** button will be enabled.
1. Click **<NT>Connect</NT>**.

</TabItem>
<TabItem value="android" label="Android">

1. Make sure your <NT>Android</NT> app is registered on <NT>Google Play</NT>.
1. Make sure you have at least one Google service account added to your <NT>workspace</NT>: [Connecting a Google service account to Bitrise](/en/bitrise-platform/integrations/connecting-a-google-service-account-to-bitrise).
1. Make sure you have a <NT>Bitrise CI</NT> project.
1. Open your app in <NT>Release Management</NT> and select **<NT>Releases</NT>** in the left navigation.
1. Click **<NT>Connect app</NT>**.

   ![release-man-connect-app.png](/img/_paligo/uuid-58be0b4a-94fa-0a4b-1bac-91d7f81078a1.png)
1. Set a service account. You have two options in the dialog:

   **Use the project's service account**: This means connecting to <NT>Google Play</NT> with the service account set on the project level. Changing the service account of the project changes the service account for the <NT>Release Management</NT> app, too.

   **Set app-level service account**: You can select any of the service accounts added to your <NT>workspace</NT> from the **<NT>Service account</NT>** dropdown menu. With this option, it doesn't matter what API key is configured on the project level.
1. Enter the <NT>Google Play</NT> app's <NT>package name</NT> in the field and click **Validate**.

   - If the <NT>package name</NT> is found on <NT>Google Play</NT>, the **<NT>Connect</NT>** button will be enabled.

   - If the validation doesn't find a match, check if you typed the <NT>package name</NT> correctly and make sure your <NT>Google Play</NT> service account is working.
1. Click **<NT>Connect</NT>**.

</TabItem>
</Tabs>
