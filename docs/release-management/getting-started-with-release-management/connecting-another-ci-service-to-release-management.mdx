You can use <NT>Release Management</NT> even if you don't use <NT>Bitrise CI</NT>. Upload your app's binary as a release candidate to access all <NT>Release Management</NT> features.

For now, using <NT>Release Management</NT> still requires a <NT>Bitrise CI</NT> project as it's the only way to set up a [connected app](/en/release-management/getting-started-with-release-management/connecting-an-app). However, you don't have to run <NT>Bitrise CI</NT> builds: once the project is added, you can use <NT>Release Management</NT> without <NT>Bitrise CI</NT>. In the near future, <NT>Release Management</NT> will become a standalone solution.

To connect your CI service to <NT>Release Management</NT>:

1. <NT>Sign up</NT> for <NT>Bitrise</NT> and [add a new Bitrise CI project](/en/bitrise-build-cache/getting-started-with-the-build-cache/getting-started-with-the-build-cache#adding-a-new-connection-to-the-build-cache).
1. [Connect an app](/en/release-management/getting-started-with-release-management/connecting-an-app) in <NT>Release Management</NT>.

   This includes setting up your project's connection to <NT>App Store Connect</NT> and/or <NT>Google Play</NT>.
1. [Use our API to upload an installable artifact](/en/release-management/installable-artifacts).
1. Select this artifact at the [release candidate stage](/en/release-management/releases/managing-the-release-process/selecting-a-release-candidate).
