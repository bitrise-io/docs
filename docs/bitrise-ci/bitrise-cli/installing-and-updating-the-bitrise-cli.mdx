The <NT>Bitrise CLI</NT> is <NT>Bitrise</NT>'s open source runner. The CLI is used to run your builds on [bitrise.io](https://www.bitrise.io) and you can install and run it on your own computer to run builds locally.

The <NT>Bitrise CLI</NT> is distributed as a single binary for macOS and <NT>Linux</NT>. You can download it using curl or Homebrew.


## Installing the <NT>Bitrise CLI</NT> with curl

1. Run the following command in a bash shell:

   ```bash
   curl -fL https://github.com/bitrise-io/bitrise/releases/latest/download/bitrise-$(uname -s)-$(uname -m) > /usr/local/bin/bitrise
   ```

   You can check the latest release of the <NT>Bitrise CLI</NT> on its [releases](https://github.com/bitrise-io/bitrise/releases) page.
1. Make the downloaded binary executable:

   ```bash
   chmod +x /usr/local/bin/bitrise
   ```
1. Run `bitrise setup`.

   This will verify if everything that is required for <NT>Bitrise</NT> to run is installed and available. If you skip this, the CLI will perform the setup anyway the first time you call `bitrise run`.


## Installing the <NT>Bitrise CLI</NT> with Homebrew

If you have the Homebrew package manager installed on your Mac, you can use it to install the <NT>Bitrise CLI</NT>.

1. Open the `Terminal` app on your Mac and run:

   ```bash
   brew update && brew install bitrise
   ```
1. Perform a first-time setup for the <NT>Bitrise CLI</NT>:

   ```bash
   bitrise setup
   ```

   This will verify if everything that is required for <NT>Bitrise</NT> to run is installed and available. If you skip this, the CLI will perform the setup anyway the first time you call `bitrise run`.


## Updating the <NT>Bitrise CLI</NT>

Updating the <NT>Bitrise CLI</NT> is easy, and it doesn’t matter if you installed it with Homebrew or from the <NT>GitHub</NT> release.

Simply run `bitrise update` - that’s it! The CLI checks for updates once every day and notifies you as soon as there is a new version.
