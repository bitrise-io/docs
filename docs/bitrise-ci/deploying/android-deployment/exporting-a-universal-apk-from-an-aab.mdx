You can test an <NT>Android</NT> app on a test device even if the generated artifact is an App Bundle (`.aab`). With the [**Export Universal APK**](https://github.com/bitrise-steplib/bitrise-step-export-universal-apk) <NT><GlossTerm baseform="Step">Step</GlossTerm></NT> you can export a universal APK from the App Bundle, sign it with a keystore (or debug keystore), and deploy the APK to your test device before releasing the app to the <NT>Google Play Store</NT>.

To configure this <NT>Step</NT>:


<Tabs>
  <TabItem value="workflow-editor" label="Workflow Editor">

  1. Insert the [**Export Universal APK**](https://github.com/bitrise-steplib/bitrise-step-export-universal-apk) <NT>Step</NT> after the [**Android Build**](https://github.com/bitrise-steplib/bitrise-step-android-build) <NT>Step</NT> in your <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>.
  1. Make sure the **<NT>Android</NT> <NT>App</NT> Bundle path** input's value is the output variable (BITRISE_AAB_PATH) of the previous build <NT>Step</NT>.

     :::note[Using a different build <NT>Step</NT>]

     If you don't use the [**Android Build**](https://github.com/bitrise-steplib/bitrise-step-android-build) <NT>Step</NT> to build an AAB file, make sure that the input points to the AAB output of the <NT>Step</NT> you used. You can use an [Environment Variable](/en/bitrise-ci/configure-builds/environment-variables), or a direct local path or URL.

     :::
  1. Make sure the **<NT>Keystore URL</NT>** input points to your <NT>Android</NT> keystore file.

     We recommend uploading the file to <NT>Bitrise</NT> and using the default Env Var: $BITRISEIO_ANDROID_KEYSTORE_URL. You can, however, use a local path or a URL as input value here.
  1. Provide your credentials in the **<NT>Keystore alias</NT>** and the **<NT>Keystore password</NT>** inputs.

     If you uploaded a keystore file to <NT>Bitrise</NT>, the <NT>default value</NT> of the inputs should not need to be changed.
  1. In the **Bundletool version** input, you can override the default Bundletool version if you need a specific one but make sure you use the [correct version](https://github.com/google/bundletool/releases).
  1. Run your <NT>Workflow</NT>.

  </TabItem>
  <TabItem value="yaml" label="Configuration YAML">

  1. In your app's <NT><GlossTerm baseform="configuration yaml">Configuration YAML</GlossTerm></NT> file, insert the `bitrise-step-export-universal-apk` <NT>Step</NT> after the `android-build` <NT>Step</NT>.

     ```yaml
     my-workflow:
       steps:
         - android-build: {}
         - bitrise-step-export-universal-apk:
             inputs:
     ```
  1. Make sure the `aab_path` input's value is the output variable ($BITRISE_AAB_PATH) of the previous build <NT>Step</NT>.

     :::note[Using a different build <NT>Step</NT>]

     If you don't use the `android-build` <NT>Step</NT> to build an AAB file, make sure that the input points to the AAB output of the <NT>Step</NT> you used. You can use an Environment Variable, or a direct local path or URL.

     :::

     ```yaml
     my-workflow:
       steps:
         - android-build: {}
         - bitrise-step-export-universal-apk:
             inputs:
             - aab_path: "$BITRISE_AAB_PATH"
     ```
  1. Make sure the `keystore_url` input points to your <NT>Android</NT> keystore file.

     We recommend uploading the file to <NT>Bitrise</NT> and using the default Env Var: $BITRISEIO_ANDROID_KEYSTORE_URL. You can, however, use a local path or a URL as input value here.

     ```yaml
     my-workflow:
       steps:
         - android-build: {}
         - bitrise-step-export-universal-apk:
             inputs:
             - aab_path: "$BITRISE_AAB_PATH"
             - keystore_url: "$BITRISEIO_ANDROID_KEYSTORE_URL"
     ```
  1. Provide your credentials in the `keystore_alias` and the `keystore_password` inputs.

     If you uploaded a keystore file to <NT>Bitrise</NT>, the <NT>default value</NT> of the inputs should not need to be changed. Otherwise store your credentials in a [Secret](/en/bitrise-ci/configure-builds/secrets) and use the <NT>Secrets</NT> as the input values.

     ```yaml
     my-workflow:
       steps:
         - android-build: {}
         - bitrise-step-export-universal-apk:
             inputs:
             - aab_path: "$BITRISE_AAB_PATH"
             - keystore_url: "$BITRISEIO_ANDROID_KEYSTORE_URL"
             - keystore_password: "$BITRISEIO_ANDROID_KEYSTORE_PASSWORD"
             - keystore_alias: "$BITRISEIO_ANDROID_KEYSTORE_ALIAS"
     ```
  1. In the `bundletool_version` input, you can override the default Bundletool version if you need a specific one but make sure you use the [correct version](https://github.com/google/bundletool/releases).

     ```yaml
     my-workflow:
       steps:
         - android-build: {}
         - bitrise-step-export-universal-apk:
             inputs:
             - aab_path: "$BITRISE_AAB_PATH"
             - keystore_url: "$BITRISEIO_ANDROID_KEYSTORE_URL"
             - keystore_password: "$BITRISEIO_ANDROID_KEYSTORE_PASSWORD"
             - keystore_alias: "$BITRISEIO_ANDROID_KEYSTORE_ALIAS"
             - bundletool_version: 1.8.1
     ```
  1. Run your <NT>Workflow</NT>.

  </TabItem>
</Tabs>


The [**Export Universal APK**](https://github.com/bitrise-steplib/bitrise-step-export-universal-apk) <NT>Step</NT> exports the APK to the `$BITRISE_APK_PATH` Environment Variable which the next <NT>Steps</NT> can pick up. If the [**Deploy to Bitrise.io - Build Artifacts, Test Reports, and Pipeline intermediate files**](https://github.com/bitrise-steplib/steps-deploy-to-bitrise-io) <NT>Step</NT> is included in your <NT>Workflow</NT>, [Release Management](/en/release-management) can deploy the APK for you.
