Deploy an app to <NT>Bitrise</NT> to be able to download the IPA file and install it on devices specified in the app’s Development type provisioning profile. This way, your internal testers can easily test the app.

:::note[Clear the cache]

When trying to install an app from the public install page, you should clear the cache: click the link appearing in the **If you synced your settings from your old device, you need to clear the cache and register your new device** line. The link redirects to the **<NT>Profile</NT> settings** page where you can follow the procedure described in our guide.

:::

:::important[Developer certificate and Development profile]

To deploy an iOS app to [bitrise.io](https://www.bitrise.io/), you will always need a Developer type certificate and a Development type provisioning profile. Even if you want to deploy to the App Store, these are still required: they are used to create the .xcodearchive file from the provided code in the process of exporting the IPA file.

:::

1. Generate an IPA file on your own machine at least once.
1. [Upload all necessary code signing files](/en/bitrise-ci/code-signing/ios-code-signing/creating-a-signed-ipa-for-xcode-projects) to <NT>Bitrise</NT>.
1. If you want your internal testers to test the app, [register test devices](/en/bitrise-ci/testing/testing-ios-apps/registering-a-test-device).
1. Make sure the **<NT>Xcode Archive & Export for iOS</NT>** <NT><GlossTerm baseform="Step">Step</GlossTerm></NT> is in your <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>.
1. Set the **<NT>Automatic code signing</NT>** input to the <NT>Apple service connection</NT> you want to use for <NT>code signing</NT>. The available options are:

   - `off`if you don’t do <NT>automatic code signing</NT>.
   - `api-key`if you use [API key authorization](/en/bitrise-platform/integrations/apple-services-connection/about-connecting-to-apple-services).
   - `apple-id`if you use [Apple ID authorization](/en/bitrise-platform/integrations/apple-services-connection/about-connecting-to-apple-services).
1. Set the **<NT>Distribution method</NT>** input of the <NT>Step</NT> to `development`.

   You can use other export methods, too, but if you only deploy to <NT>Bitrise</NT> and want to install your app on the specified devices of internal testers, `development` is sufficient.
1. Make sure the [**Deploy to Bitrise.io - Build Artifacts, Test Reports, and Pipeline intermediate files**](https://github.com/bitrise-steplib/steps-deploy-to-bitrise-io) <NT>Step</NT> is in your <NT>Workflow</NT>.

   By default, the value of the **<NT>Enable</NT> public page for the App?** input is set to `true`. This way, once the build runs, a public install page will be available with a long and random URL which can be shared with others who are not registered on <NT>Bitrise</NT>. This URL is sent to the users in an email. The **<NT>Notify: User Roles</NT>** and the **<NT>Notify: Emails</NT>** inputs determine which users get the email.
1. Start a build.
1. When the build is finished, go to the app’s **<NT>Builds</NT>** page and click the latest build.
1. Click the **<NT>Artifacts</NT>** tab to find your IPA file.

   You can also find the public install URL here. Click **View artifact** next to the IPA file to open its details page, then click **<NT>Manage access</NT>** and make sure the public install link is set to **Enabled** so you’re able to send the link to non-<NT>Bitrise</NT> users.

   To install an app from the public install page, you must use a native Safari browser of the iOS device. You cannot click the installation link if you’re browsing from a third-party app. For more information, check out [Installing an .ipa file on test devices from the Artifacts tab](/en/bitrise-ci/deploying/ios-deployment/installing-an-ipa-file-from-the-public-install-page#installing-an-ipa-file-on-test-devices-from-the-artifacts-tab)

   ![Deploying_Android_apps_to_Bitrise_and_Google_Play.png](/img/_paligo/uuid-43ba1d01-1168-f540-9414-ace775c44ac5.png)

And that’s it! The file can now be installed on all the devices included in the app’s provisioning profile. Remember: the installation link must be accessed from an iOS device’s Safari browser!
