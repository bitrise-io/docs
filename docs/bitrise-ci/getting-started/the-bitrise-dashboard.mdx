Your first stop when logging in to <NT>Bitrise</NT> is the dashboard. It allows you to navigate the <NT>Bitrise</NT> products across your current <NT><GlossTerm baseform="workspace">workspace</GlossTerm></NT>. You can navigate to:

- Your <NT><GlossTerm baseform="Project">project</GlossTerm></NT>s.
- Your [CI builds](/en/bitrise-ci/run-and-analyze-builds/finding-a-specific-build) on your <NT>workspace</NT>'s CI dashboard.
- The [Bitrise Build Cache](/en/bitrise-build-cache).
- [Release Management](/en/release-management).

![dashboard-overview.png](/img/_paligo/uuid-c88d4641-9945-2d84-dd14-ef30fdd68fc4.png)

You can also find [Insights](/en/insights) data for the <NT>workspace</NT>.

## Checking project details

Your projects are listed on the left side of the dashboard. To see more data about them, click the downward arrow.

The details display basic information about all <NT>Bitrise</NT> products associated with the project:

- The last <NT>Bitrise CI</NT> build and its status on the project's <NT>default branch</NT>. The dashboard doesn't show build information from other branches.
- The last time the <NT>Build Cache</NT> was used.
- The number of <NT>connected apps</NT> the project has in <NT>Release Management</NT>.

![project-card-dashboard.png](/img/_paligo/uuid-2c2a82f9-5bea-aa62-cd49-339aa061c265.png)

You can access each of these products from the details section of the project card.

## Adding a new project from the dashboard

On the top right corner of the dashboard, you can find the **<NT>New project</NT>** button. You can use this to [add a new project with a CI configuration](/en/bitrise-ci/getting-started/adding-a-new-project).

:::note[Projects without CI]

You can create projects without a CI configuration: add a new app to <NT>Release Management</NT> and select the option to create a new project.

:::

Any CI project added this way will also be visible on the dashboard. You can also access [Release Management](/en/release-management) to add an app and link it to the project.

## <NT>Workspace</NT> insights

In the **<NT>Insights for workspace</NT>** section, you can see some basic aggregated data for your <NT>workspace</NT>'s projects:

- **<NT>Successful build time</NT>**: The average <NT>Bitrise CI</NT> build time for successful builds across the <NT>workspace</NT>'s projects.
- **<NT>Build failure rate</NT>**: The percentage of failed <NT>Bitrise CI</NT> builds across the <NT>workspace</NT>'s projects.
- **<NT>Cache hit rate</NT> (P50)**: The percentage of data requests that were successfully served by the <NT>Bitrise</NT> <NT>Build Cache</NT>.
- **<NT>Command</NT> error rate**: The percentage of failed commands.

![workspace-insights.png](/img/_paligo/uuid-b15d9482-748d-bb6a-1aaa-c1153ed8392b.png)

Explore [Insights](/en/insights) to see more metrics, providing data-driven visibility.
