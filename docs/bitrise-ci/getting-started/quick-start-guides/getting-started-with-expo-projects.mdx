You can generate <NT>React Native</NT> projects [with the React Native CLI or with the Expo CLI](https://facebook.github.io/react-native/docs/getting-started.html). [Expo](https://docs.expo.dev/versions/latest/) is a toolchain that allows you to quickly get a <NT>React Native</NT> app up and running without having to use native code in <NT>Xcode</NT> or <NT>Android</NT> Studio.

In this guide we discuss how to set up, test, code sign and deploy your <NT>React Native</NT> project built with the [Expo CLI](https://docs.expo.dev/get-started/installation/).


## Adding an <NT>Expo</NT> project to <NT>Bitrise</NT>

Add your project the same way as any other: see [Adding a new project](/en/bitrise-ci/getting-started/adding-a-new-project) for the full walkthrough of the wizard. This section covers what's specific to <NT>Expo</NT>.

### How the scanner detects an <NT>Expo</NT> project

The <NT><GlossTerm baseform="project scanner">project scanner</GlossTerm></NT> first looks for a `package.json` file that lists `react-native` as a dependency. If it also lists `expo` as a dependency, and there's an `app.json`, `app.config.js`, or `app.config.ts` file alongside `package.json`, <NT>Bitrise</NT> registers the project as <NT>Expo</NT>-based rather than plain <NT>React Native</NT>. In this mode, the scanner skips checking for native `ios/` and `android/` directories.

### What you can configure

During the **<NT>CI configuration</NT>** stage, the wizard shows:

- **<NT>Expo</NT> project directory**: the directory containing your `package.json` and <NT>Expo</NT> config file.
- **<NT>Platform</NT>**: which platform to build for EAS Build: all, <NT>Android</NT>, or iOS.

## Deploying your <NT>Expo</NT> project

<NT>Bitrise</NT> supports [Expo Application Services](https://expo.dev/eas) (EAS) for <NT>Expo</NT> projects, and the default deploy <NT>Bitrise</NT> <NT>Workflow</NT> uses the [Run Expo Application Services (EAS) build](https://www.bitrise.io/integrations/steps/run-eas-build) <NT><GlossTerm baseform="step">Step</GlossTerm></NT> to <NT><GlossTerm baseform="trigger">trigger</GlossTerm></NT> a build on EAS.

In case you don’t want to use EAS, you can use [Turtle CLI](https://docs.expo.dev/eas/cli/) for your <NT>Bitrise</NT> <NT>Workflows</NT>. See the [Expo build using Turtle CLI](https://github.com/bitrise-io/workflow-recipes/blob/main/recipes/react-native-expo-build-using-turtle-cli.md) recipe on <NT>GitHub</NT> for details.

:::note[CodePush]

You can deploy updates to your users' devices with <NT>Bitrise</NT> CodePush. CodePush is part of [Release Management](/en/release-management) and it is supported for <NT>React Native</NT> and <NT>Expo</NT> apps.

For more information about CodePush, check out the official guides: [CodePush](/en/release-management/codepush/about-codepush).

:::
