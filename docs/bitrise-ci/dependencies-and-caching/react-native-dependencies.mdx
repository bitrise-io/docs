[npm](https://docs.npmjs.com/) and [Yarn](https://yarnpkg.com/) are both package managers for <NT>Node.js</NT>. They can manage the dependencies of a project, or globally installed <NT>Javascript</NT> tools. For mobile development, they are frequently used as package managers for <NT>React Native</NT> projects. If you have a <NT>React Native</NT> app on <NT>Bitrise</NT>, you can use our dedicated npm or Yarn <NT><GlossTerm baseform="Step">Step</GlossTerm></NT>s to install and cache your dependencies.

You can also install native dependencies by using the dedicated <NT>Steps</NT>: [Installing native dependencies for a React Native app](#installing-native-dependencies-for-a-react-native-app).

## Installing dependencies with npm and Yarn

<Tabs>
<TabItem value="npm" label="npm" default>

1. Make sure your project has [a package.json file](https://docs.npmjs.com/creating-a-package-json-file) defined in it.
1. Add the [**Run npm command**](https://github.com/bitrise-steplib/steps-npm) <NT>Step</NT> to your <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>.
1. Set the **<NT>The npm command with arguments to run</NT>** to `install`.

   You can add additional options to the `install` command. For example, if you need to [link a library with native dependencies to your React Native project](https://docs.flutter.dev/development/packages-and-plugins/developing-packages#dependencies), use the `--save` or `--save-dev` flag. For the available options, see the [npm documentation](https://docs.npmjs.com/cli/v9/commands/npm-install).

   ![run-npm-command.png](/img/_paligo/uuid-676b848d-85db-ff48-883e-6ca489d799d3.png)
1. Optionally, you can set [the npm version](https://www.npmjs.com/package/npm?activeTab=versions) that will run the `npm install` command in the **<NT>Version</NT> of npm to use** input.

</TabItem>
<TabItem value="yarn" label="Yarn">

1. Make sure your project has [a package.json file](https://docs.npmjs.com/creating-a-package-json-file) defined in it.
1. Add the [**Run yarn command**](https://github.com/bitrise-community/steps-yarn) <NT>Step</NT> to your <NT>Workflow</NT>.
1. Set the **Yarn command to run** to `install`.
1. Optionally, add arguments to the yarn command in the **<NT>Arguments for running yarn commands</NT>**.

   You can specify multiple arguments, separated by a space.

</TabItem>
</Tabs>

## Caching npm and Yarn dependencies

With key-based caching, you only need the [**Restore NPM cache**](https://github.com/bitrise-steplib/bitrise-step-restore-npm-cache) and the [**Save NPM cache**](https://github.com/bitrise-steplib/bitrise-step-save-npm-cache) <NT>Steps</NT> to cache your node modules. These <NT>Steps</NT> require no configuration as they automatically set up the <NT>cache keys</NT> needed for your dependencies.

1. Add the [**Restore NPM cache**](https://github.com/bitrise-steplib/bitrise-step-restore-npm-cache) <NT>Step</NT> to the start of your <NT>Workflow</NT>.

   ![restore-npm-cache.png](/img/_paligo/uuid-68fb84b2-a10a-f7b9-f1e3-7ab0accbae3a.png)
1. Add the [**Save NPM cache**](https://github.com/bitrise-steplib/bitrise-step-save-npm-cache) <NT>Step</NT> to the end of your <NT>Workflow</NT>.

## Installing native dependencies for a <NT>React Native</NT> app

You might need to install native dependencies that are declared in the `ios` and `android` folders. For example, <NT>CocoaPods</NT> dependencies defined in a `Podfile`. You can do this quite easily on <NT>Bitrise</NT>, too: once you bundle your <NT>React Native</NT> app to be ready to build the <NT>Android</NT> and iOS binaries, you can simply use the dependency manager <NT>Steps</NT> for <NT>Android</NT> and iOS apps.

:::tip[Demo app]

Check out our <NT>React Native</NT> demo app for a working example. If you add the app to <NT>Bitrise</NT>, you can see that the **deploy-ios-release** <NT>Workflow</NT>, for example, contains both the [**Run npm command**](https://github.com/bitrise-steplib/steps-npm) and the [**Run CocoaPods install**](https://github.com/bitrise-io/steps-cocoapods-install) <NT>Steps</NT>, to install both npm packages and pods: [React Native demo app](https://github.com/bitrise-io/react-native-demo-app).

:::

1. Make sure your <NT>Workflow</NT> installs your npm packages: [Installing dependencies with npm and Yarn](#installing-dependencies-with-npm-and-yarn).

   You can have a separate <NT>Workflow</NT> that is only responsible for installing npm packages, and it's inserted to run before your main <NT>Workflow</NT>: [Chaining Workflows together](/en/bitrise-ci/workflows-and-pipelines/workflows/managing-workflows#chaining-workflows-together). It can be a [utility Workflow](/en/bitrise-ci/workflows-and-pipelines/workflows/managing-workflows#utility-workflows).
1. Add the [**React Native Bundle**](https://github.com/bitrise-steplib/steps-react-native-bundle) <NT>Step</NT> to your <NT>Workflow</NT>.

   It must come after the npm packages are installed.
1. Add the required dependency manager <NT>Steps</NT> or build <NT>Steps</NT> to the <NT>Workflow</NT>. The exact <NT>Steps</NT> you need depend on the project type and the dependency manager you use:

   - [Android dependencies](/en/bitrise-ci/dependencies-and-caching/android-dependencies)
   - [Managing dependencies with SPM](/en/bitrise-ci/dependencies-and-caching/ios-dependencies/managing-dependencies-with-spm)
   - [Managing dependencies with CocoaPods](/en/bitrise-ci/dependencies-and-caching/ios-dependencies/managing-dependencies-with-cocoapods)
   - [Managing dependencies with Carthage](/en/bitrise-ci/dependencies-and-caching/ios-dependencies/managing-dependencies-with-carthage)
