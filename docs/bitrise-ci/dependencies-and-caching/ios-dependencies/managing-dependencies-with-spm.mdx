<NT>Swift</NT> packages are reusable components of <NT>Swift</NT>, <NT>Objective-C</NT>, <NT>Objective-C</NT>++, C, or C++ code that developers can use in their projects. <NT>Xcode</NT> uses the built-in <NT>Swift</NT> Package Manager (SPM) to support creating and publishing <NT>Swift</NT> packages and managing package dependencies.

On <NT>Bitrise</NT>, every <NT>Xcode</NT> <NT><GlossTerm baseform="Step">Step</GlossTerm></NT> can handle <NT>Swift</NT> package dependencies without any extra configuration. For example, if you wish to build an iOS app to deploy it to the App Store, the [**Xcode Archive & Export for iOS**](https://github.com/bitrise-steplib/steps-xcode-archive) <NT>Step</NT> will handle installing <NT>Swift</NT> dependencies to your app.

## Installing <NT>Swift</NT> packages

You can easily install <NT>Swift</NT> packages for your iOS apps during a <NT>Bitrise</NT> build:

1. Make sure you have correctly configured your <NT>Swift</NT> packages in <NT>Xcode</NT>.
1. Add one of our official <NT>Xcode</NT> <NT>Steps</NT> to your <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>. The available <NT>Steps</NT> are:

   - [**Xcode Archive & Export for iOS**](https://github.com/bitrise-steplib/steps-xcode-archive)
   - [**Xcode Test for iOS**](https://github.com/bitrise-steplib/steps-xcode-test)
   - [**Xcode Build for testing for iOS**](https://github.com/bitrise-steplib/steps-xcode-build-for-test)
   - [**Xcode Build for Simulator**](https://github.com/bitrise-steplib/steps-xcode-build-for-simulator)

   Any of these <NT>Steps</NT> can handle the <NT>Swift</NT> package dependencies.

## Caching <NT>Swift</NT> packages

With key-based caching, you only need the [**Restore SPM cache**](https://github.com/bitrise-steplib/bitrise-step-restore-spm-cache) and the [**Save SPM cache**](https://github.com/bitrise-steplib/bitrise-step-save-spm-cache) <NT>Steps</NT> to cache your <NT>Swift</NT> packages. These <NT>Steps</NT> require no configuration as they automatically set up the <NT>cache keys</NT> needed for your dependencies.

1. Add the [**Restore SPM cache**](https://github.com/bitrise-steplib/bitrise-step-restore-spm-cache) <NT>Step</NT> to your <NT>Workflow</NT>

   ![restore_spm-cache.png](/img/_paligo/uuid-e51ea807-fef9-bfc9-8666-2092740c7a52.png)

   It should come before any <NT>Step</NT> that handles SPM packages in any way, such as [**Xcode Archive & Export for iOS**](https://github.com/bitrise-steplib/steps-xcode-archive).
1. Add the [**Save SPM cache**](https://github.com/bitrise-steplib/bitrise-step-save-spm-cache) <NT>Step</NT> to the end of your <NT>Workflow</NT>.
