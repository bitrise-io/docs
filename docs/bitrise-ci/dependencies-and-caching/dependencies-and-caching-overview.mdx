Handling platform and application dependencies is a vital part of development. On <NT>Bitrise</NT>, we offer several dedicated <NT><GlossTerm baseform="Step">Step</GlossTerm></NT>s to make sure you can install your dependencies regardless of project type.

Dependencies, and just about any other build files, can be cached, too. <NT>Bitrise</NT> offers several dedicated <NT>Steps</NT> to make caching as convenient as possible.

## iOS dependencies

Install iOS dependencies via our official <NT>Xcode</NT> <NT>Steps</NT> which offer built-in dependency management, or use our <NT>Steps</NT> for the most commonly used third-party dependency managers: [iOS dependencies](/en/bitrise-ci/dependencies-and-caching/ios-dependencies/managing-dependencies-with-carthage)

- Use the official <NT>Xcode</NT> <NT>Steps</NT> to handle <NT>Swift</NT> Package Manager dependencies without having to worry about configuring anything outside <NT>Xcode</NT>: [Managing dependencies with SPM](/en/bitrise-ci/dependencies-and-caching/ios-dependencies/managing-dependencies-with-spm).
- If you use <NT>CocoaPods</NT>, and have your `Podfile` configured, you can use our [**Run CocoaPods install**](https://github.com/bitrise-io/steps-cocoapods-install) <NT>Step</NT> to install your pods: [Managing dependencies with CocoaPods](/en/bitrise-ci/dependencies-and-caching/ios-dependencies/managing-dependencies-with-cocoapods).
- If you use Carthage in your project, you can use our [**Carthage**](https://github.com/bitrise-steplib/steps-carthage) <NT>Step</NT> to install your dependencies: [Managing dependencies with Carthage](/en/bitrise-ci/dependencies-and-caching/ios-dependencies/managing-dependencies-with-carthage).

## <NT>Android</NT> dependencies

Our official <NT>Android</NT> <NT>Steps</NT> run your <NT>Gradle</NT> tasks for you in a CI environment. As such, if your dependencies are appropriately configured within your project, they will handle installing them for you: [Android dependencies](/en/bitrise-ci/dependencies-and-caching/android-dependencies).

## Flutter dependencies

For Flutter, our [**Flutter Build**](https://github.com/bitrise-steplib/bitrise-step-flutter-build) <NT>Step</NT> can handle both packages and plugins so you don't need to add any additional <NT>Step</NT> to install your dependencies: [Flutter dependencies](/en/bitrise-ci/dependencies-and-caching/flutter-dependencies).

## <NT>React Native</NT> dependencies

For <NT>React Native</NT>, you can install npm packages via either npm or Yarn, as we have official <NT>Steps</NT> for both package managers. You can also install your native dependencies separately: [React Native dependencies](/en/bitrise-ci/dependencies-and-caching/react-native-dependencies)

## Caching

Our caching solution allows you to cache just about any build file you want. You can cache all of your dependencies with barely any configuration as several <NT>Steps</NT> can automatically collect and cache dependency content. For other build files, the main caching <NT>Steps</NT> are fully configurable to fit your needs.

### Key-based caching

Key-based caching relies on using key-value pairs to identify cache archives. The keys can be dynamically generated so you can ensure, for example, that the archive is only updated if certain files change. You have full control over assigning keys to your cache archives: [Key-based caching](/en/bitrise-ci/dependencies-and-caching/key-based-caching/accessing-key-based-cache-archives).

## <NT>Bitrise</NT> <NT>Build Cache</NT>

The <NT>Bitrise</NT> <NT>Build Cache</NT> is a custom, hosted implementation for remote caching of certain build systems. You can use the remote cache both locally and in the <NT>Bitrise CI</NT> environment: in this way your builds will work consistently regardless of the environment and you can save a significant amount of build time.

<NT>Bitrise</NT> supports the <NT>Bitrise</NT> <NT>Build Cache</NT> for the following build systems:

- [Build Cache for Gradle](/en/bitrise-build-cache/build-cache-for-gradle/configuring-the-build-cache-for-gradle-in-the-bitrise-ci-environment)
- [Build Cache for Bazel](/en/bitrise-build-cache/build-cache-for-bazel/configuring-the-build-cache-for-bazel-in-the-bitrise-ci-environment)
- [Build Cache for Xcode](/en/bitrise-build-cache/build-cache-for-xcode/configuring-the-build-cache-for-xcode-in-the-bitrise-ci-environment)
- [Build Cache for React Native](/en/bitrise-build-cache/build-cache-for-react-native/configuring-the-build-cache-for-react-native-in-the-bitrise-ci-environment)
