:::note

Overview on iOS <NT>code signing</NT> asset management

For a comprehensive overview on what <NT>Steps</NT> are available for <NT>code signing</NT> asset management, visit the [iOS code signing page](/en/bitrise-ci/code-signing/ios-code-signing/creating-a-signed-ipa-for-xcode-projects).

:::

You can create multiple IPA files with different <NT>code signing</NT> identities within a single <NT>Bitrise</NT> build.

During the development of your iOS app you will need multiple types of distributions for different purposes, such as internal testing or deployment to the App Store. The good news is that this does not require setting up two separate workflows on [bitrise.io](https://www.bitrise.io).

In this example we'll be setting up a <NT>Workflow</NT> to create two signed IPA files: one with the `development` and one with the `app-store` export method.

:::tip[Advanced configuration options]

The procedure lists the bare minimum requirements to configure these <NT>Steps</NT> to export <NT>Xcode archive</NT>s. Check out the <NT>Step</NT> in the <NT>Workflow Editor</NT> or the <NT>Step</NT>'s `step.yml` in <NT>GitHub</NT> to see all potential configuration options, including but not limited to overriding the <NT>Bitrise</NT>-managed <NT>automatic code signing</NT> options, recompiling from and including bitcode, or using a specific `.plist` file to configure exporting.

:::

<Tabs>
<TabItem value="workflow-editor" label="Workflow Editor" default>

1. Make sure all the [necessary code signing files](/en/bitrise-ci/code-signing/ios-code-signing/creating-a-signed-ipa-for-xcode-projects) are available for your build.
1. Make sure you have the **[Xcode Archive & Export for iOS](https://www.bitrise.io/integrations/steps/xcode-archive)** <NT>Step</NT> in your <NT>Workflow</NT>.
1. In the list of input variables, navigate to **<NT>Distribution method</NT>** and select **development** from the dropdown menu.

   ![multipleexport.png](/img/_paligo/uuid-1e2babf9-a88c-0bd8-653d-9b5c7e8217a5.png)
1. Set the **<NT>Automatic code signing method</NT>** input to the <NT>Apple service connection</NT> you want to use for <NT>code signing</NT>. The available options are:

   - `off` if you don't do <NT>automatic code signing</NT>.
   - `api-key` [if you use API key authorization](/en/bitrise-platform/integrations/apple-services-connection/about-connecting-to-apple-services).
   - `apple-id` [if you use Apple ID authorization](/en/bitrise-platform/integrations/apple-services-connection/about-connecting-to-apple-services).
1. Add the **[Export iOS and tvOS Xcode archive](https://www.bitrise.io/integrations/steps/export-xcarchive)** <NT>Step</NT> to your <NT>Workflow</NT>. This <NT>Step</NT> reuses the archive generated by the **<NT>Xcode Archive & Export for iOS</NT>** <NT>Step</NT> and does a second export from the archive.
1. Set the **<NT>Automatic code signing method</NT>** input to the <NT>Apple service connection</NT> you want to use for <NT>code signing</NT>. The available options are:

   - `off` if you don't do <NT>automatic code signing</NT>.
   - `api-key` <NT>if you use API key authorization</NT>.
   - `apple-id` <NT>if you use Apple ID authorization</NT>.
1. In the list of input variables, navigate to **<NT>Distribution method</NT>** and select **app-store** from the dropdown menu.

   ![app-store-method.png](/img/_paligo/uuid-734ff7ef-99f2-2ad4-e962-6fb90c3f936a.png)

</TabItem>
<TabItem value="configuration-yaml" label="Configuration YAML">

1. Make sure all the [necessary code signing files](/en/bitrise-ci/code-signing/ios-code-signing/creating-a-signed-ipa-for-xcode-projects) are available for your build.
1. Open the Configuration YAML of your app.
1. Make sure you have the `xcode-archive` <NT>Step</NT> in your <NT>Workflow</NT>.

   ```yaml
   my-workflow:
     steps:
     - xcode-archive:
         inputs:
   ```
1. Set the `distribution_method` input to `development`.

   ```yaml
   my-workflow:
     steps:
     - xcode-archive:
         inputs:
         - distribution_method: development
   ```
1. Set the `automatic_code_signing` input to the <NT>Apple service connection</NT> you want to use for <NT>code signing</NT>. The available options are:

   - `off` if you don't do <NT>automatic code signing</NT>.
   - `api-key` [if you use API key authorization](/en/bitrise-platform/integrations/apple-services-connection/about-connecting-to-apple-services).
   - `apple-id` [if you use Apple ID authorization](/en/bitrise-platform/integrations/apple-services-connection/about-connecting-to-apple-services).

   ```yaml
   my-workflow:
     steps:
     - xcode-archive:
         inputs:
         - automatic_code_signing: api-key
         - distribution_method: development
   ```
1. Add the `export-xcarchive` <NT>Step</NT> to your <NT>Workflow</NT>. This <NT>Step</NT> reuses the archive generated by the `xcode-archive` <NT>Step</NT> and does a second export from the archive.

   ```yaml
   my-workflow:
     steps:
     - xcode-archive:
         inputs:
         - automatic_code_signing: api-key
         - distribution_method: development
     - export-xcarchive:
         inputs:
   ```
1. Set the `automatic_code_signing` input to the <NT>Apple service connection</NT> you want to use for <NT>code signing</NT>. The available options are:

   - `off` if you don't do <NT>automatic code signing</NT>.
   - `api-key` <NT>if you use API key authorization</NT>.
   - `apple-id` <NT>if you use Apple ID authorization</NT>.

   ```yaml
   my-workflow:
     steps:
     - xcode-archive:
         inputs:
         - automatic_code_signing: api-key
         - distribution_method: development
     - export-xcarchive:
         inputs:
         - automatic_code_signing: api-key
   ```
1. Set the `distribution_method` input to `app-store`.

   ```yaml
   my-workflow:
     steps:
     - xcode-archive:
         inputs:
         - automatic_code_signing: api-key
         - distribution_method: development
     - export-xcarchive:
         inputs:
         - automatic_code_signing: api-key
         - distribution_method: app-store
   ```

</TabItem>
</Tabs>

And you're done! Feel free to add multiple **Export iOS and tvOS <NT>Xcode archive</NT>** <NT>Steps</NT> to your <NT>Workflows</NT> to create multiple different signed .ipa files if necessary.
