Naturally, <NT>Bitrise</NT> supports iOS applications built with either **<NT>Ionic</NT>** or **<NT>Cordova</NT>**. However, the <NT>code signing</NT> process is slightly different compared to a native <NT>Xcode</NT> project.

<NT>Bitrise</NT> supports both manual and automatic provisioning for <NT>Ionic</NT> and <NT>Cordova</NT> apps as well - and once again, the processes are somewhat different.


## <NT>Ionic</NT>/<NT>Cordova</NT> <NT>code signing</NT> with manual <NT>code signing</NT> asset management

1. Generate the native <NT>Xcode</NT> project locally from your <NT>Ionic</NT> or <NT>Cordova</NT> project by calling `cordova platform add ios` or `ionic cordova platform add ios`.
1. Upload the files to [bitrise.io](https://www.bitrise.io): open the **<NT>Project settings</NT>** page and select **<NT>Code signing</NT>** on the left. Upload a certificate and a provisioning profile.
1. Make sure you have the **<NT>Certificate and profile installer</NT>** <NT><GlossTerm baseform="step">Step</GlossTerm></NT> in your <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT>.
1. Add the **<NT>Generate cordova build configuration</NT>** <NT>Step</NT> to your <NT>Workflow</NT>. It must come after the **<NT>Certificate and profile installer</NT>** <NT>Step</NT>.
1. Fill in the required inputs for the <NT>Step</NT>. Please note that both the **<NT>Code Signing</NT> Identity** and the **Provisioning <NT>Profile</NT>** are required inputs for iOS apps even though they are not marked as such.

   - **<NT>Build</NT> configuration**: you can set it to either `debug` or `release`.
   - **Code Sign Identity**: enter a Developer or a Distribution identity.
   - **Provisioning <NT>Profile</NT>**: enter the appropriate provisioning profile.
   - **Packaging <NT>Type</NT>**: this controls what type of build is generated by <NT>Xcode</NT>. Set the type of <NT>code signing</NT> you need.
1. Add the **<NT>Cordova</NT> archive** or the **<NT>Ionic archive</NT>** <NT>Step</NT> to your <NT>Workflow</NT>.
1. Fill in the required inputs.

   - The **<NT>Platform</NT>** input needs to be set to: `device`.
   - The **<NT>Build command configuration</NT>** input must match the **<NT>Build</NT> configuration** input of the **<NT>Generate cordova build configuration</NT>** <NT>Step</NT>.

   This <NT>Step</NT> must come after the **<NT>Generate cordova build configuration</NT>** <NT>Step</NT> in the <NT>Workflow</NT>.
1. Run your build!


## <NT>Ionic</NT>/<NT>Cordova</NT> <NT>code signing</NT> with <NT>automatic code signing</NT> asset management

1. Make sure your .p12 <NT>signing certificate</NT>s are uploaded to [bitrise.io](https://www.bitrise.io).
1. Add the **<NT>Cordova prepare</NT>** or the **<NT>Ionic prepare</NT>** <NT>Step</NT> to your <NT>Workflow</NT>. These <NT>Steps</NT> call the `platform rm` and `platform add` commands.
1. Add the **[Manage iOS Code Signing](https://www.bitrise.io/integrations/steps/manage-ios-code-signing)** <NT>Step</NT> to your <NT>Workflow</NT>. If you have both the **Certificate and Profile Installer** and the **<NT>Manage iOS Code Signing</NT>** <NT>Steps</NT> in your <NT>Workflow</NT>, your build might encounter unexpected issues.

   The <NT>Step</NT> will export:

   - The project’s development team.
   - The installed codesign identity’s name.
   - The installed provisioning profile.

   :::caution[One <NT>code signing</NT> <NT>Step</NT> only]

   If you have both the **<NT>Certificate and profile installer</NT>** and the **<NT>Manage iOS Code Signing</NT>** <NT>Steps</NT> in your <NT>Workflow</NT>, your build might encounter unexpected issues.

   :::
1. Select the **<NT>Apple service connection method</NT>** (based on the [Apple service you have set up in Bitrise](/en/bitrise-platform/integrations/apple-services-connection/about-connecting-to-apple-services)) and the **<NT>Distribution method</NT>**.
1. Add the **<NT>Generate cordova build configuration</NT>** <NT>Step</NT> to your <NT>Workflow</NT>.
1. Configure the <NT>Step</NT> to use the <NT>code signing</NT> settings exported by the **<NT>Manage iOS Code Signing</NT>** <NT>Step</NT>:

   **Development distribution example**:

   ```yaml
   - generate-cordova-build-configuration:
       inputs:
       - development_team: $BITRISE_DEVELOPER_TEAM 
       - package_type: development 
       - code_sign_identity: iPhone Developer
       - configuration: debug
   ```

   **Production distribution example**:

   ```yaml
   - generate-cordova-build-configuration:
       inputs:
       - development_team: $BITRISE_DEVELOPER_TEAM 
       - package_type: app-store 
       - code_sign_identity: iPhone Developer
       - configuration: release
   ```
1. Add the **<NT>Cordova Archive</NT>** or the **<NT>Ionic</NT> Archive** <NT>Step</NT> to your <NT>Workflow</NT>.
1. Fill in the required inputs.

   - The **<NT>Platform</NT>** input needs to be set to: `device`.
   - The **<NT>Build command configuration</NT>** input must match the **<NT>Build</NT> configuration** input of the **<NT>Generate cordova build configuration</NT>** <NT>Step</NT>.
1. Run your build!
