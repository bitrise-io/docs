To manage your <NT>code signing</NT> files manually, you need to upload your .p12 <NT>signing certificate</NT>s and the app's provisioning profiles to <NT>Bitrise</NT>.

You always need to upload at least a Developer certificate and a Development type provisioning profile! That is necessary for our <NT>Xcode</NT> <NT><GlossTerm baseform="step">Step</GlossTerm></NT>s to be able to test and build your apps properly.

:::tip[Generating <NT>code signing</NT> files]

If you don't have the necessary <NT>code signing</NT> files ready, you can generate new files: [Generating iOS code signing files](/en/bitrise-ci/code-signing/ios-code-signing/generating-ios-code-signing-files)

:::

To upload the <NT>code signing</NT> files to <NT>Bitrise</NT>, first you need to export the files in the .p12 file format. Once you successfully exported the files, you need to upload them to <NT>Bitrise</NT>. You have two options:

- [Via an API call](/en/bitrise-ci/api/managing-ios-code-signing-files#uploading-an-ios-code-signing-file).
- Uploading it manually to <NT>Bitrise</NT>. This guide focuses on this option.

To manually upload your <NT>code signing</NT> files to <NT>Bitrise</NT>:

1. Make sure you have your .p12 certificates and provisioning profiles exported and ready. Check out [Exporting iOS code signing files](/en/bitrise-ci/code-signing/ios-code-signing/exporting-ios-code-signing-files-without-codesigndoc) for more information.
1. <Partial_GettingToTheProjectSettingsPage />
1. On the left, select **<NT>Code signing</NT>** from the menu.
1. Add your files:

   - To add a certificate, click the **<NT>Add .p12 file</NT>** button. In the dialog box, upload the file and, if required, provide the password for the certificate.
   - To add provisioning profiles, click the **<NT>Add</NT> profile(s)** button. In the dialog box, upload the file(s).
1. Make sure you have the [**Certificate and profile installer**](https://github.com/bitrise-steplib/steps-certificate-and-profile-installer) <NT>Step</NT> in your app’s <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>. You can check it on the **<NT>Workflow</NT>** tab of the <NT><GlossTerm baseform="Workflow Editor">Workflow Editor</GlossTerm></NT>.

   Please note that these <NT>Steps</NT> must be **BEFORE** the <NT>Steps</NT> that archive and export your app (for example, [**Xcode Archive & Export for iOS**](https://github.com/bitrise-steplib/steps-xcode-archive)) in your <NT>Workflow</NT>.
