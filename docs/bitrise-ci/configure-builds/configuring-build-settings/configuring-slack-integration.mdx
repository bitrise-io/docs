You can send <NT>Slack</NT> messages during a <NT>Bitrise</NT> build to individual users, groups or channels with the [**Send a Slack message**](https://github.com/bitrise-io/steps-slack-message) <NT><GlossTerm baseform="Step">Step</GlossTerm></NT>. Customize the messages, include attachments, and link buttons that will take the users to the build page.

To use the <NT>Step</NT> to send messages:

1. <NT>Set up</NT> the [workspace Slack integration](/en/bitrise-platform/workspaces/workspace-slack-integration) and [get the integration ID](/en/bitrise-platform/workspaces/workspace-slack-integration).

   The <NT>Step</NT> needs the integration ID.
1. Add the [**Send a Slack message**](https://github.com/bitrise-io/steps-slack-message) <NT>Step</NT> to your <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT>.
1. Make sure that [it is always set to run even if the previous Step failed](/en/bitrise-ci/workflows-and-pipelines/steps/enabling-or-disabling-a-step-conditionally)!

   This is the default setting of the <NT>Step</NT>. If you change it, messages won’t be sent if the build fails.
1. Find the **<NT>Workspace</NT> <NT>Slack</NT> Integration ID** input and add the integration ID.
1. Customize your <NT>Slack</NT> message with the relevant inputs of the <NT>Step</NT>.

   There are several options, including but not limited to:

   - Setting the target channel, group or username: this can be a name or an encoded ID.
   - The text of the message to send.
   - The bot’s username for the message
   - The message’s color
   - File attachment
   - Link buttons attached to the message

   Check out all the inputs in the <NT><GlossTerm baseform="workflow editor">Workflow Editor</GlossTerm></NT> to see all the ways in which you can customize your <NT>Slack</NT> messages.
