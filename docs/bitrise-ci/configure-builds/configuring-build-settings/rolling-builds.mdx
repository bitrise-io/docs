You can cancel running builds or builds on hold with the **<NT>Rolling Builds</NT>** feature. The previous builds of your project can be automatically aborted once a new one is started.

Please note that manual and scheduled builds are also cancelled if you start a new build with the **<NT>Rolling builds</NT>** option enabled.

:::note[When is a build aborted?]

If you <NT><GlossTerm baseform="trigger">trigger</GlossTerm></NT> a build on a branch where a build is already running, the running build is aborted only if it runs the same <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>.

For example, if you trigger a build on the **main** branch of your repository with the **deploy** <NT>Workflow</NT>, it will NOT cancel a <NT>build running on</NT> the same **main** branch with the **primary** <NT>Workflow</NT>.

:::

## Configuring <NT>rolling builds</NT>

1. <Partial_GettingToTheProjectSettingsPage />
1. On the left, select **<NT>Builds</NT>**.
1. Find the **<NT>Build strategy</NT>** section.
1. Select the type(s) of builds you wish to abort when a new build is started.

   ![rolling-builds.png](/img/_paligo/uuid-74153d13-4c04-d223-7389-268265192245.png)

   - **<NT>Abort on-hold builds triggered by pull requests</NT>**: <NT>Cancel</NT> all previous builds still **on-hold** for <NT>Pull Requests</NT> and all related Pushes. **<NT>Running</NT>** builds will **not** be canceled unless **<NT>Running builds</NT> are aborted** is also enabled.
   - **<NT>Abort on-hold builds triggered by pushes</NT>**: <NT>Cancel</NT> all previous builds still **on-hold** for Pushes to the same branch. **<NT>Running</NT>** builds will **not** be canceled unless **<NT>Running builds</NT> are aborted** is also enabled. You can configure exclusions for push events: [Managing exclusions for push events](#managing-exclusions-for-push-events)
   - **<NT>Abort on-hold builds triggered by tags</NT>**: <NT>Cancel</NT> all previous builds still **on-hold** if they were triggered by Git tags.
   - **<NT>Abort running builds</NT>**: Auto-cancel running builds in addition to on-hold ones.

## Managing exclusions for push events

You can configure <NT>Bitrise</NT> to never abort certain code push builds. To do so, add exclusions when configuring <NT>rolling builds</NT>. You can exclude:

- Repository branches.
- <NT><GlossTerm baseform="pipeline">Pipeline</GlossTerm></NT>s and <NT>Workflows</NT>.

A build that involves an excluded branch, <NT>Pipeline</NT> or <NT>Workflow</NT> will never be aborted.

1. [Go to the Build strategy section to configure rolling builds](#configuring-rolling-builds).
1. Enable **<NT>Abort on-hold builds triggered by pushes</NT>**.
1. Click **<NT>Add</NT> exclusions** (or **<NT>Manage</NT> exclusions** if you have existing exclusions).
1. Enter <NT>excluded branches</NT> in the **<NT>Excluded branches</NT>** field, and excluded <NT>Pipelines</NT> or <NT>Workflows</NT> in the **<NT>Excluded pipelines and workflows</NT>** field. Both fields accept a comma-separated list of values.
