[The build stack](/en/bitrise-build-hub/infrastructure/build-stacks/about-build-stacks) indicates the virtual machine version that we will use to run your build. You can set the <NT><GlossTerm baseform="Stack">stack</GlossTerm></NT> for all your builds, or you can set <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>-specific stacks.

:::tip[Changing machine types using the API]

You can also change <NT>build machine</NT> types for all apps owned by a single user or <NT><GlossTerm baseform="workspace">Workspace</GlossTerm></NT> using the API: [Changing machine types in all apps at the same time](/en/bitrise-ci/api/adding-and-managing-apps#changing-machine-types-in-all-apps-at-the-same-time).

:::


## Setting the stack in the <NT>Workflow Editor</NT>

The simplest way to configure the [build stack](/en/bitrise-build-hub/infrastructure/build-stacks/about-build-stacks) is in the <NT><GlossTerm baseform="Workflow Editor">Workflow Editor</GlossTerm></NT>. You can set both the default stack and <NT>Workflow</NT>-specific stacks.

### Setting the default stack

The default stack of your project is the stack that is used if you haven't set a specific stack for a <NT>Workflow</NT>.

:::note[Project type]

Note that if you change the project type on the app's **<NT>Settings</NT>** tab to a type that isn't compatible with your selected default stack, we'll automatically change the stack to a compatible one.

:::

1. <Partial_OpeningTheWorkflowEditor />
1. On the left, select **<NT>Stacks & Machines</NT>**.
1. In the **<NT>Default stack & machine</NT>** section, open the dropdown menu and select the stack you need.
1. Click **<NT>Save changes</NT>** in the top right corner.

### Setting a <NT>Workflow</NT>-specific stack

When you set a stack for a <NT>Workflow</NT>, that <NT>Workflow</NT> will always run on that stack, regardless of the default stack.

1. <Partial_OpeningTheWorkflowEditor />
1. On the left, select **<NT>Stacks & Machines</NT>**.
1. Select the **<NT>Workflows</NT>** tab.

   ![Workflows tab on the Stacks & Machines page](/img/configure-builds/2026-07-08-stacks-workflows-tab.png)
1. Find the <NT>Workflow</NT> you need and select a stack.
1. Click **<NT>Save changes</NT>** in the top right corner.

### Setting the machine type in the <NT>Workflow Editor</NT>

You can define the machine type you want to use for your build in the <NT>Workflow Editor</NT>. The available machine types depend on the type of stack you use: [Build machine types](/en/bitrise-build-hub/infrastructure/build-machine-types).

To set the machine type:

1. <Partial_OpeningTheWorkflowEditor />
1. On the left, select **<NT>Stacks & Machines</NT>**.
1. In the **<NT>Default stack & machine</NT>** section, choose a machine type from the machine type selector.

   ![Machine type selector in the Default stack & machine section](/img/configure-builds/2026-07-08-stacks-default-machine-type.png)
1. Optionally, choose a <NT>Workflow</NT>-specific machine type for any <NT>Workflow</NT> that has a <NT>Workflow</NT>-specific stack set.


## Setting the stack in the Configuration YAML

You can set both the default stack and <NT>Workflow</NT>-specific tasks in your project's Configuration YAML. Use the `meta` field with the appropriate <NT>stack ID</NT>:

```yaml
meta:      
  bitrise.io:       
    stack: <stack-id>
```

This is particularly useful if you [keep your `bitrise.yml` file stored in your app's repository](/en/bitrise-ci/configure-builds/configuration-yaml/storing-an-apps-configuration-yaml#storing-the-bitriseyml-file-in-your-repository).

:::note[<NT>Stack ID</NT>s]

You can find the <NT>stack ID</NT>s on the top of each stack report: [Bitrise stack reports](https://stacks.bitrise.io/stack_reports/). Alternatively, you can set a stack in the <NT>Workflow Editor</NT> and then check out the ID on the **<NT><GlossTerm baseform="bitrise.yml">bitrise.yml</GlossTerm></NT>** tab, by finding the `meta` field.

The stack reports also show every [pre-installed tool](/en/bitrise-platform/infrastructure/build-stacks/preinstalled-tools-on-bitrise-stacks) that is available on the stack.

:::

### Setting the default stack in the Configuration YAML

The default stack of your project is the stack that is used if you haven't set a specific stack for a <NT>Workflow</NT>.

1. Open your project's Configuration YAML.
1. Add a `meta` entry outside the `workflows` property or at the end of your Configuration YAML:

   ```yaml
   meta:      
     bitrise.io:       
       stack:
   ```

   Make sure you use double indentation.
1. Add the <NT>stack ID</NT>.

   You can find the <NT>stack ID</NT>s on the [stack reports page](https://stacks.bitrise.io/stack_reports/): the filenames without the `.log` extension are the <NT>stack ID</NT>s.

   ```yaml
   meta:      
     bitrise.io:       
       stack: <stack-id>
   ```

### Setting <NT>Workflow</NT>-specific stacks in the Configuration YAML

When you set a stack for a <NT>Workflow</NT>, that <NT>Workflow</NT> will always run on that stack, regardless of the default stack.

1. Open your project's Configuration YAML.
1. Find the `workflows` property and find the name of the <NT>Workflow</NT> you need within.
1. Add a `meta` entry under the <NT>Workflow</NT> itself:

   ```yaml
   workflows:
     deploy:  
       meta:      
         bitrise.io:       
           stack:
   ```
1. Add the <NT>stack ID</NT>.

   You can find the <NT>stack ID</NT>s on the top of each [stack report page](https://stacks.bitrise.io/stack_reports/).

   ```yaml
   workflows:
     deploy:  
       meta:      
         bitrise.io:       
           stack: <stack-id>
   ```

### Setting the machine type in the Configuration YAML

You can also define the machine type you want to use for your build in your project's Configuration YAML. Machine type refers to the hardware resources used in your build; for example, an Elite machine has more CPU cores and available RAM than a Standard machine.

To set the machine type in Configuration YAML file, you need to use the `machine_type_id` attribute in a `meta` entry:

:::note[Available machine IDs]

You can find the available machine type IDs here: [Build machine types](/en/bitrise-build-hub/infrastructure/build-machine-types).

:::

1. Open your project's Configuration YAML file.
1. Find the `workflows` property and find the name of the <NT>Workflow</NT> you need within.
1. Add a `meta` entry:

   ```yaml
   # setting an 8-core Gen2 machine for an Xcode stack as the default machine type of the app:
   meta:
     bitrise.io:
       machine_type_id: g2.8core
   # setting a 12-core Gen2 machine as a Workflow-specific machine type
   workflows:
     deploy:  
       meta:      
         bitrise.io:       
           machine_type_id: <machine-id>
   ```
