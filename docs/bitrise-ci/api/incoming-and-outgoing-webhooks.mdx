Both incoming and outgoing webhooks can be set up with the Bitrise API. They are important for automatic build <GlossTerm baseform="trigger">triggering</GlossTerm> and the reporting of build events to other services.

| Endpoints | Function | Required role on the app's team |
| --- | --- | --- |
| POST /apps/\{app-slug\}/register-webhook | Register an incoming webhook for a specific application. | Owner or Admin |
| GET /apps/\{app-slug\}/outgoing-webhooks | List the outgoing webhooks of an app. | Owner or Admin |
| POST /apps/\{app-slug\}/outgoing-webhooks | Create an outgoing webhook for an app. | Owner or Admin |
| PUT /apps/\{app-slug\}/outgoing-webhooks/\{app-webhook-slug\} | Update an outgoing webhook of an app. | Owner or Admin |
| DELETE /apps/\{app-slug\}/outgoing-webhooks/\{app-webhook-slug\} | Delete an outgoing webhook of an app. | Owner or Admin |
