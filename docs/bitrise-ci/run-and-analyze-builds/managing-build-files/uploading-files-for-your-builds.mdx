If your build requires any files to make it work, you can upload them to <NT>Bitrise</NT> on the **<NT>Project settings</NT>** page. It accepts any file type, all you need to do is provide a <NT>unique ID</NT> and upload the file.

Once a file is uploaded, it is stored [as an Environment Variable (Env Var)](/en/bitrise-ci/configure-builds/environment-variables). You can use this Env Var to access the file and use it in your builds. The file can also be:

- Downloaded by anyone who has either [admin or owner role](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci) on the app’s team on <NT>Bitrise</NT>. You can prevent this: [Protecting your uploaded files](#protecting-your-uploaded-files)
- Exposed to pull request builds.

:::important[File restrictions]

There are certain restrictions on upload size and number of files:

- You can't upload a file bigger than 5 MB.
- You can only store a total of 100 different files at the same time. If you want to upload more, you need to delete one of the files in the storage.

:::

To upload a file:

1. <Partial_GettingToTheProjectSettingsPage />
1. On the left, select **Files**.
1. Click **<NT>Add file</NT>**.
1. In the dialog box, enter a <NT>unique ID</NT> in the **<NT>File Storage ID</NT>** input field.

   The <NT>unique ID</NT> will be part of the generated download URL that <NT>Bitrise</NT> stores as an [Environment Variable](/en/bitrise-ci/configure-builds/environment-variables).

   ![Add file dialog on the Project settings Files page](/img/run-and-analyze-builds/2026-07-14-add-file-dialog.png)
1. Upload the file. You have two options:

   - Click anywhere in the **<NT>Select</NT> a file to upload** section and select the file from your computer.
   - Drag and drop a file into the **<NT>Select</NT> a file to upload** section.

   Remember that the file size cannot exceed 5 MB.


## Protecting your uploaded files

Once you uploaded a file to <NT>Bitrise</NT>, you can set your uploaded files to Protected mode. This means that no one can download or reveal the file from your account but your builds can still use them. <NT>Bitrise</NT> will handle the Environment Variable attached to your uploaded files as a [Secret](/en/bitrise-ci/configure-builds/secrets).

1. Open your project on <NT>Bitrise</NT> with a user that has the **Admin** [role on the project](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci).
1. On the main page of the project, click on the **<NT>Project settings</NT>** button.

   ![project-settings-button.png](/img/_paligo/uuid-14defaa4-472c-2d09-84df-145dc3aef4f5.png)
1. On the left, select **Files**.
1. Click the horizontal ellipsis button next to the name of the file, and select **<NT>Make protected</NT>**.
1. Click **Make it protected** in the dialog box.
