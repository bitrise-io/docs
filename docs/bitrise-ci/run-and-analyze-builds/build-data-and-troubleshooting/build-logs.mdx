Build logs allow users to analyze their builds and find out what went wrong - and what went right! On <NT>Bitrise</NT>, build logs are easily accessible: they can be viewed on the website in their entirety or they can be downloaded to view them on your own device.

<Partial_NOTEBuildRetentionFor200Days />


## Finding a build log

When you run a build, a build log is generated automatically. Every build has its own log, with its own unique build log URL. The build log URL contains the build slug: a hexadecimal identifier for a specific build.

:::note[Assisting <NT>Bitrise</NT> Support]

When <NT>Bitrise</NT> Support asks for your build logs, the best thing to do is send the build URL.

:::

1. Open the [Bitrise CI](http://app.bitrise.io/dashboard) page and select your project from the project list.
1. Select the build you want to check out.
1. Make sure you have the **<NT>Log</NT>** tab selected.
1. On the **<NT>Log</NT>** tab, you can see the <NT><GlossTerm baseform="step">Step</GlossTerm></NT>s of the <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT> and their status. By default, all failed <NT>Steps</NT> are expanded.

   ![Build log tab showing Steps and their status](/img/run-and-analyze-builds/2026-07-14-build-log-step-list.png)
1. Expand any <NT>Step</NT>'s info by clicking the downward arrow on the right to the <NT>Step</NT>. This will show the relevant part of the build log.


## The <NT>build log</NT> page

The <NT>build log</NT> page shows all <NT>Steps</NT> that were part of the build, and their status. By default, failed <NT>Steps</NT> are expanded, with the error message highlighted for convenient troubleshooting. Expanding a <NT>Step</NT> also allows you to see:

:::note

The **<NT>Duration</NT>** field in the header of the log might not be equal to the sum of the duration of all <NT>Steps</NT>. There are two possible reasons for such a discrepancy:

- The duration value includes one-off initialization time for the <NT>Workflow</NT>/<NT><GlossTerm baseform="pipeline">Pipeline</GlossTerm></NT> as well as the sum of <NT>Step</NT> duration.
- When using [remote access](/en/bitrise-ci/run-and-analyze-builds/build-data-and-troubleshooting/remote-access), the VM stays open a full 10 minutes after the steps are all complete to allow review/download of logs

:::

- The start time of the <NT>Step</NT>'s run.
- The <NT>Step ID</NT>. This is how the `bitrise.yml` configuration file identifies the <NT>Step</NT>. For more information, see [Step reference/ID format](/en/bitrise-ci/references/steps-reference/step-reference-id-format).
- The current version of the <NT>Step</NT>. Underneath the version number, you can check the release notes for the <NT>Step</NT>. If the build didn't run with the <NT>latest version</NT> of the <NT>Step</NT>, you can also click **<NT>Update</NT>** to open the <NT><GlossTerm baseform="workflow editor">Workflow Editor</GlossTerm></NT> and update the <NT>Step</NT> to a different version.
- Under **<NT>Step resources</NT>**, you can check out the <NT>Step</NT>'s documentation on our [Integrations](http://bitrise.io/integrations) page, as well as its source code and issue tracker on <NT>GitHub</NT>.

To view a build log's contents in full, in a single file, [download the log](/en/bitrise-ci/run-and-analyze-builds/build-data-and-troubleshooting/build-logs#downloading-a-build-log).


## Following a build log live

You can check the log of a build live - that is, while the build is running. What’s more, you can even follow the log as the build is happening. That means that as the build progresses, the log will automatically scroll to the new sections as they appear.

To do so:

1. Start a build.
1. Go to the build’s page.
1. Scroll down to the log.
1. Click **<NT>Follow</NT>**.

   ![Follow button on a running build's log](/img/run-and-analyze-builds/2026-07-14-build-log-follow-button.png)

To stop following, you just need to manually navigate anywhere within the log. You can restart following any time.


## Downloading a build log

If you need to send your build logs to people who do not have access to the app on <NT>Bitrise</NT>, or you want to store your logs in your own archives, you can simply download the log file from <NT>Bitrise</NT>.

:::important[Log security]

Please note that your build log can contain sensitive information! Make sure to check its contents before downloading the log file and sending it out to anyone. We recommend using [Secrets](/en/bitrise-ci/configure-builds/environment-variables) to make sure nothing sensitive appears in build logs.

:::

1. Open the [Bitrise CI](http://app.bitrise.io/dashboard) page and select your project from the project list.
1. Select the build you want to check out.
1. Open the **<NT>Logs</NT>** dropdown menu and click the **<NT>Download logs</NT>** button.


## Deleting a build log

If necessary, you can delete the logs of any build on <NT>Bitrise</NT>. It can be handy if, for example, you do not want new team members to see potentially sensitive information that is displayed in previous logs.

Not all team members are authorized to delete logs: only those with either **Admin** or **<NT>Owner</NT>** role in the team can do it.

:::warning[Deletion is final]

Be aware that you cannot undo deleting a log. Once you delete it, there is no way to recover the log file.

:::

1. Open the [Bitrise CI](http://app.bitrise.io/dashboard) page and select your project from the project list.
1. Select the build you want to check out.
1. Open the **<NT>Logs</NT>** dropdown menu and click the **<NT>Delete Logs</NT>** button.

   ![Logs dropdown menu with Download logs and Delete logs options](/img/run-and-analyze-builds/2026-07-14-logs-dropdown-delete-logs.png)
1. In the pop-up window, click **<NT>Delete logs</NT>**.
