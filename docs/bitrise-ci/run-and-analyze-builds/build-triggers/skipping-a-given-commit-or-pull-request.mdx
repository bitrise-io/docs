Depending on your settings, every code change in your repository can <GlossTerm baseform="Trigger">trigger</GlossTerm> Bitrise builds. However, if you need to, you can skip a specific commit or pull request. Skipping means, in this context, that a code change will NOT trigger a build on Bitrise, even if the triggers are set up to do so.
