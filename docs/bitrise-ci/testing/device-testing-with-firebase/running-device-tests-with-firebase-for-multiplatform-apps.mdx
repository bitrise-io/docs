You can run native device tests for Flutter and <NT>React Native</NT> apps using <NT>Bitrise</NT>'s device testing solution.

## Testing the iOS app

1. Add tests using the [XCTest](https://developer.apple.com/documentation/xctest) framework to the iOS app of the project.
1. [Build your app for testing](/en/bitrise-ci/testing/testing-ios-apps/building-an-ios-app-for-testing) with the [**Xcode Build for testing for iOS**](https://github.com/bitrise-steplib/steps-xcode-build-for-test) <NT><GlossTerm baseform="Step">Step</GlossTerm></NT>.

   Make sure that the **<NT>Project path</NT>** input of the <NT>Step</NT> points to the `.xcodeproj` or `.xcworkspace` file of your project. You can find these in the `iOS` folder. If you used automatic configuration to add your <NT>React Native</NT> app to <NT>Bitrise</NT>, you don't need to modify the <NT>default value</NT> of the input ($BITRISE_PROJECT_PATH.
1. Run your device tests with the [**iOS Device Testing**](https://github.com/bitrise-steplib/steps-virtual-device-testing-for-ios) <NT>Step</NT>.

   For detailed information on configuring an iOS app for device testing, see [Running device tests for iOS apps](/en/bitrise-ci/testing/device-testing-with-firebase/device-testing-for-ios#running-device-tests-with-firebase-for-ios-apps).

## Testing the <NT>Android</NT> app

You can run robo-, instrumentation- and gameloop tests using the [**Virtual Device Testing for Android**](https://github.com/bitrise-steplib/steps-virtual-device-testing-for-android) <NT>Step</NT>. However, one instance of the <NT>Step</NT> can only run one type of test. If you need to run multiple test types, you need multiple instances of the <NT>Step</NT>.

1. Add tests to the <NT>Android</NT> app of the project.
1. Build the app for UI testing with the [**Android Build for UI Testing**](https://github.com/bitrise-steplib/bitrise-step-android-build-for-ui-testing) <NT>Step</NT>.

   Make sure the **<NT>Project Location</NT>** input of the <NT>Step</NT> points to the root directory of the <NT>Android</NT> app. For example, `./android`.
1. Run your device tests with the [**Virtual Device Testing for Android**](https://github.com/bitrise-steplib/steps-virtual-device-testing-for-android) <NT>Step</NT>.

   For detailed information on configuring an <NT>Android</NT> app for device testing, see [Device testing for Android](/en/bitrise-ci/testing/device-testing-with-firebase/device-testing-for-android).
