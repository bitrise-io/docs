Enhanced <NT>Xcode</NT> reporting allows you to immediately view your <NT>Xcode</NT> test results in a rich HTML format. It will show both successful and failed test cases, as well as any videos and screenshots generated by the test.

To use it, your build needs to do two things:

- Run your <NT>Xcode</NT> tests and place the `.xcresult` file in the correct location. The **<NT>Xcode Test for iOS</NT>** and the **<NT>Xcode</NT> Test for Building** <NT>Steps</NT> do this by default and require no further configuration. You can also use other <NT>Steps</NT>, for example, **fastlane**, but you need to specify where <NT>Xcode</NT> should place the `.xcresult` file.
- Run the **<NT>Generate Xcode HTML report</NT>** <NT>Step</NT>: it takes the `.xcresult` file and generates the report. If you use the **<NT>Xcode Test for iOS</NT>** <NT>Step</NT> to run your tests, this <NT>Step</NT> requires no configuration. When using other <NT>Steps</NT>, you have to manually set the location of your `.xcresult` file.

We strongly recommend using the official **<NT>Xcode Test for iOS</NT>** <NT>Step</NT> to run your tests: this <NT>Step</NT> generates an `.xcresult` file and the **<NT>Generate Xcode HTML report</NT>** <NT>Step</NT> finds it without any additional configuration.

To view the rich test results:

<Tabs>
<TabItem value="workflow-editor" label="Workflow Editor" default>

1. Add a <NT>Step</NT> that runs your <NT>Xcode</NT> tests to your <NT>Workflow</NT>.

   :::note[We recommend using the <NT>Xcode Test for iOS</NT> <NT>Step</NT>]

   Using our **<NT>Xcode Test for iOS</NT>** <NT>Step</NT> (as well as the **<NT>Xcode Test without building</NT>** <NT>Step</NT>) means you don't need any additional configuration to export your results.

   You can use other <NT>Steps</NT> to run your tests and then view the rich results afterwards. In that case, you have to make sure that the <NT>Step</NT> generates an `.xcresult` file and point the **<NT>Generate Xcode HTML report</NT>** <NT>Step</NT> to its location. For example, you can use the **fastlane** <NT>Step</NT> or your own **Script** <NT>Step</NT>.

   :::

   For more information on configuring the <NT>Step</NT> to run your <NT>Xcode</NT> tests, see [Running unit and UI tests for iOS apps](/en/bitrise-ci/testing/testing-ios-apps/running-unit-and-ui-tests-for-ios-apps).
1. Add the **<NT>Generate Xcode HTML report</NT>** <NT>Step</NT> to your <NT>Workflow</NT>.

   It should, of course, come after the <NT>Step</NT> that runs your tests.
1. Make sure the **Xcresult file paths** input points to the location of your `.xcresult` files.

   You can add multiple `.xcresult` files to this input, either by specifying one or more directories or by specifying filepaths, separated by a newline (/n). Each file will have its own report.

   :::important[No configuration required]

   If you use the **<NT>Xcode Test for iOS</NT>** <NT>Step</NT>, you don't have to modify this input!

   :::

   ![generate-xcode-reports_input.png](/img/_paligo/uuid-a928eed7-d498-758b-6a6c-5a8ef7b202d1.png)
1. Make sure the <NT>Workflow</NT> includes the **<NT>Deploy to Bitrise.io</NT>** <NT>Step</NT> at the end.

   Without this <NT>Step</NT>, the results won't be exported.

   :::important[Version requirement]

   The <NT>Step</NT> must be version 2.7.2 or newer.

   :::
1. After the build is finished, go to the <NT>build summary</NT> page.

   :::note[Failed builds]

   The test results will be available even if the build fails.

   :::
1. Select the **<NT>Tests</NT>** tab to view the results.

   :::important[Expiry date]

   The HTML test reports are available for seven days after they are generated. After seven days, you can no longer access them.

   :::

   ![html-test-result.png](/img/_paligo/uuid-bf85c17c-277e-088a-13b0-3fe96e79cc7d.png)

</TabItem>
<TabItem value="configuration-yaml" label="Configuration YAML">

1. Add a <NT>Step</NT> that runs your <NT>Xcode</NT> tests to your <NT>Workflow</NT>.

   We strongly recommend using the `xcode-test` <NT>Step</NT> for this.

   ```yaml
   my-workflow:
     steps:
     - xcode-test: {}
   ```

   :::note[We recommend using the xcode-test <NT>Step</NT>]

   Using our `xcode-test` <NT>Step</NT> (as well as the `xcode-test-without-building` <NT>Step</NT>) means you don't need any additional configuration to export your results.

   You can use other <NT>Steps</NT> to run your tests and then view the rich results afterwards. In that case, you have to make sure that the <NT>Step</NT> generates an `.xcresult` file and point the `generate-xcode-html-report` <NT>Step</NT> to its location. For example, you can use the `fastlane` <NT>Step</NT> or your own `script` <NT>Step</NT>.

   :::

   For more information on configuring the <NT>Step</NT> to run your <NT>Xcode</NT> tests, see [Running unit and UI tests for iOS apps](/en/bitrise-ci/testing/testing-ios-apps/running-unit-and-ui-tests-for-ios-apps).
1. Add the `generate-xcode-html-report` <NT>Step</NT> to your <NT>Workflow</NT>.

   It should, of course, come after the <NT>Step</NT> that runs your tests.

   ```yaml
   my-workflow:
     steps:
     - xcode-test: {}
     - generate-xcode-html-report: {}
   ```
1. Make sure the `xcresult_patterns` input points to the location of your `.xcresult` files.

   You can add multiple `.xcresult` files to this input, either by specifying one or more directories or by specifying filepaths, separated by a newline (/n). Each file will have its own report.

   ```yaml
   my-workflow:
     steps:
     - xcode-test: {}
     - generate-xcode-html-report:
         is_always_run: true
         inputs: 
         - xcresult_patterns: /path/to/MyApp.xcresult
   ```

   :::important[No configuration required]

   If you use the **<NT>Xcode Test for iOS</NT>** <NT>Step</NT>, you don't have to modify this input!

   :::
1. Make sure the <NT>Workflow</NT> includes the `deploy-to-bitrise-io` <NT>Step</NT> at the end.

   Without this <NT>Step</NT>, the results won't be exported.

   :::important[Version requirement]

   The <NT>Step</NT> must be version 2.7.2 or newer.

   :::

   ```yaml
   my-workflow:
     steps:
     - xcode-test: {}
     - generate-xcode-html-report:
         is_always_run: true
         inputs: 
         - xcresult_patterns: /path/to/MyApp.xcresult
     - deploy-to-bitrise-io: {}
   ```
1. After the build is finished, go to the <NT>build summary</NT> page.

   :::note[Failed builds]

   The test results will be available even if the build fails.

   :::
1. Select the **<NT>Tests</NT>** tab to view the results.

   :::important[Expiry date]

   The HTML test reports are available for seven days after they are generated. After seven days, you can no longer access them.

   :::

   ![html-test-result.png](/img/_paligo/uuid-bf85c17c-277e-088a-13b0-3fe96e79cc7d.png)

</TabItem>
</Tabs>

:::caution

Please note that the generated HTML report is not official <NT>Bitrise</NT> content! It is generated entirely according to your <NT>Workflow</NT> configuration. <NT>Bitrise</NT> has no direct control over what appears in the report.

:::
