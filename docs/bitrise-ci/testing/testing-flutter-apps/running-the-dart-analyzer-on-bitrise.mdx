You can perform static analysis of your Flutter code with [the Dart analyzer](https://docs.flutter.dev/testing/debugging#the-dart-analyzer) by using the **<NT>Flutter Analyze</NT>** <NT>Step</NT> on <NT>Bitrise</NT>. The <NT>Step</NT> runs the `flutter analyze` command that makes heavy use of type annotations that you put in your code to help track problems down.

To run the analyzer:

<Tabs>
<TabItem value="workflow-editor" label="Workflow Editor" default>

1. Add the **<NT>Flutter Install</NT>** <NT>Step</NT> to your <NT>Workflow</NT>.

   This <NT>Step</NT> runs the initial setup of the Flutter SDK and installs any missing components.
1. Make sure that the **<NT>Flutter Install</NT>** <NT>Step</NT> installs the correct version of the Flutter SDK for your app: check the **<NT>Flutter SDK version or bundle URL</NT>** input.

   ![flutter-install-step.png](/img/2026-07-16-flutter-install-step.png)

   The input's <NT>default value</NT> is **stable**, which installs the latest stable version of the SDK. You can, however, set the value to a specific version tag, a branch label, or a Flutter SDK bundle URL:

   - You can find the available version tags here: [Version tags](https://github.com/flutter/flutter/tags).
   - You can find the available branch labels here: [Branch labels](https://github.com/flutter/flutter/branches).

   :::tip[Installation bundle URL]

   You can also install the Flutter SDK from an installation bundle URL: set the URL as the value of the same **<NT>Flutter SDK version or bundle URL</NT>** input, for example `https://storage.googleapis.com/flutter_infra/releases/beta/macos/flutter_macos_v1.6.3-beta.zip`.

   :::
1. Add the **<NT>Flutter Analyze</NT>** <NT>Step</NT> to your <NT>Workflow</NT>.
1. Make sure the **<NT>Project Location</NT>** input points to the root directory of your Flutter project.

   - If the project scanner automatically detected it as a Flutter project when [adding it as an app](/en/bitrise-ci/getting-started/adding-a-new-project) on <NT>Bitrise</NT>, you don't have to change the <NT>default value</NT>.
   - If you configured the app manually, check that the location is correct.
1. Set the **Fail Severity** input to the desired level.

   The input determines the minimum severity to fail the <NT>Step</NT>. Any issue with a severity at least as high as the specified fails the <NT>Step</NT>. It has three possible settings, from minimum to maximum severity:

   - **info**: When this setting is selected, info-, warning-, and error-level issues all fail the <NT>Step</NT>.
   - **warning**: When this setting is selected, only warning- and error-level issues fail the <NT>Step</NT>.
   - **error**: This is the <NT>default value</NT>. Only error-level issues fail the <NT>Step</NT>.

   ![fail-severity.png](/img/2026-07-16-flutter-analyze-fail-severity.png)
1. Optionally, add additional flag to the `flutter analyze` command in the **<NT>Additional Parameters</NT>** input.

   For a list of available flags, run `flutter help analyze`. For example, you can use the `--no-congratulate` flag if you don't want to see any output if there are no errors, warnings, hints or lints.

</TabItem>
<TabItem value="configuration-yaml" label="Configuration YAML">

1. Add the `flutter-installer` <NT>Step</NT> to your <NT>Workflow</NT>.

   This <NT>Step</NT> runs the initial setup of the Flutter SDK and installs any missing components.

   ```yaml
   primary:
     description: |
       Builds project and runs tests.
     steps:
       - activate-ssh-key: {}
       - git-clone: {}
       - flutter-installer:
           inputs:
   ```
1. Make sure that the <NT>Step</NT> installs the correct version of the Flutter SDK for your app: you can specify the version using the `version` input.

   The input's <NT>default value</NT> is `stable` which installs the latest stable version of the SDK. If you do not see the `version` input in your Configuration YAML file, it is set to the <NT>default value</NT>. You can, however, set the value to a specific version tag, a branch label, or a Flutter SDK bundle URL:

   - You can find the available version tags here: [Version tags](https://github.com/flutter/flutter/tags).
   - You can find the available branch labels here: [Branch labels](https://github.com/flutter/flutter/branches).

   ```yaml
   # Installing version 3.7.7 of the Flutter SDK
   - flutter-installer:
       inputs:
       - version: 3.7.7
   ```

   :::tip[Installation bundle URL]

   You can also install the Flutter SDK from an installation bundle URL: set the URL as the value of the same `version` input. The URL is expected to begin with `https://storage.googleapis.com/flutter_infra`. For example:

   ```yaml
   - flutter-installer:
       inputs:
       - version: https://storage.googleapis.com/flutter_infra/releases/beta/macos/flutter_macos_v1.6.3-beta.zip
   ```

   :::
1. Add the `flutter-analyze` <NT>Step</NT> to your <NT>Workflow</NT>.

   ```yaml
   primary:
     description: |
       Builds project and runs tests.
     steps:
       - activate-ssh-key: {}
       - git-clone: {}
       - flutter-installer:
           inputs: 
           - version: stable
       - flutter-analyze:
           inputs:
   ```
1. Make sure the `project_location` input points to the root directory of your Flutter project.

   ```yaml
   - flutter-analyze:
       inputs:
       - project_location: "$BITRISE_SOURCE_DIR"
   ```

   - If the project scanner automatically detected it as a Flutter project when [adding it as an app](/en/bitrise-ci/getting-started/adding-a-new-project) on <NT>Bitrise</NT>, you don't have to change the <NT>default value</NT>.
   - If you configured the app manually, check that the location is correct.
1. Set the `fail_severity` input to the desired level.

   The input determines the minimum severity to fail the <NT>Step</NT>. Any issue with a severity at least as high as the specified fails the <NT>Step</NT>. It has three possible settings, from minimum to maximum severity:

   - `info`: When this setting is selected, info-, warning-, and error-level issues all fail the <NT>Step</NT>.
   - `warning`: When this setting is selected, only warning- and error-level issues fail the <NT>Step</NT>.
   - `error`: This is the <NT>default value</NT>. Only error-level issues fail the <NT>Step</NT>.

   ```yaml
   - flutter-analyze:
       inputs:
       - project_location: "$BITRISE_SOURCE_DIR"
       - fail_severity: error
   ```
1. Optionally, add additional flag to the `flutter analyze` command in the `additional_params` input.

   For a list of available flags, run `flutter help analyze`. For example, you can use the `--no-congratulate` flag if you don't want to see any output if there are no errors, warnings, hints or lints.

   ```yaml
   - flutter-analyze:
       inputs:
       - project_location: "$BITRISE_SOURCE_DIR"
       - fail_severity: error
       - additional_params: --no-congratulate
   ```

</TabItem>
</Tabs>

Once your build has run, you can check the <NT>Step</NT>'s output in the [build logs](/en/bitrise-ci/run-and-analyze-builds/build-data-and-troubleshooting/build-logs).
