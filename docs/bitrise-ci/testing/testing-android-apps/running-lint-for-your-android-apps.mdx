<NT>Android</NT> Studio provides [a code scanning tool called lint](https://developer.android.com/studio/write/lint) that can help you identify and correct problems with the structural quality of your code. When using <NT>Android</NT> Studio, configured lint checks run automatically when building your app. Lint checks don't require test cases.

On <NT>Bitrise</NT>, you can run lint checks with our dedicated <NT><GlossTerm baseform="Step">Step</GlossTerm></NT> called [**Android Lint**](https://github.com/bitrise-steplib/bitrise-step-android-lint). To do so:


<Tabs>
  <TabItem value="workflow-editor" label="Workflow Editor">

  1. Add the [**Android Lint**](https://github.com/bitrise-steplib/bitrise-step-android-lint) <NT>Step</NT> to your <NT>Android</NT> app's <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>.

     It should come before any <NT>Step</NT> that builds your app (for example, [**Android Build**](https://github.com/bitrise-steplib/bitrise-step-android-build)), if you have such a <NT>Step</NT> in your <NT>Workflow</NT>.
  1. Make sure the **<NT>Project Location</NT>** input points to the root directory of your <NT>Android</NT> project, where the top level `build.gradle` file is located.
  1. In the **<NT>Module</NT>** and **Variant** inputs, specify the module and variant of the app you want to check.

     ![android-lint.png](/img/_paligo/uuid-624b318c-ddc6-58e7-4c58-e426c2f3db50.png)

     If you are unsure about the exact names of modules and variants in your project, you can check them [in the Project view in Android Studio](https://developer.android.com/studio/projects).
  1. In the **Options** input group, the **Report location pattern** input allows you to specify where the lint report should be found once the <NT>Step</NT> has run.

     The input accepts asterisks as a wildcard. You can set the input to return the results either as html or xml.
  1. In the **Options** input group, the **<NT>Additional Gradle Arguments</NT>** input allows you to pass arguments to the <NT>Gradle</NT> task.

  </TabItem>
  <TabItem value="yaml" label="Configuration YAML">

  1. In the <NT><GlossTerm baseform="configuration yaml">Configuration YAML</GlossTerm></NT> file, add the `android-lint` <NT>Step</NT> to your <NT>Android</NT> app's <NT>Workflow</NT>.

     It should come before any <NT>Step</NT> that builds your app (for example, [**Android Build**](https://github.com/bitrise-steplib/bitrise-step-android-build)), if you have such a <NT>Step</NT> in your <NT>Workflow</NT>.

     ```yaml
     my-workflow:
       steps:
         - git-clone: {}
         - android-lint:
     ```
  1. Make sure the `project_location` input points to the root directory of your <NT>Android</NT> project, where the top level `build.gradle` or `build.gradle.kts` file is located.

     ```yaml
     my-workflow:
       steps:
         - git-clone: {}
         - android-lint:
             inputs:
             - project_location: $BITRISE_SOURCE_DIR
     ```
  1. In the `module` and `variant` inputs, specify the module and variant of the app you want to check.

     If you are unsure about the exact names of modules and variants in your project, you can check them [in the Project view in Android Studio](https://developer.android.com/studio/projects).

     ```yaml
     my-workflow:
       steps:
         - git-clone: {}
         - android-lint:
             inputs:
             - module: module
             - variant: variant
             - project_location: $BITRISE_SOURCE_DIR
     ```
  1. The `report_path_pattern` input allows you to specify where the lint report should be found once the <NT>Step</NT> has run.

     The input accepts asterisks as a wildcard. You can set the input to return the results either as html or xml.

     ```yaml
     my-workflow:
       steps:
         - git-clone: {}
         - android-lint:
             inputs:
             - module: module
             - variant: variant
             - report_path_pattern: '*/build/reports/lint-results*.xml'
             - project_location: $BITRISE_SOURCE_DIR
     ```
  1. The `arguments` input allows you to pass arguments to the <NT>Gradle</NT> task.

     ```yaml
     my-workflow:
       steps:
         - git-clone: {}
         - android-lint:
             inputs:
             - module: module
             - variant: variant
             - report_path_pattern: '*/build/reports/lint-results*.xml'
             - arguments: arg
             - project_location: $BITRISE_SOURCE_DIR
     ```

  </TabItem>
</Tabs>
