A <NT><GlossTerm baseform="step">Step</GlossTerm></NT> encapsulates a build task: the code to perform that task, the inputs and parameters you can define for the task, and the outputs the task generates.

For example the `Git Clone` <NT>Step</NT> performs a `git clone` of the specified repository, with the specified inputs, such as the branch or the commit to clone.

From a technical perspective, a <NT>Step</NT> is a semver versioned repository which includes the code of the <NT>Step</NT> and the interface definition of the <NT>Step</NT>.

The <NT>Step</NT> interface definition is defined in the <GlossTerm baseform="step.yml">`step.yml`</GlossTerm> file for every <NT>Step</NT>. It includes:

- The dependencies of the <NT>Step</NT>.
- The inputs and outputs of the <NT>Step</NT>.
- The title and description of the <NT>Step</NT>.
- Properties such as the issue tracker or support URL, or certain filter properties.

To include <NT>Steps</NT> in your <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>, you have to reference them in the <NT><GlossTerm baseform="configuration yaml">configuration YAML</GlossTerm></NT> file of your project. To do so, you will need the correct reference format for the <NT>Step</NT>. In the configuration file, you can also specify input values for the inputs defined in the <NT>Step</NT>'s interface definition.
