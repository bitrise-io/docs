The <NT><GlossTerm baseform="Step">Step</GlossTerm></NT> data and information you specify in the `<GlossTerm baseform="bitrise.yml">bitrise.yml</GlossTerm>` file are the parameters of the <NT>Step</NT> you want to change, compared to the <NT>Step</NT>’s default definition.

To see the <NT>Step</NT>’s raw interface definition you can check it in the step library. The <NT>Step</NT> interface definitions can be found in the StepLib’s [steps directory](https://github.com/bitrise-io/bitrise-steplib/tree/master/steps).

If you don’t specify any input or other <NT>Step</NT> property in the `bitrise.yml` configuration, only the <NT>Step</NT> (reference/ID), the <NT>Step</NT> will run with the <NT>default value</NT>s as defined by the <NT>Step</NT>’s developer in the interface definition.

Let’s see an example with a single [**Script**](https://github.com/bitrise-io/steps-script) <NT>Step</NT>, which will be executed when you run `bitrise run test`:

```yaml
format_version: 11
default_step_lib_source: https://github.com/bitrise-io/bitrise-steplib.git
    
workflows:
  test:
    steps:
    - script:
```

Specify inputs for the <NT>Step</NT> with the `inputs:` list property. An input consists of a *key* and a *value*:

:::warning[Indentation]

Indentation in the [YAML format](https://github.com/yaml/yaml-spec) is very important! You should use two- or four-space indentation, and you can’t use tabs to indent!

:::

```yaml
format_version: 11
default_step_lib_source: https://github.com/bitrise-io/bitrise-steplib.git
    
workflows:
  test:
    steps:
    - script:
        inputs:
        - content: "echo 'Hello World!'"
```

If the <NT>Step</NT> doesn’t have any required inputs you don’t have to specify an input. You can specify values for as many inputs as you want to.

<NT>Step</NT> input values are always **string** / text values and they are passed to the <NT>Step</NT> as <NT>Environment Variables</NT>. The value can be multiline too, using the standard YAML multiline format:

```yaml
format_version: 11
default_step_lib_source: https://github.com/bitrise-io/bitrise-steplib.git

workflows:
  test:
    steps:
    - script@1.1.3:
        inputs:
        - content: |
            #!/bin/bash
            set -ex
            var_to_print='Hello World!'
            echo "${var_to_print}"
```

If you use a multiline value, like the one above, you have to indent the value with either two or four spaces, compared to the key!

Force a <NT>Step</NT> to run even if a previous <NT>Step</NT> fails by setting the `is_always_run` property to `true`:

```yaml
format_version: 1.3.1
default_step_lib_source: https://github.com/bitrise-io/bitrise-steplib.git

workflows:
  test:
    steps:
    - script@1.1.3:
        is_always_run: true
        inputs:
        - content: "puts 'Hello Ruby!'"
        - runner_bin: ruby
```

Use the `title` property to add a descriptive title to your <NT>Step</NT>:

```yaml
format_version: 11
default_step_lib_source: https://github.com/bitrise-io/bitrise-steplib.git

workflows:
  test:
    steps:
    - script@1.1.3:
        title: Print Hello Ruby
        is_always_run: true
        inputs:
        - content: "puts 'Hello Ruby!'"
        - runner_bin: ruby
```
