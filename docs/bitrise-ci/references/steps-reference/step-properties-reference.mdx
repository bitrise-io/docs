<NT><GlossTerm baseform="Step">Step</GlossTerm></NT> properties provide important data of a <NT>Step</NT>, such as its project type, its source code or the dependencies it requires. A <NT>Step</NT>’s inputs and outputs are also defined as <NT>Step</NT> properties. Let’s take a look at the properties!

- `title`, `summary` and `description` : metadata, for comments, tools and GUI.

  <Partial_TIPMetaPropertiesAsPermanentComments />
- `website` : official website of the <NT>Step</NT> / service.
- `source_code_url` : the url where the <NT>Step</NT>’s source code can be viewed.
- `support_url` : url to the <NT>Step</NT>’s support / issue tracker.
- `published_at` : *auto-generated at share* - the StepLib publish date of the <NT>Step</NT>’s version
- `source` : *auto-generated at share* <NT>git clone</NT> information.
- `asset_urls` : *auto-generated at share* <NT>Step</NT> assets (StepLib specific), like icon image.
- `host_os_tags` : supported operating systems. *Currently unused, reserved for future use.*
- `project_type_tags` : project type tags if the <NT>Step</NT> is project type specific. Example: `ios` or `android`. Completely optional, and only used for search and filtering in <NT>Step</NT> lists.
- `type_tags` : generic type tags related to the <NT>Step</NT>. Example: `utility`, `test` or `notification`. Similar to `project_type_tags`, this property is completely optional, and only used for search and filtering in <NT>Step</NT> lists.
- `deps` : specifies the required dependencies of the <NT>Step</NT>. To declare a dependency, specify a package manager and then the dependency you wish to install.
- `inputs` : inputs (Environments) of the <NT>Step</NT>.
- `outputs` : outputs (Environments) of the <NT>Step</NT>.
