<NT><GlossTerm baseform="step">Step</GlossTerm></NT>s are the heart of <NT>Bitrise</NT>. A <NT>Step</NT> is a build task: for example, the [**Git Clone Repository**](https://github.com/bitrise-steplib/steps-git-clone) <NT>Step</NT> clones your Git repository at the start of a build while the [**Google Play Deploy**](https://github.com/bitrise-io/steps-google-play-deploy) <NT>Step</NT> can deploy your finished app to the Play Store.

A <NT>Step</NT> contains the code that performs the build task. You can configure the inputs and parameters that define the task, and view and reuse the outputs a <NT>Step</NT> generates. Reusing the output means that another <NT>Step</NT> can use it as the value of one of its inputs.

Our <NT>Steps</NT> are defined in YAML format, and written in either bash or Go.

You can find a list of our <NT>Steps</NT> in:

- [The Integrations page](https://www.bitrise.io/integrations).
- [The Bitrise StepLib](https://github.com/bitrise-io/bitrise-steplib).
- The <NT><GlossTerm baseform="workflow editor">Workflow Editor</GlossTerm></NT>.

You can also create and run any custom script you want as part of your <NT>Bitrise</NT> build using the [**Script**](https://github.com/bitrise-io/steps-script) <NT>Step</NT>. All you need to do is specify the script runner (the default is bash), add the script, and run a build. For more information on how to set up these custom scripts, check out [Bitrise Script Step - Running (Bash, Python, NodeJS, Ruby, Swift, Kotlin)](https://support.bitrise.io/hc/en-us/articles/8932506599955-Bitrise-Script-Step-Running-Bash-Python-NodeJS-Ruby-Swift-Kotlin-).

For the purposes of your builds, <NT>Steps</NT> can be managed directly from the <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT> Editor. You can rearrange them, set their versions, add or remove <NT>Steps</NT> at any time.


## Types of <NT>Steps</NT>

<NT>Bitrise</NT> has over 400 <NT>Steps</NT> in its [Step Library (StepLib)](https://github.com/bitrise-steplib) which third-party companies or open-source teams can enrich with <NT>Steps</NT> based on their services/tools.

There are three different types of <NT>Steps</NT> at <NT>Bitrise</NT>. You can identify each one based on their labels on our GUI:

- **Official <NT>Bitrise Steps</NT>**: These <NT>Steps</NT> are created and maintained by <NT>Bitrise</NT>. These <NT>Steps</NT> are labeled by a green badge and a "B".
- **<NT>Verified Steps</NT>**: These <NT>Steps</NT> are created and maintained by the community, but they are owned by a service or tool or an open-source team that guarantees secure, maintained, consistent, and high-quality performance for any <NT>Bitrise</NT> user. These <NT>Steps</NT> are labeled by a blue badge and a check mark. For more information about <NT>Verified Steps</NT>, check out [our guide](/en/bitrise-ci/workflows-and-pipelines/developing-your-own-bitrise-step/verified-steps#about-verified-steps).
- **Community <NT>Steps</NT>**: These <NT>Steps</NT> can be created by anyone in the community. These <NT>Steps</NT> are not labeled and don't have a badge. If you would like to learn how to develop <NT>Steps</NT> and share them with the <NT>Bitrise</NT> community, check out [Developing a new Step](/en/bitrise-ci/workflows-and-pipelines/developing-your-own-bitrise-step/developing-a-new-step).
