You can detect and abort hanging <NT><GlossTerm baseform="Step">Step</GlossTerm></NT>s that do not produce any log output for a certain amount of time using the **<NT>No Output Timeout</NT>** function. You can use this function to automatically abort hung builds after a configurable timeout period to save credits and to enable <NT>Bitrise</NT> to gather data on hanging builds.

You can use the **<NT>No Output Timeout</NT>** function in two ways:

- [Enable globally for all Steps](#enabling-the-no-output-timeout-function-globally-for-all-steps).
- [Enable for specific Steps only](#enabling-the-no-output-timeout-function-for-specific-steps).

The **<NT>No Output Timeout</NT>** function can be used alongside other timeout functionalities, such as the one described in the [M1 Hanging builds issue guide](https://devcenter.bitrise.io/en/infrastructure/build-stacks/apple-silicon-m1-stacks.html#hanging-builds-issue) or in [Setting a time limit for Steps](/en/bitrise-ci/workflows-and-pipelines/steps/setting-a-time-limit-for-steps).


## Enabling the <NT>No Output Timeout</NT> function globally for all <NT>Steps</NT>

:::important[<NT>Bitrise</NT> 1.50.0 or newer is required]

You must use <NT>Bitrise</NT> 1.50.0 or a newer version to use the **<NT>No Output Timeout</NT>** function.

:::

To enable the **<NT>No Output Timeout</NT>** globally for all <NT>Steps</NT>:

1. <Partial_OpeningTheWorkflowEditor />
1. Go to the **<NT><GlossTerm baseform="secret">Secret</GlossTerm></NT>s** tab.
1. Add a new secret Env Var: `BITRISE_NO_OUTPUT_TIMEOUT` and set its value to the number of seconds you want the build to wait for output logs before aborting.

   We recommend setting it to 600 seconds (10 minutes).

   ![app-level-secret.png](/img/_paligo/uuid-7d34c319-a1a2-05a1-90b0-ded10c048482.png)

   :::note[Aborting a hanging build automatically]

   Compared to [setting a time limit for Steps](/en/bitrise-ci/workflows-and-pipelines/steps/setting-a-time-limit-for-steps), the **<NT>No Output Timeout</NT>** function only aborts a build no output log is generated for a set amount of time.

   Any log output will reset the timeout. This can result in a <NT>Step</NT> running longer than the value you set in `BITRISE_NO_OUTPUT_TIMEOUT`.

   :::
1. Click **<NT>Save</NT>**. You can check the Build log of your build to determine if the function is enabled.

   ![step-force-output.png](/img/_paligo/uuid-dc3bfafa-9720-8935-557a-97c317a8a624.png)

And that's it! From now on, whenever a <NT>Step</NT> does not produce an output log for the number of seconds you set in `BITRISE_NO_OUTPUT_TIMEOUT` , your build automatically aborts with the message: `Abort via Bitrise CLI (no output timeout)`.

:::tip[Disabling the <NT>No Output Timeout</NT> function for a specific <NT>Step</NT>]

If you enabled the **<NT>No Output Timeout</NT>** function globally and want to disable it for specific <NT>Steps</NT>, you can add the `no_output_timeout: 0` <NT>Step</NT> property just above the `inputs:` part of the chosen <NT>Step</NT>(s) in the `bitrise.yml`.

For example:

```yaml
steps:    
    - virtual-device-testing-for-ios@1:
        no_output_timeout: 0
        inputs:
        - zip_path: "$BITRISE_PROJECT_PATH"
```

:::


## Enabling the <NT>No Output Timeout</NT> function for specific <NT>Steps</NT>

:::important[<NT>Bitrise</NT> 1.50.0 or newer is required]

You must use <NT>Bitrise</NT> 1.50.0 or a newer version to use the **<NT>No Output Timeout</NT>** function.

:::

You can enable the **<NT>No Output Timeout</NT>** function for specific <NT>Steps</NT> by adding the `no_output_timeout` <NT>Step</NT> property below a <NT>Step</NT> in your <NT><GlossTerm baseform="bitrise.yml">bitrise.yml</GlossTerm></NT> file.

To do so with the <NT><GlossTerm baseform="workflow editor">Workflow Editor</GlossTerm></NT>:

1. <Partial_OpeningTheWorkflowEditor />
1. Go to the **bitrise.yml** tab.
1. Search for the <NT>Step</NT>(s) where you would like to use the function.
1. Insert `no_output_timeout: 12` <NT>Step</NT> property and value just above the `inputs:` part of the chosen <NT>Step</NT>(s) to automatically abort a build if the <NT>Step</NT> does not produce an output log for 12 seconds.

   :::tip[Setting the `no_output_timeout` <NT>Step</NT> property to 0]

   You can disable the **<NT>No Output Timeout</NT>** function for a <NT>Step</NT> by setting the `no_output_timeout` <NT>Step</NT> property to 0.

   :::

   Let's look at an example where a <NT>Script</NT> <NT>Step</NT> will always be aborted automatically:

   ```ruby
   output_slows_down:
       steps:
       - script@1:
           title: Output is slower and slower
           no_output_timeout: 12
           inputs:
           - content: |-
               #!/usr/bin/env bash
               for i in {1..5}
               do
                 DURATION=$((5*i))
                 echo "🏃‍step output (sleeping ${DURATION}s)"
                 sleep $DURATION
               done
   ```

And that's it! Now, if your <NT>Step</NT> hangs and does not produce a single output log for the number of seconds you specified in the `no_output_timeout` <NT>Step</NT> property, the build will be aborted automatically.
