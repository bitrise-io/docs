Ensure that your builds do not exceed the time limit by setting up <NT><GlossTerm baseform="Step">Step</GlossTerm></NT> timeout for <NT>Steps</NT> that might cause builds to hang.

A <NT>Step</NT> timeout, defined in seconds, sets a maximum time a <NT>Step</NT> is allowed to run. If the <NT>Step</NT> exceeds that limit, it is marked as failed and the build fails. <NT>Steps</NT> with **<NT>Run even if previous Step(s) failed</NT>** enabled will still run. This is useful if, for example, your builds hang for not immediately obvious reasons — you can set timeouts for the <NT>Step</NT> or <NT>Steps</NT> which are suspected to have caused the problem.

1. Find the <NT>Step</NT> in the `bitrise.yml` file.

   You can edit the file locally, or in the YAML editor in the <NT><GlossTerm baseform="workflow editor">Workflow Editor</GlossTerm></NT>.
1. Add a `timeout` property before the other <NT>Step inputs</NT> and specify its value in seconds:

   ```yaml
   - xcode-test@1.18.14:
        timeout: 120
        inputs:
        - project_path: "$BITRISE_PROJECT_PATH"
        - scheme: "$BITRISE_SCHEME"
   ```
