You can configure a <NT><GlossTerm baseform="step">Step</GlossTerm></NT> to run even if a previous <NT>Step</NT> in the <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT> failed. For example, if the [**Bitrise.io Cache:Pull**](https://github.com/bitrise-steplib/steps-cache-pull.git) <NT>Step</NT> fails, there is no reason not to run the subsequent <NT>Steps</NT>.

:::note[Enabling a <NT>Step</NT> conditionally]

You can also configure <NT>Steps</NT> to run only in certain conditions: [Enabling or disabling a Step conditionally](/en/bitrise-ci/workflows-and-pipelines/steps/enabling-or-disabling-a-step-conditionally).

:::

To set a given <NT>Step</NT> to run even if a previous <NT>Step</NT> failed:

1. <Partial_OpeningTheWorkflowEditorAndSelectingAWorkflow />
1. Select the <NT>Step</NT> you want to configure to always run.
1. On the right, find and open the **<NT>When to run</NT>** section and toggle the **<NT>Run even if previous Step(s) failed</NT>** option.
