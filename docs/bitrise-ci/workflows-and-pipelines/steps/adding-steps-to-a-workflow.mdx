<NT><GlossTerm baseform="step">Step</GlossTerm></NT>s can be added or removed any time from your <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT>s. When adding a new app to <NT>Bitrise</NT>, the <NT><GlossTerm baseform="project scanner">project scanner</GlossTerm></NT> generates at least one <NT>Workflow</NT> with some default <NT>Steps</NT> but you don't have to use these at all: it's up to you what <NT>Steps</NT> you want to include.

You can add any <NT>Step</NT> to your <NT>Workflow</NT> - there are absolutely no restrictions. Please note that this means that it’s possible to add a <NT>Step</NT> specific to, for example, iOS apps to a <NT>Workflow</NT> of an <NT>Android</NT> app. Always make sure you only add the relevant <NT>Steps</NT> to your <NT>Workflow</NT>!


## Adding <NT>Steps</NT> in the <NT>Workflow Editor</NT>

1. <Partial_OpeningTheWorkflowEditor />
1. Click the **+** symbol between two <NT>Steps</NT> to insert a <NT>Step</NT> at that position.
1. Search for the <NT>Step</NT> you need.

   :::important[Search filter]

   Be aware of the filter tags below the search field. You can filter by category (such as **<NT>Build</NT>**, **Deploy**, or **Test**) and by maintainer (**Official**, **Verified**, **Community**). You can select multiple tags at the same time. By default, no filters are applied and all <NT>Steps</NT> are shown.

   :::
1. Click the <NT>Step</NT> to add it to the <NT>Workflow</NT>.


## Adding <NT>Steps</NT> from alternative sources

Generally, we recommend using <NT>Steps</NT> that are part of the official <NT>Bitrise Step</NT> Library. But you can add <NT>Steps</NT> from other sources, if you want to: you can use either a Git URL or a local path.

On the <NT>Bitrise website</NT>, the `git::` special source is the easiest way to use a <NT>Step</NT> that is not in the <NT>Bitrise Step</NT> Library. Let’s see how.

1. <Partial_OpeningTheWorkflowEditor />
1. Switch to **YAML** at the top of the <NT><GlossTerm baseform="workflow editor">Workflow Editor</GlossTerm></NT>.
1. Add the git URL of the <NT>Step</NT> you want to use.

   In this example, we’re adding the <NT>Script</NT> <NT>Step</NT> from a git source:

   ```yaml
   - git::https://github.com/bitrise-io/steps-script.git@1.1.3:
   ```
1. Click **<NT>Save</NT>**.


## Removing a <NT>Step</NT> from a <NT>Workflow</NT>

You can remove a <NT>Step</NT> at any time. Be aware, however, that if you add it back at a later date, the <NT>Step inputs</NT> will be set to the <NT>default value</NT> - any custom configuration will be gone.

:::tip[Disabling a <NT>Step</NT>]

If you don't want a <NT>Step</NT> to run but want to keep the <NT>Step</NT> configuration intact, you can disable the <NT>Step</NT>: [Enabling or disabling a Step conditionally](/en/bitrise-ci/workflows-and-pipelines/steps/enabling-or-disabling-a-step-conditionally).

:::

1. <Partial_OpeningTheWorkflowEditor />
1. Click the <NT>Step</NT> you want to remove.
1. Click the three-dot menu icon and select **<NT>Delete</NT> item**.
1. Click **<NT>Save</NT>**.
