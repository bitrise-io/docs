A <NT>Bitrise</NT> <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT> is a collection of <NT><GlossTerm baseform="step">Step</GlossTerm></NT>s. When a build of a project is running, each <NT>Step</NT> will be executed in the order that is defined in the <NT>Workflow</NT>. <NT>Workflows</NT> can be created, defined and modified in two ways:

- Using the graphical <NT><GlossTerm baseform="workflow editor">Workflow Editor</GlossTerm></NT> on [bitrise.io](https://www.bitrise.io), or [the offline version on your own device](/en/bitrise-ci/bitrise-cli/installing-and-upgrading-the-offline-workflow-editor).
- Directly editing the <NT><GlossTerm baseform="bitrise.yml">`bitrise.yml`</GlossTerm></NT> file of your project.

Ultimately, both methods modify the `bitrise.yml` file - the **<NT>Workflow Editor</NT>** is simply a friendlier way of doing so!

By default, a single build is a single <NT>Workflow</NT>. But you can also chain <NT>Workflows</NT> together so they run in succession, as well as to <NT><GlossTerm baseform="trigger">trigger</GlossTerm></NT> multiple <NT>Workflow</NT> to run simultaneously.

<NT>Workflows</NT> can also be arranged into [Pipelines](/en/bitrise-ci/workflows-and-pipelines/build-pipelines/pipelines-with-stages/configuring-a-pipeline-with-stages). A <NT>Pipeline</NT> consists of multiple Stages and each Stage consists of one or more <NT>Workflows</NT> which run in parallel.
