:::note[<NT>Workflows</NT> in YAML]

This guide is about creating a <NT>Workflow</NT> in the <NT><GlossTerm baseform="workflow editor">Workflow Editor</GlossTerm></NT>. For YAML configuration syntax, check out [Workflow level properties](/en/bitrise-ci/references/configuration-yaml-reference#workflow-level-properties).

:::

It’s very simple to create your own <NT>Workflow</NT> with the <NT>Workflow Editor</NT>. You can create new <NT>Workflows</NT> based on any of the existing ones, or you can simply create an empty <NT>Workflow</NT> and add the <NT><GlossTerm baseform="step">Step</GlossTerm></NT>s yourself.

If, for example, you create a <NT>Workflow</NT> based on your **primary** one, it means that it will be created with the exact same <NT>Steps</NT> and input values as the **primary** <NT>Workflow</NT>.

1. <Partial_OpeningTheWorkflowEditor />
1. Open the <NT>Workflow</NT> selector dropdown and click **<NT>Create</NT> <NT>Workflow</NT>**.

   ![Workflow selector dropdown with Create Workflow button](/img/workflows-and-pipelines/2026-07-06-create-workflow-dropdown.png)
1. In the dialog, give your <NT>Workflow</NT> an ID.

   The ID can only contain the following characters: `A-Za-z0-9-_.`

   ![Create Workflow dialog with ID field](/img/workflows-and-pipelines/2026-07-06-create-workflow-dialog.png)
1. From the dropdown menu labeled **<NT>Based on</NT>**, select the <NT>Workflow</NT> you want to use as the basis for the new one.

   Alternatively, choose the **An empty workflow** option to create an empty <NT>Workflow</NT>.
1. Add the <NT>Steps</NT> you need to your <NT>Workflow</NT>.

   Click the **+** (**<NT>Add Step or Step bundle</NT>**) symbol between two <NT>Steps</NT> to insert a <NT>Step</NT> at that position. Remove <NT>Steps</NT> you do not need by clicking on the <NT>Step</NT>, opening the three-dot menu, and selecting **<NT>Delete</NT> item**.
1. Click **<NT>Save changes</NT>** in the top right corner.
