If you have a lot of <GlossTerm baseform="Project">projects</GlossTerm>, you might not want to spend time with setting up <GlossTerm baseform="workflow">Workflows</GlossTerm> for each and every one of them separately, especially if there’s little difference between the Workflows you want to run for them. If so, the easiest thing to do is to simply copy an existing Workflow.

:::important[YAML mode only]

Copying a Workflow to another project is only possible in YAML mode. [You can create a new Workflow based on an existing one](/en/bitrise-ci/workflows-and-pipelines/workflows/creating-a-workflow) but only for the same app.

:::

You can copy a Workflow from any [configuration YAML file](/en/bitrise-ci/configure-builds/configuration-yaml/configuration-yaml-overview), including your local files. To copy an existing Workflow from one Bitrise <GlossTerm baseform="project">project</GlossTerm> to another on our website:

1. <Partial_OpeningTheWorkflowEditor />
1. On the left, select **<GlossTerm baseform="configuration yaml">Configuration YAML</GlossTerm>** from the navigation menu.
1. Select the Workflow you want and copy it.
1. Open the project you want to copy it to.
1. Go to **Workflows** and then select **Configuration YAML** from the navigation menu.
1. Paste the Workflow under the `workflows` property.
