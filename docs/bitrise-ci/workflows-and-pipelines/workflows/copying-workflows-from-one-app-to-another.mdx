If you have a lot of <NT><GlossTerm baseform="Project">project</GlossTerm></NT>s, you might not want to spend time with setting up <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT>s for each and every one of them separately, especially if there’s little difference between the <NT>Workflows</NT> you want to run for them. If so, the easiest thing to do is to simply copy an existing <NT>Workflow</NT>.

:::important[YAML mode only]

Copying a <NT>Workflow</NT> to another project is only possible in YAML mode. [You can create a new Workflow based on an existing one](/en/bitrise-ci/workflows-and-pipelines/workflows/creating-a-workflow) but only for the same app.

:::

You can copy a <NT>Workflow</NT> from any [configuration YAML file](/en/bitrise-ci/configure-builds/configuration-yaml/configuration-yaml-overview), including your local files. To copy an existing <NT>Workflow</NT> from one <NT>Bitrise</NT> <NT><GlossTerm baseform="project">project</GlossTerm></NT> to another on our website:

1. <Partial_OpeningTheWorkflowEditor />
1. Switch to **YAML** at the top of the <NT><GlossTerm baseform="workflow editor">Workflow Editor</GlossTerm></NT>.
1. Select the <NT>Workflow</NT> you want and copy it.
1. Open the project you want to copy it to.
1. Go to **<NT>Workflows</NT>** and switch to **YAML** at the top of the <NT>Workflow Editor</NT>.
1. Paste the <NT>Workflow</NT> under the `workflows` property.
