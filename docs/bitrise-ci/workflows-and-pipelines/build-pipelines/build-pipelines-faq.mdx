**Can I use <NT>Pipelines</NT> if I store my bitrise.yml file in my own repository?**

Yes, absolutely. It makes no difference to <NT>Pipelines</NT>.

**How can I set the stack for my <NT>Pipeline</NT>?**

In the current version, you can set the default stack for your app, or you can set <NT>Workflow</NT>-specific stacks, just like with standalone builds: [Setting the stack for your builds](/en/bitrise-ci/configure-builds/configuring-build-settings/setting-the-stack-for-your-builds).

**How can I use <NT>Environment Variables</NT> with a <NT>Pipeline</NT>?**

You can keep using project-level and <NT>Workflow</NT>-level [Environment Variables](/en/bitrise-ci/configure-builds/environment-variables).

**Can I rerun a failed <NT>Pipeline</NT>?**

Yes. Go to the **<NT>Pipeline</NT> details** page, and click the **<NT>Rebuild</NT>** button. From there you have the option to rebuild unsuccessful <NT>Workflows</NT> or to rebuild the entire <NT>Pipeline</NT>. Both options are also available with remote access.

**Does the <NT>Rolling Builds</NT> feature work on <NT>Pipelines</NT>?**

Yes! You don’t have to worry about wasting credits with builds that are no longer necessary because of new commits or pull requests.
