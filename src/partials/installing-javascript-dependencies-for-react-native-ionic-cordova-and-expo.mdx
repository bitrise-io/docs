## Installing <NT>JavaScript</NT> dependencies

If the <NT>Bitrise</NT> <NT><GlossTerm baseform="project scanner">project scanner</GlossTerm></NT> has successfully scanned your project, **<NT>Run npm command</NT>** or **<NT>Run yarn command</NT>** <NT><GlossTerm baseform="Step">Step</GlossTerm></NT>s will be included in your default <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT>s. These <NT>Steps</NT> can install the missing <NT>JavaScript</NT> dependencies for your app.

For native <NT>Android</NT> dependencies, you can use the **<NT>Install missing Android SDK components</NT>** <NT>Step</NT>.

For native iOS dependencies, you can use, among others, the **<NT>Brew install</NT>** <NT>Step</NT> or the **<NT>Run CocoaPods install</NT>** <NT>Step</NT>.

To install <NT>JavaScript</NT> dependencies with npm:

:::note[Using Yarn instead of npm]

In this guide, we're using npm to install <NT>JavaScript</NT> dependencies. However, you can use the **<NT>Run yarn command</NT>** <NT>Step</NT>: it can install missing JS dependencies without any additional configuration required.

:::

1. <NT>Log in</NT> to <NT>Bitrise</NT> and select **<NT>Bitrise CI</NT>** on the left, then select your project.
1. Click the **<NT>Workflows</NT>** button on the main page.
1. Make sure your <NT>Workflow</NT> includes the **<NT>Run npm command</NT>** <NT>Step</NT>.
1. In **The 'npm' command with arguments to run** input variable, type `install`.

   ![Getting_started_with_React_Native_apps.png](/img/_paligo/uuid-021a140f-5b4c-697c-a5ac-4b5429ce1bab.png)

   :::tip[Using the `npm ci` command instead of `npm install`]

   If you already have an up to date `package-lock.json` file in your project, we recommend using the `ci` command in **The 'npm' command with arguments to run** input. Using `npm ci` can not only result in much faster build times compared to `npm install` but more reliable builds as well.

   :::
