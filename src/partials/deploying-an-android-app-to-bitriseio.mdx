## Deploying an <NT>Android</NT> app to bitrise.io

In this section, we'll go through how to deploy your <NT>Android</NT> <NT><GlossTerm baseform="Project">project</GlossTerm></NT> to [bitrise.io](https://www.bitrise.io/).

Deploying to [bitrise.io](https://www.bitrise.io) means that the build artifacts generated during the build will be available for download once the build is finished. You can use this to test your <NT>Android</NT> app on your own test devices, for example.

To deploy your app to [bitrise.io](https://www.bitrise.io/):

<Tabs>
<TabItem value="workflow-editor" label="Workflow Editor" default>

1. Make sure your <NT><GlossTerm baseform="Workflow">Workflow</GlossTerm></NT> contains the **[Android Build](https://www.bitrise.io/integrations/steps/android-build)** <NT><GlossTerm baseform="Step">Step</GlossTerm></NT> to build your app.

   Optionally, you can build your app with the **[Gradle Runner](https://www.bitrise.io/integrations/steps/gradle-runner)** <NT>Step</NT>. This requires a bit more configuration from you but allows for more extensive customization.
1. Add the **[Deploy to Bitrise.io](https://www.bitrise.io/integrations/steps/deploy-to-bitrise-io)** <NT>Step</NT> to your <NT>Workflow</NT>.

   :::tip[Notifying other users]

   You can use the **<NT>Notify: User Roles</NT>** and the **<NT>Notify: Emails</NT>** inputs of the <NT>Step</NT> to set up notifications about your deploy. Click the input names to reveal more information about how to configure them.

   :::
1. Optionally, set the **<NT>Enable</NT> public page for the App?** input of the <NT>Step</NT> to **true** so the <NT>Step</NT> [enables the public install page](/en/bitrise-ci/deploying/bitrise-ota-app-deployment#deploying-with-the-deploy-to-bitriseio-step) for your app.
1. Run a build.

</TabItem>
<TabItem value="bitriseyml" label="Configuration YAML">

1. Open the `bitrise.yml` file of your app.
1. Make sure your <NT>Workflow</NT> contains the `android-build` <NT>Step</NT> to build your app.

   ```yaml
   workflows:
     example-workflow:
       steps:
       - android-build@1:
   ```

   Optionally, you can build your app with the `gradle-runner` <NT>Step</NT>. This requires a bit more configuration from you but allows for more extensive customization.
1. Set the `module` and/or `variant` input to tell the <NT>Step</NT> what to build.

   In this example, we're building a debug variant of the <NT>Android</NT> project.

   ```yaml
   workflows:
     example-workflow:
       steps:
       - android-build@1:
           inputs:
           - variant: debug
       - deploy-to-bitrise-io: {}
   ```
1. Add the `deploy-to-bitrise-io` <NT>Step</NT> to your <NT>Workflow</NT>.

   ```yaml
   workflows:
     example-workflow:
       steps:
       - android-build@1:
           inputs:
           - variant: debug
       - deploy-to-bitrise-io: {}
   ```

   :::tip[Notifying other users]

   You can use the `notify_user_groups` and the `notify_email_list` inputs of the <NT>Step</NT> to set up notifications about your deploy:

   - The `notify_user_groups` input allows you to send notifications based on the [access roles](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci) granted to users. For example, you can set the input to notify everyone with an **Admin** and a **Developer** role. Set multiple roles separated by a comma: `- notify_user_groups: admins, testers`.
   - The `notify_email_list` input only accepts <NT><GlossTerm baseform="secret">Secret</GlossTerm></NT>s, and the Secret should contain comma-separated lists of email addresses.

   :::
1. Optionally, set the **<NT>Enable</NT> public page for the App?** input of the <NT>Step</NT> to **true** so the <NT>Step</NT> [enables the public install page](/en/bitrise-ci/deploying/bitrise-ota-app-deployment#deploying-with-the-deploy-to-bitriseio-step) for your app.
1. Run a build.

</TabItem>
</Tabs>

The **<NT>Deploy to Bitrise.io</NT>** <NT>Step</NT> will deploy the app. You can share the generated binary with your team members using the build’s URL.
