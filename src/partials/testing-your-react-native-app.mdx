## Testing your <NT>React Native</NT> app

You can use <NT>React Native</NT>’s built in testing method, called **jest** to perform unit tests.

1. <NT>Log in</NT> to <NT>Bitrise</NT> and select **<NT>Bitrise CI</NT>** on the left, then select your project.
1. Click the **<NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT>s** button on the main page.
1. Add the **run npm command** <NT><GlossTerm baseform="step">Step</GlossTerm></NT> to your <NT>Workflow</NT>.
1. In the **npm command with arguments to run** input field, type `test`.

   ![Getting_started_with_React_Native_apps.png](/img/_paligo/uuid-f5f86a6c-d609-1970-a451-0de6d5b4edb3.png)

For more detailed guides on <NT>React Native</NT> testing, check out [Testing React Native apps](/en/bitrise-ci/testing/testing-react-native-apps/running-detox-tests-on-bitrise).

### Viewing <NT>React Native</NT> test results

<NT>Bitrise</NT> allows you to view and analyze your test results in one convenient place: [Deploying and viewing test results](/en/bitrise-ci/testing/deploying-and-viewing-test-results).

By default, tests run with the **<NT>Run npm command</NT>** <NT>Step</NT> won't show up in test reports. However, you can export the results. . The basic process is as follows:

1. Generate a `junit.xml` file during the build.

   For example, you can use [jest-junit](https://www.npmjs.com/package/jest-junit) to prepare a report.
1. Add the **<NT>Export test results to Test Reports</NT>** <NT>Step</NT> to your <NT>Workflow</NT>.

   ![Managing_an_app_s_bitrise.png](/img/_paligo/uuid-fd13332e-571f-e706-88ed-a20a29db7aea.png)
1. In the **<NT>Path</NT> where custom test results reside** input, add the folder in which your `junit.xml` file and other test results are located.
1. In the **<NT>Test result search pattern</NT>** input, set `*.xml`.
1. In the **<NT>The name of the test</NT>** input, set <NT>the name of the test</NT> run.

   The test results will be under this name.
1. Make sure you have the **<NT>Deploy to Bitrise.io</NT>** <NT>Step</NT> in your <NT>Workflow</NT>.
