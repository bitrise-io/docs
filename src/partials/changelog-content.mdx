## 2026 July

### <time dateTime="2026-07-14">2026-07-14</time> RDE CLI docs refreshed to match current commands {#2026-07-14-rde-cli-docs-refreshed-to-match-current-commands}

The RDE CLI reference now covers stack-based session creation, running commands in a session with `session exec`, VNC tunneling, <NT>workspace</NT> selection, and the reconnect behavior of `rde claude`. See [Bitrise RDE CLI](/en/bitrise-rde/rde-options/bitrise-rde-cli).

### <time dateTime="2026-07-10">2026-07-10</time> Building a <NT>Slack</NT>-native coding agent {#2026-07-10-building-a-slack-native-coding-agent}

A new recipe walks through a reference architecture for a <NT>Slack</NT>-native coding agent: a locked-down orchestrator that delegates coding work to disposable Remote Dev Environments, plus an open source reference implementation you can try. See [Building a Slack-native coding agent](/en/bitrise-rde/recipes/building-a-slack-native-coding-agent-recipe).

### <time dateTime="2026-07-09">2026-07-09</time> <NT>Bitrise</NT> Desktop App for macOS {#2026-07-09-bitrise-for-mac-menu-bar-app-guide}

A new guide covers the <NT>Bitrise</NT> Desktop App for macOS: installing it, signing in, selecting a <NT>workspace</NT> and project, configuring filters and notifications, and troubleshooting. See [Bitrise Desktop App for macOS](/en/bitrise-ci/run-and-analyze-builds/bitrise-desktop-app-for-macos).

### <time dateTime="2026-07-07">2026-07-07</time> Product subscriptions can now be canceled independently {#2026-07-07-product-subscriptions-can-now-be-canceled-independently}

Each product (<NT>Bitrise CI</NT>, <NT>Build Cache</NT>, <NT>Release Management</NT>, and so on) has its own independently cancellable subscription, and add-ons can be canceled on their own too. The docs now walk through the updated **<NT>Plan & Billing</NT>** cancellation flow. See [Workspace billing and invoicing](/en/bitrise-platform/workspaces/workspace-billing-and-invoicing).

### <time dateTime="2026-07-02">2026-07-02</time> <NT>Bitrise MCP</NT> server drops self-registration tools {#2026-07-02-bitrise-mcp-server-drops-self-registration-tools}

The `register` and `verify_registration` tools, along with the `registration` API group, are no longer part of the <NT>Bitrise MCP</NT> server. See [Bitrise MCP tools](/en/bitrise-platform/ai/bitrise-mcp/tools).

### <time dateTime="2026-06-17">2026-06-17</time> <NT>GitHub</NT> App integration: event subscriptions clarified {#2026-06-17-github-app-integration-event-subscriptions-clarified}

The [GitHub App integration](/en/bitrise-platform/repository-access/github-app-integration) page now has an Event subscriptions section explaining that the <NT>Bitrise</NT> <NT>GitHub</NT> App subscribes to `push`, `pull_request`, and `issue_comment` events via <NT>GitHub</NT>'s app installation mechanism, not through repo-level webhooks. The section also warns that any existing manual webhooks pointing to `hooks.bitrise.io` must be removed after switching to the <NT>GitHub</NT> App to avoid duplicate builds.

### <time dateTime="2026-06-15">2026-06-15</time> <NT>Bitrise MCP</NT>: new registration tools and API group {#2026-06-15-bitrise-mcp-new-registration-tools-and-api-group}

The <NT>Bitrise MCP</NT> server now includes a `registration` API group with two unauthenticated tools — `register` and `verify_registration` — that let an AI agent onboard a brand-new <NT>Bitrise</NT> user without a token. The agent sends a one-time password to the user's email via `register`, then calls `verify_registration` with the OTP to receive a personal access token. These tools are most useful with the remote (HTTP) MCP server. See [Bitrise MCP tools](/en/bitrise-platform/ai/bitrise-mcp/tools) for the full reference.

### <time dateTime="2026-06-15">2026-06-15</time> <NT>Bitrise MCP</NT>: install guides added for six tools {#2026-06-15-bitrise-mcp-install-guides-added-for-six-tools}

<NT>Step</NT>-by-step install guides for the <NT>Bitrise MCP</NT> server are now available for VS Code, Cursor, Claude, Windsurf, Kiro, Gemini CLI, and other Copilot-compatible IDEs. Each guide covers how to configure the MCP server and authenticate with your <NT>Bitrise</NT> token. See [Bitrise MCP](/en/bitrise-platform/ai/bitrise-mcp) to get started.

### <time dateTime="2026-06-11">2026-06-11</time> <NT>Workflow Editor</NT> can now push bitrise.yml directly to the repository {#2026-06-11-workflow-editor-can-now-push-bitrise-yml-directly-to-the-repository}
