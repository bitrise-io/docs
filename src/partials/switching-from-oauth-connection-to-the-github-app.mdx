## Switching from OAuth connection to the <NT>GitHub</NT> app

:::tip[Using the API]

This section describes how to configure the feature on the <NT>Bitrise website</NT>. You can also configure it via the <NT>Bitrise</NT> API: [GitHub app configuration API](/en/bitrise-ci/api/github-app-configuration-api).

:::

If your <NT>Bitrise</NT> <NT><GlossTerm baseform="project">project</GlossTerm></NT> has an OAuth connection set up to your <NT>GitHub</NT> account, you can change it to the <NT>Bitrise</NT> <NT>GitHub</NT> app at any time without losing any functionality. We strongly recommend doing the switch: with the <NT>Bitrise</NT> <NT>GitHub</NT> app installed, you no longer need the <NT>service credential user</NT>, an SSH key, or a webhook to run <NT>Bitrise</NT> builds.

:::important[HTTPS URL required]

Please note that the <NT>GitHub</NT> app requires an HTTPS URL for your repository instead of an SSH one. Normally, you don't have to worry about this: setting up the <NT>GitHub</NT> app connection changes the URL of your project. If there's an issue, you can change the URL manually: [Changing the repository URL](/en/bitrise-platform/projects/configuring-the-repository-url-and-the-default-branch#changing-the-repository-url).

:::

To switch:

1. <NT>Install the Bitrise GitHub App</NT> as described in the relevant guide: [Installing the GitHub App integration](/en/bitrise-platform/repository-access/github-app-integration#installing-the-github-app-integration).
1. Open your project on <NT>Bitrise</NT> with a user that has the **Admin** [role on the project](/en/bitrise-platform/projects/roles-and-permissions-for-bitrise-ci).
1. On the main page of the project, click on the **<NT>Project settings</NT>** button.

   ![project-settings-button.png](/img/_paligo/uuid-14defaa4-472c-2d09-84df-145dc3aef4f5.png)
1. On the left navigation menu, select **Repository**.
1. You can see a blue card with information about <NT>GitHub</NT> App support. To switch, click **<NT>Setup</NT> <NT>GitHub</NT> <NT>App</NT>**.

   ![SCR-20260331-prho.png](/img/_paligo/uuid-3de8dcba-e678-997e-20f0-3055cd889bb0.png)
1. Make sure you remove any pre-existing manual Webhooks from the **<NT>Incoming Webhooks</NT>** page of the **<NT>Webhooks</NT>** tab once the <NT>GitHub</NT> App is configured.

   You can get to this page via **<NT>Project Settings</NT>** → **<NT>Integrations</NT>** → **<NT>Webhooks</NT>** tab → **<NT>Incoming Webhooks</NT>**. If you fail to do this, duplicate builds will get triggered. For more information, see [Event subscriptions](/en/bitrise-platform/repository-access/github-app-integration#event-subscriptions).
1. Optionally, you can remove the <NT>service credential user</NT> and any SSH keys or personal access tokens used for OAuth connection.
