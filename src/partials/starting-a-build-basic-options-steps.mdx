1. Enter the branch you want to run into the **<NT>Source branch</NT>** input field.

   You can choose between typing the name or selecting it from a dropdown menu. Click **<NT>Select branch</NT>** to bring up the dropdown menu.

   You can only set one branch. If a pull request to your repository is made from a forked branch, type the name of that branch to run a build of the pull request.

   :::important[Pull request build]

   Starting a build of a pull request’s branch is NOT the same as a pull request build. The manually started build of a pull request branch will only build the state of the code on that branch. An automatically triggered pull request build, however, builds the state of the code as it will look like once you merged the pull request.

   :::
1. Optionally, enter a build message in the **<NT>Message</NT>** field.

   Please note that entering the Git commit message of a specific commit doesn't <NT><GlossTerm baseform="trigger">trigger</GlossTerm></NT> a build of that commit!
1. From the **<NT>Target</NT>** menu, select a <NT><GlossTerm baseform="workflow">Workflow</GlossTerm></NT> or <NT><GlossTerm baseform="pipeline">Pipeline</GlossTerm></NT>.

   If you don't select one, <NT>Bitrise</NT> will choose a <NT>Workflow</NT> or <NT>Pipeline</NT> based on your app's [trigger setup](/en/bitrise-ci/run-and-analyze-builds/build-triggers/yaml-syntax-for-build-triggers).
